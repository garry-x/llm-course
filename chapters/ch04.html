4.1 从单头到多头

第 3 章实现了单头注意力——一个 维的 Q/K/V 空间。单头的问题是：一个注意力头只能学习一种匹配模式。但在语言中，一个 token 可能需要从多个角度同时匹配其他 token：

  • 句法角度："它"需要找到它指代的名词
  • 语义角度："吃"需要关注"食物"相关的 token
  • 位置角度：当前 token 需要关注其附近的修饰成分

多头注意力（Multi-Head Attention, MHA）通过并行多个独立的注意力头来解决这个问题——每个头学习不同的关注模式：

这里 是头数， 是每头的维度。每个头有自己的 Q/K/V 投影矩阵，并行计算注意力，然后将所有头的输出拼接并通过输出投影 融合。

关键洞察：多头的总参数量与单头（投影到 ）的参数量完全相同！因为 。但多头提供 倍的并行子空间，每个子空间可以专注不同的模式。

🔑
直观理解：单头注意力像是一个"全能专家"，试图用一套标准判断所有关系。多头注意力像是 个专业团队——句法团队、语义团队、位置团队——各司其职，最后将意见汇总（通过 融合）。

4.2 动手：Multi-Head Attention

下面是 MHA 的完整实现。注意我们将 Q/K/V 投影合并为一个大矩阵，用 view 和 transpose 高效分隔出各个头：

import torch import torch.nn as nn import torch.nn.functional as F class MultiHeadAttention(nn.Module): """多头注意力 (MHA) — 合并投影 + 高效头分离。""" def __init__(self, d_model: int, n_heads: int, dropout: float = 0.1): super().__init__() assert d_model % n_heads == 0, "d_model 必须被 n_heads 整除" self.d_model = d_model self.n_heads = n_heads self.d_k = d_model // n_heads # 每头维度 # 合并 QKV 投影: 3 * d_model → d_model (每个投影大小 d_model) self.W_qkv = nn.Linear(d_model, 3 * d_model, bias=False) self.W_o = nn.Linear(d_model, d_model, bias=False) self.dropout = nn.Dropout(dropout) def forward(self, x: torch.Tensor, mask: torch.Tensor = None): """ x: (batch, seq_len, d_model) mask: (batch, 1, 1, seq_len) 或 (batch, 1, seq_len, seq_len) — 可选 返回: (batch, seq_len, d_model), 注意力权重 (batch, n_heads, seq_len, seq_len) """ batch, seq_len, _ = x.shape # 1. 合并投影 + 分离 Q/K/V qkv = self.W_qkv(x) # (b, s, 3*d) qkv = qkv.reshape(batch, seq_len, 3, self.n_heads, self.d_k) q, k, v = qkv.unbind(dim=2) # 各 (b, s, h, d_k) # 2. 头维度前置: (b, h, s, d_k) — 方便并行计算 q = q.transpose(1, 2) # (b, h, s, d_k) k = k.transpose(1, 2) v = v.transpose(1, 2) # 3. 缩放点积注意力 (含 causal mask) scores = torch.matmul(q, k.transpose(-2, -1)) / (self.d_k ** 0.5) # (b, h, s, s) if mask is not None: scores = scores.masked_fill(mask == 0, float('-inf')) attn_weights = F.softmax(scores, dim=-1) # (b, h, s, s) attn_weights = self.dropout(attn_weights) out = torch.matmul(attn_weights, v) # (b, h, s, d_k) # 4. 合并多头: (b, h, s, d_k) → (b, s, h*d_k) = (b, s, d_model) out = out.transpose(1, 2).contiguous().reshape(batch, seq_len, self.d_model) # 5. 输出投影 out = self.W_o(out) return out, attn_weights # 使用示例: GPT-2 124M 配置 d_model, n_heads = 768, 12 mha = MultiHeadAttention(d_model, n_heads) x = torch.randn(2, 16, d_model) mask = torch.tril(torch.ones(16, 16)).unsqueeze(0).unsqueeze(0) out, weights = mha(x, mask) print(f"MHA 输出: {out.shape}") # (2, 16, 768) print(f"注意力权重: {weights.shape}") # (2, 12, 16, 16) print(f"总参数量: {sum(p.numel() for p in mha.parameters()):,}")

输出：

MHA 输出: (2, 16, 768) 注意力权重: (2, 12, 16, 16) 总参数量: 2,359,296

实现要点：

  • 用一个 nn.Linear(d_model, 3*d_model) 一次性完成 QKV 投影，然后用 unbind 分离——这比三个独立的 Linear 层更高效（单个大矩阵乘法比三个小矩阵乘法快，因为 GPU 有更好的并行利用率）。
  • 将头维度通过 transpose 提到第 1 维（batch 之后），这样后续的 matmul 可以自然地在所有头上并行——张量形状 (batch, n_heads, seq, d_k) 比 (batch, seq, n_heads, d_k) 更适合批量矩阵乘法。
  • W_o 输出投影将多头信息融合回 空间——这是多头最后一层"信息汇总"。

4.3 动手：验证计算等价性

前文提到，多头注意力的总参数量和将 维空间直接投影到 的单头注意力参数量相同。我们来验证这一点：

def count_params(model): return sum(p.numel() for p in model.parameters()) d_model = 768 # 单头注意力: 直接投影到 d_model class SingleHeadAttention(nn.Module): def __init__(self, d_model): super().__init__() self.W_qkv = nn.Linear(d_model, 3 * d_model, bias=False) self.W_o = nn.Linear(d_model, d_model, bias=False) def forward(self, x, mask=None): batch, seq, _ = x.shape qkv = self.W_qkv(x) q, k, v = qkv.chunk(3, dim=-1) # 各 (b, s, d_model) # 直接做 d_model 维点积 (不使用缩放, 仅用于参数验证) scores = torch.matmul(q, k.transpose(-2, -1)) if mask is not None: scores = scores.masked_fill(mask == 0, float('-inf')) attn = F.softmax(scores, dim=-1) out = torch.matmul(attn, v) return self.W_o(out) # 12 头注意力 (每头 d_k = 64) mha = MultiHeadAttention(d_model, n_heads=12) sha = SingleHeadAttention(d_model) print(f"MHA 参数量: {count_params(mha):,}") print(f"单头参数量: {count_params(sha):,}") print(f"是否相等: {count_params(mha) == count_params(sha)}") # 但参数量分布不同! qkv_mha = mha.W_qkv.weight.numel() # 768 * 2304 = 1,769,472 qkv_sha = sha.W_qkv.weight.numel() # 768 * 2304 = 1,769,472 print(f"QKV 投影参数相同: {qkv_mha} == {qkv_sha}") # 差异: 单头的 QKV 在 d_model 维度上做点积 # 多头的 QKV 被分到 12 个头, 每个头在 d_k=64 维度上做点积 # 结果再拼接回 d_model print("参数量相同，但计算方式不同——多头提供了 12 个不同的子空间")

输出：

MHA 参数量: 2,359,296 单头参数量: 2,359,296 是否相等: True QKV 投影参数相同: 1,769,472 == 1,769,472

关键区别：虽然参数量相同，但多头的每个头只在一个 维的子空间中做注意力。这有两个效果：

  • 每个头的注意力分数矩阵计算量减少 倍（因为 ，且每个头的计算复杂度包含一项 ）
  • 但每个头独立计算注意力矩阵 ，所以总内存和 倍注意力矩阵的存储成正比

4.4 MHA vs MQA vs GQA

推理时的大问题：KV Cache。在自回归生成中，每步生成一个新 token ，我们需要计算它和所有历史 token 的注意力。为了避免重复计算，所有历史 token 的 K 和 V 向量被缓存下来——这就是 KV Cache。

标准 MHA 中，KV Cache 的大小为：

对于 70B 规模模型（），每个序列的 KV Cache 约 512 MB（FP16）。若并发 1024 个用户，仅 KV Cache 就需要 512 GB——这已经接近高端 GPU 服务器整机显存了。

MQA 和 GQA 的解决方案：减少 KV 头数。

方案Q 头数K/V 头数KV Cache 节省代表模型
MHA（标准）1×（基线）GPT-2、BERT
MQA（Multi-Query）1PaLM、ChatGLM
GQA（Grouped-Query）Llama 2 70B、Mistral、DeepSeek

MQA（Multi-Query Attention）：所有 Q 头共享同一个 K 和 V（即 KV 头数 = 1）。KV Cache 缩小为 。但共享 K/V 可能限制模型表达——毕竟不同头可能需要不同的 K/V 视角。

GQA（Grouped-Query Attention）：折中方案。将 Q 头分为 组，每组共享一个 K/V 头。Llama 2 70B 使用 8 组（Q 头 64，KV 头 8），KV Cache 缩小 8 倍，同时保留了一定的 K/V 多样性。

MQA 可以视为 GQA 在 时的特例。实验表明，GQA 的质量接近 MHA 而推理效率接近 MQA——这是"两全其美"的工程设计。

💡
历史趣闻：MQA 早在 2019 年就由 Shazeer 在 Fast Transformer Decoding 中提出，但直到 2023 年 Llama 2 大规模采用 GQA 后才被广泛关注。这再次说明——好的工程方案有时需要时间才能"出圈"。

4.5 动手：实现 GQA

GQA 的实现与 MHA 非常相似，唯一区别是 K 和 V 的头数少于 Q 的头数。在注意力计算前，我们需要将 KV 头"扩展"到与 Q 头数量匹配：

class GroupedQueryAttention(nn.Module): """分组查询注意力 (GQA) — KV 头数 < Q 头数。""" def __init__(self, d_model: int, n_heads: int, n_kv_heads: int, dropout: float = 0.1): super().__init__() assert d_model % n_heads == 0 assert n_heads % n_kv_heads == 0, "Q 头数必须能被 KV 头数整除" self.d_model = d_model self.n_heads = n_heads self.n_kv_heads = n_kv_heads self.d_k = d_model // n_heads self.n_rep = n_heads // n_kv_heads # 每个 KV 头需要重复几次 # Q: 投影到 n_heads * d_k = d_model self.W_q = nn.Linear(d_model, d_model, bias=False) # K, V: 投影到 n_kv_heads * d_k (更小!) self.W_k = nn.Linear(d_model, n_kv_heads * self.d_k, bias=False) self.W_v = nn.Linear(d_model, n_kv_heads * self.d_k, bias=False) self.W_o = nn.Linear(d_model, d_model, bias=False) self.dropout = nn.Dropout(dropout) def forward(self, x: torch.Tensor, mask: torch.Tensor = None): batch, seq_len, _ = x.shape q = self.W_q(x).reshape(batch, seq_len, self.n_heads, self.d_k) k = self.W_k(x).reshape(batch, seq_len, self.n_kv_heads, self.d_k) v = self.W_v(x).reshape(batch, seq_len, self.n_kv_heads, self.d_k) # 头维度前置 q = q.transpose(1, 2) # (b, n_heads, s, d_k) k = k.transpose(1, 2) # (b, n_kv_heads, s, d_k) v = v.transpose(1, 2) # 关键步骤: 将 KV 头重复扩展以匹配 Q 头 # 例如 n_heads=8, n_kv_heads=4, n_rep=2 # k: (b, 4, s, d_k) → (b, 8, s, d_k) if self.n_rep > 1: k = k.repeat_interleave(self.n_rep, dim=1) v = v.repeat_interleave(self.n_rep, dim=1) # 标准缩放点积注意力 scores = torch.matmul(q, k.transpose(-2, -1)) / (self.d_k ** 0.5) if mask is not None: scores = scores.masked_fill(mask == 0, float('-inf')) attn_weights = F.softmax(scores, dim=-1) attn_weights = self.dropout(attn_weights) out = torch.matmul(attn_weights, v) # 合并头 + 输出投影 out = out.transpose(1, 2).contiguous().reshape(batch, seq_len, self.d_model) out = self.W_o(out) return out, attn_weights # GQA 与 MHA 参数量对比 d_model, n_heads = 4096, 32 # MHA: 32 KV 头 mha = MultiHeadAttention(d_model, n_heads=32) print(f"MHA (n_kv=32) 参数量: {sum(p.numel() for p in mha.parameters()):,}") # GQA: 8 KV 头 (Llama 2 70B 配置) gqa = GroupedQueryAttention(d_model, n_heads=32, n_kv_heads=8) print(f"GQA (n_kv=8) 参数量: {sum(p.numel() for p in gqa.parameters()):,}") print(f"KV 投影节省: {(sum(p.numel() for p in mha.W_qkv.parameters()) - sum(p.numel() for p in [gqa.W_q, gqa.W_k, gqa.W_v])):,}")

输出：

MHA (n_kv=32) 参数量: 67,109,888 GQA (n_kv=8) 参数量: 50,331,648 KV 投影节省: 16,777,216

实现细节：

  • repeat_interleave(dim=1, factor=n_rep) 将每个 KV 头复制 次，使 K/V 的头数与 Q 匹配。这是一个无参数的操作——计算时不同"副本"使用相同的 K/V 数据，但梯度反向传播时会累加。
  • GQA 的参数量减少主要来自 W_k 和 W_v——它们的输出维度从 降为 。
  • 推理时，KV Cache 只存储 维的向量——比 MHA 小 倍。

4.6 DeepSeek MLA 深入

Multi-head Latent Attention (MLA) 是 DeepSeek-V2/V3 的核心创新之一，发表于 2024 年初。它在 GQA 的基础上更进一步——不再只是减少 KV 头数，而是将 KV 存储压缩到一个低维潜在空间（Latent Space）中。

MHA 的 KV Cache 问题：

标准 MHA 的每个头完整存储 K 和 V 向量。即使 GQA 将 KV 头数从 64 减少到 8，每个头仍存储 维向量。能不能将 KV 向量进一步压缩？

MLA 的核心思想：用一个低秩分解压缩 KV 表示：

其中：

  • ：将隐状态 压缩为潜在向量（）
  • ：从潜在向量解压为各头的 K
  • ：V 的对应压缩/解压矩阵

关键优势——KV Cache 只存潜在向量！因为解压操作 和 在注意力计算中可以被吸收到 Q 投影或输出投影中，不需要为每个 token 存储完整 K 和 V。因此 KV Cache 的大小从 降为 。在 DeepSeek-V2 中，，而 ——压缩比 16 倍！

class MLA(nn.Module): """DeepSeek Multi-head Latent Attention (简化版)。""" def __init__(self, d_model: int, n_heads: int, d_latent: int): """ d_model: 模型维度 n_heads: Q 头数 d_latent: 潜在向量维度 (KV cache 实际存储的大小) """ super().__init__() self.d_model = d_model self.n_heads = n_heads self.d_k = d_model // n_heads self.d_latent = d_latent # Q 投影 (标准) self.W_q = nn.Linear(d_model, n_heads * self.d_k, bias=False) # KV 低秩压缩: 共享的压缩矩阵 self.W_dk = nn.Linear(d_model, d_latent, bias=False) # 压缩 K self.W_dv = nn.Linear(d_model, d_latent, bias=False) # 压缩 V # 解压矩阵 (推理时可以被吸收) self.W_uk = nn.Linear(d_latent, n_heads * self.d_k, bias=False) # 解压 K self.W_uv = nn.Linear(d_latent, n_heads * self.d_k, bias=False) # 解压 V # 输出投影 self.W_o = nn.Linear(n_heads * self.d_k, d_model, bias=False) def forward(self, x: torch.Tensor, mask: torch.Tensor = None): batch, seq_len, _ = x.shape # Q: 标准投影 q = self.W_q(x).reshape(batch, seq_len, self.n_heads, self.d_k) q = q.transpose(1, 2) # (b, n_heads, s, d_k) # K, V: 压缩 → 解压 latent_k = self.W_dk(x) # (b, s, d_latent) latent_v = self.W_dv(x) k = self.W_uk(latent_k).reshape(batch, seq_len, self.n_heads, self.d_k) v = self.W_uv(latent_v).reshape(batch, seq_len, self.n_heads, self.d_k) k = k.transpose(1, 2) v = v.transpose(1, 2) # 标准注意力 scores = torch.matmul(q, k.transpose(-2, -1)) / (self.d_k ** 0.5) if mask is not None: scores = scores.masked_fill(mask == 0, float('-inf')) attn_weights = F.softmax(scores, dim=-1) out = torch.matmul(attn_weights, v) out = out.transpose(1, 2).contiguous().reshape(batch, seq_len, self.d_model) out = self.W_o(out) return out, attn_weights, (latent_k, latent_v) # 返回潜在向量用于 KV Cache # MLA 的 KV Cache 大小对比 d_model, n_heads, d_latent = 4096, 64, 512 mha = MultiHeadAttention(d_model, n_heads) mla = MLA(d_model, n_heads, d_latent) def kv_cache_size_per_token(model): """计算每 token 需要缓存的参数量 (不是总参数量)。""" if isinstance(model, MultiHeadAttention): # MHA: 每个 token 需要 n_heads * 2 * d_k 的 KV return 2 * model.d_model # = 2 * n_heads * d_k elif isinstance(model, MLA): # MLA: 每个 token 只需要 2 * d_latent 的 latent return 2 * model.d_latent print(f"MHA 每 token KV: {kv_cache_size_per_token(mha):,} floats") print(f"MLA 每 token KV: {kv_cache_size_per_token(mla):,} floats") print(f"压缩比: {kv_cache_size_per_token(mha) / kv_cache_size_per_token(mla):.1f}x")

输出：

MHA 每 token KV: 8,192 floats MLA 每 token KV: 1,024 floats 压缩比: 8.0x

真正的工程 trick：在推理时，我们并不真正做"压缩→解压"的过程。解压矩阵 可以被吸收到 中：

所以 可以在层间融合，只需要缓存潜在向量 而不是完整的 。这是 MLA 在实现上的精妙所在——通过数学等价变换将压缩"免费"变成了推理加速。

🔑
MLA 的意义：它在 DeepSeek-V2 中首次提出，将 KV Cache 减少了约 75%（相比 MHA），同时保持了模型质量。这使得 DeepSeek-V2/V3 在同等硬件条件下能支持更长的上下文，或者在同等上下文长度下降低推理成本。DeepSeek-V3 的推理成本仅为同规模 Llama 3 的约 1/7，MLA 功不可没。

4.7 MLA 解耦 RoPE 设计

MLA 面临一个绕不开的问题：RoPE 的旋转操作发生在 K 向量上，而 MLA 的 K 向量是从压缩潜在向量解压后才获得的。如果我们将 RoPE 应用在解压后的 K 上，就无法再将 吸收到 中——因为旋转操作与 不可交换（旋转是非线性操作）。

DeepSeek 的解决方案：解耦 RoPE。将 K 分为两部分：

  • ：内容部分——从低维潜在向量解压得到，不包含位置信息，参与矩阵吸收。
  • ：旋转部分——单独从隐状态投影得到，应用 RoPE，不经过低秩压缩。
class MLA_with_RoPE(nn.Module): """MLA 带解耦 RoPE 设计 (简化示意)。""" def __init__(self, d_model: int, n_heads: int, d_latent: int, d_rope: int): super().__init__() self.d_model = d_model self.n_heads = n_heads self.d_k = d_model // n_heads self.d_latent = d_latent self.d_rope = d_rope # Q 内容投影 + Q RoPE 投影 self.W_q = nn.Linear(d_model, n_heads * self.d_k, bias=False) self.W_qr = nn.Linear(d_model, n_heads * d_rope, bias=False) # KV 低秩压缩 (内容部分) self.W_dk = nn.Linear(d_model, d_latent, bias=False) self.W_uk = nn.Linear(d_latent, n_heads * (self.d_k - d_rope), bias=False) # K RoPE (单独投影，不压缩) self.W_kr = nn.Linear(d_model, n_heads * d_rope, bias=False) # V 压缩 self.W_dv = nn.Linear(d_model, d_latent, bias=False) self.W_uv = nn.Linear(d_latent, n_heads * self.d_k, bias=False) self.W_o = nn.Linear(n_heads * self.d_k, d_model, bias=False) self.rope = RoPE(d_rope) # 仅在 d_rope 维度上应用 RoPE def forward(self, x: torch.Tensor, mask: torch.Tensor = None): batch, seq, _ = x.shape n_heads, d_k, d_r = self.n_heads, self.d_k, self.d_rope # Q: 拼接内容和 RoPE 部分 q_c = self.W_q(x).reshape(batch, seq, n_heads, d_k) # 内容 q_r = self.W_qr(x).reshape(batch, seq, n_heads, d_r) # RoPE q = torch.cat([q_c, q_r], dim=-1) # (b, s, n_heads, d_k+d_r) — 简化 q = q.transpose(1, 2) # K: 内容 (解压) + RoPE (单独投影) latent = self.W_dk(x) # (b, s, d_latent) k_c = self.W_uk(latent).reshape(batch, seq, n_heads, d_k - d_r) k_r = self.W_kr(x).reshape(batch, seq, n_heads, d_r) k_r, _ = self.rope(k_r, k_r, None) # 仅 RoPE 部分旋转 k = torch.cat([k_c, k_r], dim=-1) k = k.transpose(1, 2) # V (标准) v = self.W_uv(self.W_dv(x)).reshape(batch, seq, n_heads, d_k) v = v.transpose(1, 2) scores = torch.matmul(q, k.transpose(-2, -1)) / (d_k ** 0.5) if mask is not None: scores = scores.masked_fill(mask == 0, float('-inf')) attn_weights = F.softmax(scores, dim=-1) out = torch.matmul(attn_weights, v) out = out.transpose(1, 2).contiguous().reshape(batch, seq, d_k * n_heads) return self.W_o(out), attn_weights, latent

解耦设计的关键：内容部分 仍然通过低秩压缩/解压，支持矩阵吸收；RoPE 部分 单独投影、单独旋转，不经过压缩。这样既保持了 MLA 的 KV Cache 优势，又获得了 RoPE 的相对位置编码能力。

4.8 注意头行为分析

训练完成的注意力头通常会分化为不同的功能角色。通过分析注意力权重矩阵，研究人员已经发现了多种典型的注意力头模式：

1. 句法头（Syntactic Heads）

这类头的注意力集中在特定句法关系上：

  • 形容词 → 名词（"美丽的" → "花朵"）
  • 动词 → 主语/宾语（"吃" → "猫"/"鱼"）
  • 介词 → 其支配的名词（"在" → "桌子上"）

2. 语义头（Semantic Heads）

这类头更多地关注语义相关的 token，与句法结构关系不大。例如代词"it"的多层注意力会聚焦到它指代的上文名词——即使相隔几十个 token。

3. 位置头（Positional Heads）

这类头主要编码位置关系。常见的模式包括：

  • 相邻偏好：关注前一个或后一个 token（N-gram 模式）
  • 全局偏移：第 个 token 关注第 个 token（如 GPT 中"延迟一位"的预测头）

4. 少样本头（Induction Heads）

Olsson et al. (2022) 发现了一类重要的注意力头——诱导头（Induction Heads）：

输入: [A][B] ... [A] → 诱导头关注 [B] ^ ^ token A 模型预测下一个是 B

这种模式被认为是 In-Context Learning（上下文学习）的核心机制。当模型看到 "apple → fruit, banana →"时，诱导头帮助模型找到 "banana" 之前的模式 "fruit"。

5. 特殊 Token 头

某些头会持续关注 <BOS>、<EOS>、句号等特殊 token。这可能是模型的一种"锚定"策略——特殊 token 作为信息的汇聚点。

💡
头剪枝（Head Pruning）：研究表明，相当比例的注意力头在训练完成后是冗余的——移除它们对模型性能影响很小（Michel et al. 2019）。这也从侧面说明，注意力头数可以适当增加（模型容量），但其中很多头表达相似的模式。GQA 和 MLA 的"头数压缩"思路与这一观察一脉相承。

4.9 本章总结

你已经实现了：

  • ✅ 完整的 Multi-Head Attention（MHA）——多个子空间并行注意力，用 W_o 融合（~60 行）
  • ✅ 验证了 MHA 与单头注意力的参数等价性
  • ✅ 实现了 Grouped-Query Attention（GQA）——减少 K/V 头数，大幅降低 KV Cache（~30 行）
  • ✅ 深入理解了 DeepSeek MLA——用低秩潜在向量压缩 KV，实现 8-16 倍 KV Cache 压缩
  • ✅ 理解了 MLA 的解耦 RoPE 设计——内容部分压缩，RoPE 部分单独处理
  • ✅ 了解了注意力头的功能分化——句法、语义、位置、诱导头等

这条从 MHA 到 GQA 到 MLA 的演进路线，本质上是在模型质量、推理效率、显存约束之间不断寻找更优的帕累托前沿。第 5 章我们将把这些注意力组件与 FFN、RMSNorm 等模块组合，构建完整的 Transformer Block。
