2.1 One-Hot 到稠密嵌入

在第 1 章中，我们完成了文本到 token 序列的转换。假设我们有一个词表大小 ，每个 token 是一个整数索引 。那么，如何将这个整数转化为神经网络可以处理的向量？

最直觉的方案：One-Hot 编码。将 token 映射为一个长度为 的向量 ，其中第 位为 1，其余全是 0。例如词表大小为 5 时，"apple" 对应的 token 3 被编码为 。

但 One-Hot 有严重问题：

  • 维度灾难：词表 5 万，每个 token 就是一个 5 万维向量——计算和存储都不可接受。
  • 语义"零亲"： 和 的余弦相似度永远是 0——"king"和"queen"与"king"和"apple"一样不相似。
  • 无信息量：每个向量只编码了"我是谁"而不编码"我是什么"。

解决方案：可学习的稠密嵌入。用一个矩阵 将 One-Hot 投影到低维语义空间：

其中 （通常 128~8192）远小于 。实际上，这等价于直接"查表"——取 的第 行。这个查表操作就是 Token Embedding，而 就是嵌入矩阵，在训练中反向传播更新。

🔑
核心洞察：嵌入层的本质就是一个可训练的查表操作。嵌入矩阵的每一行就是一个 token 的"身份证"——通过训练，语义相近的 token 会获得相近的向量表示。这就是分布式表示（Distributional Representation）的核心思想。

2.2 动手：nn.Embedding

在 PyTorch 中，嵌入层就是 nn.Embedding(V, d)——一个简单的查表操作：

import torch import torch.nn as nn class TokenEmbedding(nn.Module): """将 token id 映射为稠密向量。""" def __init__(self, vocab_size: int, d_model: int): super().__init__() self.embed = nn.Embedding(vocab_size, d_model) self.d_model = d_model def forward(self, x: torch.Tensor) -> torch.Tensor: """x: (batch, seq_len) → (batch, seq_len, d_model)""" return self.embed(x) * (self.d_model ** 0.5) # 缩放初始化 # 使用示例 vocab_size, d_model = 50000, 768 emb = TokenEmbedding(vocab_size, d_model) x = torch.randint(0, vocab_size, (2, 16)) # batch=2, seq_len=16 out = emb(x) print(f"输入形状: {x.shape} → 输出形状: {out.shape}") print(f"嵌入矩阵大小: {emb.embed.weight.shape}") # [50000, 768] print(f"参数量: {emb.embed.weight.numel():,}")

输出结果：

输入形状: (2, 16) → 输出形状: (2, 16, 768) 嵌入矩阵大小: torch.Size([50000, 768]) 参数量: 38,400,000

注意两个设计细节：

  • 为什么乘以 ？GPT-2 原始实现中这样做了，目的是将嵌入向量的 L2 范数控制在合理范围（约等于 ），避免后续 LayerNorm 被大数值主导。这并非必需，但已成为常见惯例。
  • 嵌入矩阵的"双刃剑"角色：在 GPT 等因果语言模型中，嵌入层和最后的 LM Head（输出投影层）共享权重（weight tying）。这意味着同一矩阵既将 token 映射为向量，也将向量映射回 logits——大量参数被两个地方复用。
💡
参数量记忆：。GPT-2 124M 中 ，嵌入层就有约 3860 万参数——占模型总参数 31%！这也是为什么现代 LLM（如 Llama 3）会扩展 到 4096/8192 的同时控制词表大小。

2.3 余弦相似度与语义方向

嵌入空间最有魅力的特性之一是类比推理。在训练良好的词嵌入空间中（如 Word2Vec 或 GPT 的嵌入层），向量运算能捕捉语义关系：

这个著名结果的背后是余弦相似度——衡量两个向量方向的接近程度：

余弦相似度范围 。1 表示方向完全相同，0 表示正交（无关），-1 表示完全相反。在嵌入空间中，"king"和"queen"的余弦相似度高，"king"和"apple"则接近 0。

为什么向量运算能捕捉语义？因为训练过程（语言建模）迫使嵌入空间具有线性结构：相似的上下文语境产生相似的向量，"性别"、"时态"、"层级"等语义维度被编码为空间中的特定方向。这种线性编码被称为"语言学正则性"——它是分布式表示最显著的经验发现之一。

def cosine_similarity(a: torch.Tensor, b: torch.Tensor) -> torch.Tensor: """计算两个向量之间的余弦相似度""" dot = (a * b).sum(dim=-1) norm_a = a.norm(dim=-1) norm_b = b.norm(dim=-1) return dot / (norm_a * norm_b + 1e-8) # 假设我们训练好的嵌入 king = emb.weight[token_id["king"]] # 向量 man = emb.weight[token_id["man"]] # 向量 woman = emb.weight[token_id["woman"]] # 向量 queen = emb.weight[token_id["queen"]] # 目标向量 analogy = king - man + woman sim = cosine_similarity(analogy.unsqueeze(0), emb.weight) top5 = sim[0].topk(5) print(f"king - man + woman 最接近的 5 个 token: {top5.indices}")

需要注意的是，在 GPT 这样的深层 Transformer 中，底层嵌入本身并不具备完美的类比能力——类比推理往往在模型的中间层表示中更明显。但嵌入层作为"起点"，已经包含了 token 的基本语义信息，高层注意力在此基础上进行复杂的上下文建模。

2.4 为什么需要位置信息？

嵌入层将 "dog" 和 "bite" 映射为两个固定的向量 和 。但 "狗咬人"和"人咬狗"使用完全相同的 token 序列输出相同的向量序列——模型无法区分！

这源自 Self-Attention 的置换等变性（Permutation Equivariance）。更精确地说：

其中 是置换矩阵。注意力操作对输入 token 的顺序是"无感"的——交换两个 token 的位置，输出也只会对应交换，而不会改变每个 token 的输出内容。

这意味着如果没有位置编码：

  • "我打你"和"你打我"在嵌入层眼中完全等价
  • "第 1 个 token"和"第 100 个 token"同样处理——长距离衰减不存在
  • 模型无法利用词序这一最基本的语法信号

解决方案：向每个 token 的嵌入向量中加入位置信息，使模型知道每个 token 在序列中的绝对位置和相对位置。最简单的方法是将嵌入向量与位置向量相加：

其中 是位置 的位置向量。接下来的两节，我们将实现两种主流方案：Sinusoidal Encoding 和 RoPE。

⚠️
重要概念区分：位置编码（Positional Encoding）是固定的函数，不需要训练；位置嵌入（Positional Embedding）是可训练的参数（如 BERT 的做法）。RoPE 并非简单地将位置信息加在嵌入层，而是巧妙地修改注意力计算本身——我们将在 2.6 节详细解释。

2.5 动手：正弦位置编码

Sinusoidal Positional Encoding 由 Vaswani et al. (2017) 提出，是 Transformer 原始论文中使用的位置编码方案。它使用不同频率的正弦和余弦函数为每个位置生成一个唯一向量：

其中 是位置索引， 是维度索引。这是一个精心设计的编码系统：

  • 不同维度有不同的"频率"——低维度（小 ）频率高，变化快，适合编码局部位置；高维度（大 ）频率低，变化慢，适合编码全局位置
  • 任意位置 的编码可以表示为 编码的线性函数——这意味着模型可以轻松学习相对位置关系
  • 无需训练参数，且可以外推到训练时未见过的长度
import torch import math class SinusoidalEncoding(nn.Module): """正弦位置编码——固定函数，无需训练参数。""" def __init__(self, d_model: int, max_len: int = 8192): super().__init__() pe = torch.zeros(max_len, d_model) position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1) # 计算每个维度的频率: 1 / (10000^(2i/d)) div_term = torch.exp( torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model) ) pe[:, 0::2] = torch.sin(position * div_term) # 偶数维度用 sin pe[:, 1::2] = torch.cos(position * div_term) # 奇数维度用 cos self.register_buffer('pe', pe.unsqueeze(0)) # (1, max_len, d_model) def forward(self, x: torch.Tensor) -> torch.Tensor: """x: (batch, seq_len, d_model) → 附加位置编码""" seq_len = x.size(1) return x + self.pe[:, :seq_len, :]

我们来观察不同频率维度的变化模式：

pe = SinusoidalEncoding(d_model=128, max_len=100) pos_encodings = pe.pe.squeeze(0).numpy() # (100, 128) # 观察维度 0 (高频) 和维度 60 (低频) 随位置的变化 import matplotlib.pyplot as plt plt.figure(figsize=(10, 4)) plt.subplot(1,2,1) plt.plot(pos_encodings[:50, 0]) # dim 0: 高频 plt.title("维度 0 — 高频 (快速变化)") plt.subplot(1,2,2) plt.plot(pos_encodings[:50, 60]) # dim 60: 低频 plt.title("维度 60 — 低频 (缓慢变化)") plt.show()

高频维度（dim 0）在相邻几个位置之间就有明显差异——它们帮助模型区分紧邻的 token。低频维度（dim 60）变化缓慢，相隔几十个位置才有明显变化——它们帮助模型感知全局位置。这种"多尺度"设计是 Sinusoidal 编码的精妙之处。

2.6 动手：RoPE（旋转位置编码）

旋转位置编码（Rotary Position Embedding, RoPE）由 Su et al. (2021) 提出，是目前 GPT-NeoX、Llama 2/3、Mistral、DeepSeek 等主流模型的首选位置编码方案。它与 Sinusoidal 有一个根本性区别：不再将位置编码加到嵌入向量上，而是修改注意力分数计算，使 Q 和 K 的点积自动编码相对位置信息。

核心思想：对查询向量 （位置 ）和键向量 （位置 ）施加旋转：

旋转矩阵 是块对角正交矩阵：

其中 。旋转后的 Q 和 K 做点积：

点积结果只依赖于相对位置 ，并且带有位置衰减——距离越远，点积上限越小（因为旋转角度越大，向量对齐越困难）。这正是 RoPE 名字中"旋转"的由来。

class RoPE(nn.Module): """旋转位置编码 (Rotary Position Embedding)。""" def __init__(self, d_model: int, max_len: int = 8192, base: float = 10000.0): super().__init__() self.d_model = d_model self.base = base # 预计算所有位置的旋转角度 theta = torch.pow(base, -torch.arange(0, d_model, 2).float() / d_model) # theta.shape = (d_model/2,) position = torch.arange(max_len).float().unsqueeze(1) # position.shape = (max_len, 1) angles = position * theta.unsqueeze(0) # (max_len, d_model/2) cos = torch.cos(angles) sin = torch.sin(angles) self.register_buffer('cos', cos.unsqueeze(0).unsqueeze(2)) # (1, max_len, 1, d_model/2) self.register_buffer('sin', sin.unsqueeze(0).unsqueeze(2)) def _rotate_half(self, x: torch.Tensor) -> torch.Tensor: """将 x 的后半部分取反并与前半部分交换位置。""" x1 = x[..., :x.shape[-1] // 2] x2 = x[..., x.shape[-1] // 2:] return torch.cat((-x2, x1), dim=-1) def forward(self, q: torch.Tensor, k: torch.Tensor, pos_ids: torch.Tensor = None): """ q, k: (batch, n_heads, seq_len, d_head) pos_ids: (batch, seq_len) — 可选，默认为 0..seq_len-1 返回: 旋转后的 q, k（形状不变） """ batch, n_heads, seq_len, d_head = q.shape if pos_ids is None: pos_ids = torch.arange(seq_len, device=q.device).reshape(1, seq_len) cos = self.cos[:, :seq_len, :, :d_head//2] # (1, seq_len, 1, d_head/2) sin = self.sin[:, :seq_len, :, :d_head//2] # 将 d_head 最后 2 维合并为 (d_head/2)，处理完后再展开 q_reshaped = q.reshape(batch, n_heads, seq_len, d_head // 2, 2) q_complex = q_reshaped[..., 0] + 1j * q_reshaped[..., 1] # 视为复数 # 更高效的纯实数实现： q_out = (q * cos) + (self._rotate_half(q) * sin) k_out = (k * cos) + (self._rotate_half(k) * sin) return q_out, k_out

上述实现的核心是 _rotate_half 函数：它将向量后半部分取反并与前半部分交换。这个操作等价于复数乘法中的 。事实上，RoPE 的数学本质就是将 维向量视为 个复数，每个复数旋转 弧度。

注意 RoPE 的关键特性：

  • 它只修改 Q 和 K，不影响 V（值向量）——因为位置信息只影响"谁关注谁"，而不影响"被关注的内容"
  • 旋转矩阵是正交的——，所以添加位置信息不改变向量本身的范数
  • ——点积只依赖于相对位置，这是理想的归纳偏置

2.7 序列长度外推

一个关键的实际问题：模型在训练时只见过 的序列长度，但推理时用户可能输入 。模型能正确处理吗？

这取决于位置编码的外推能力（Extrapolation）：

  • Learned Positional Embedding（如 BERT）：完全不能外推——训练时未出现的绝对位置索引没有对应的嵌入向量。
  • Sinusoidal：可以外推（函数本身定义在所有位置上），但外推性能随距离增加而下降。
  • RoPE：天然支持一定长度的外推——因为相对位置 在训练中已经见过各种值（虽然绝对值超出范围，但相对差仍在分布内）。

NTK-aware Scaling：一种增强 RoPE 外推能力的技术。核心思想是改变 的 base 值，使高频维度旋转得更慢：

def ntk_scaled_theta(d_model: int, scale: float, base: float = 10000.0): """ NTK-aware 缩放: 对高频维度的 base 进行缩放 scale = max_len_train / max_len_test (一般是 >1) """ # 标准 RoPE 的 theta theta_original = torch.pow(base, -torch.arange(0, d_model, 2).float() / d_model) # NTK-aware 缩放: 高频维度调整更多 theta_scaled = theta_original * (scale ** (torch.arange(0, d_model, 2).float() / d_model)) return theta_scaled # 例: 将 4096 → 8192 的外推 scale = 8192 / 4096 # = 2.0 new_theta = ntk_scaled_theta(d_model=128, scale=scale)

YaRN（Yet another RoPE extensioN）是目前最先进的 RoPE 外推方法，它结合了 NTK-aware 缩放和"温度"调整，在 Llama 和 Mistral 等模型上实现 32x 的外推（从 4K 到 128K 上下文窗口）。

🔑
实际应用：DeepSeek-V2/V3 使用 YaRN 将 4K 训练的模型扩展到 128K 上下文。Mistral 使用了"滑动窗口 + RoPE"的组合——通过将 RoPE 的 base 从 10000 改为 1000000（Llama 3 的做法），显著增强了长距离位置区分能力。base 越大，位置区分度越高，但有效上下文长度也相应缩短。

2.8 位置编码方案对比

方案参数外推能力相对位置代表模型
Sinusoidal0（固定函数）中等隐含（线性变换性质）Transformer 原始
Learned PE无隐含（需模型自己学习）BERT、GPT-2
RoPE0（固定旋转）好（NTK 增强后很好）显式（点积仅依赖差）Llama 2/3、Mistral、DeepSeek

为什么 RoPE 成为主流？虽然 ALiBi 外推能力更强，但 RoPE 具有更好的长距离语义建模能力——它不仅仅是"让注意力衰减"，而是让位置信息自然地融入表示空间。在许多实际基准上，RoPE + NTK/YaRN 方案的语义理解分数优于 ALiBi。

💡
选择指南：如果你追求简洁和强外推，用 ALiBi。如果你追求最佳下游性能且有适度外推需求，用 RoPE + YaRN。如果你资金充裕且序列长度可控，Learned PE 也是简单可用的方案。实际上，2024-2025 年的主流模型几乎全部选择了 RoPE。
