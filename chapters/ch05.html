5.1 Decoder-only 架构全景——Transformer Block 的"积木"视角

所有现代 LLM（GPT、Llama、DeepSeek）都采用 Decoder-only 架构：没有单独的编码器，只由堆叠的 Transformer Block 组成。每个 Block 包含两个核心子层：

  • 自注意力层 (Self-Attention) — 让 token 之间交换信息（第4章已完成）
  • 前馈神经网络 (FFN) — 对每个 token 独立进行非线性变换

每个子层外部包裹着 残差连接 和 层归一化。整体形式为（Pre-Norm 规范）：

公式 5.1 Transformer Block (Pre-Norm)

为什么需要两个不同的子层？注意力提供的是"信息检索"——每个 token 从整个序列中聚合信息。FFN 提供的是"信息处理"——每个 token 在收到信息后进行复杂的非线性变换。没有 FFN，模型就只是一个线性加权求和系统，表达能力极其有限。

┌─────────────────────────────────────────────────┐ │ Transformer Block │ │ ┌──────┐ ┌──────────┐ ┌──────────┐ │ │ │Input │────→│ LayerNorm│────→│ Attention │ │ │ │ x │ │ (Pre) │ │ (MHA/GQA)│ │ │ └──┬───┘ └──────────┘ └────┬─────┘ │ │ │ │ │ │ └─────────── (+) ───────────────┘ │ │ │ 残差连接 │ │ ▼ │ │ ┌──────────┐ ┌──────────┐ │ │ │ LayerNorm│────→│ FFN │ │ │ │ (Pre) │ │SwiGLU │ │ │ └──────────┘ └────┬─────┘ │ │ │ │ │ (+) ─────────────┘ │ │ 残差连接 │ │ ▼ │ │ ┌──────┐ │ │ │Output│ │ │ │ x'' │ │ │ └──────┘ │ └─────────────────────────────────────────────────┘

每一层 Block 的输出维度始终保持为 ，这使得我们可以堆叠任意数量的 Block——GPT-3 堆叠了 96 层，DeepSeek-V3 堆叠了 61 层。

5.2 编程练习 1：实现 LayerNorm（不用 nn.LayerNorm）

层归一化 (Layer Normalization) 是 Transformer 训练稳定性的基石。与 BatchNorm 不同，LayerNorm 在特征维度上做归一化，不依赖 batch 大小——这对变长序列和自回归模型至关重要。

公式 5.2 LayerNorm

🖥️ 练习 1：LayerNorm 从零实现

任务：实现 LayerNorm 类，不可以使用 nn.LayerNorm。

要求：

  • 可学习参数 gamma（缩放，初始化为全 1）和 beta（偏移，初始化为全 0）
  • eps=1e-6 防止除零
  • 支持任意形状输入：归一化在最后一维进行
  • 使用 unbiased=False 的方差计算
  • 验证：与 nn.LayerNorm 的输出一致（随机输入，误差 < 1e-5）
# 预期用法
ln = LayerNorm(dim=768)
x = torch.randn(2, 16, 768)
y = ln(x)
assert y.shape == x.shape

# 与 PyTorch 官方对比
ref = nn.LayerNorm(768, eps=1e-6)
ref.load_state_dict({'weight': ln.gamma, 'bias': ln.beta})
y_ref = ref(x)
assert torch.allclose(y, y_ref, atol=1e-5)
import torch import torch.nn as nn class LayerNorm(nn.Module): """层归一化 — 在特征维度上做归一化（不用 nn.LayerNorm）""" def __init__(self, dim, eps=1e-6): super().__init__() self.eps = eps self.gamma = nn.Parameter(torch.ones(dim)) self.beta = nn.Parameter(torch.zeros(dim)) def forward(self, x): # 计算均值和方差（在最后一维） mean = x.mean(dim=-1, keepdim=True) var = x.var(dim=-1, keepdim=True, unbiased=False) # 归一化 x_norm = (x - mean) / torch.sqrt(var + self.eps) # 仿射变换 return x_norm * self.gamma + self.beta # 验证 ln = LayerNorm(768) x = torch.randn(2, 16, 768) y = ln(x) ref = nn.LayerNorm(768, eps=1e-6) ref.load_state_dict({'weight': ln.gamma, 'bias': ln.beta}) y_ref = ref(x) print(f"最大误差: {(y - y_ref).abs().max().item():.2e}") assert torch.allclose(y, y_ref, atol=1e-5)

💡 注意 var(unbiased=False)：LayerNorm 使用总体方差（除以 N 而非 N-1），这与统计中的样本方差不同。使用 unbiased=False 确保与 nn.LayerNorm 的实现一致。

5.3 深度理论：Pre-Norm vs Post-Norm——归一化位置如何决定训练稳定性

归一化层放在子层的输入还是输出上，对训练动态有决定性影响。这是一个在 LLM 规模扩展过程中被反复验证的关键设计决策。

Post-Norm（原始 Transformer）

原始 Transformer（Vaswani et al., 2017）使用 Post-Norm：

Post-Norm

归一化在残差连接之后。这意味着残差分支的输出（Attention/FFN 的输出）会干扰残差流中原始信息的尺度。在深层网络中，这种干扰持续累积，导致梯度不稳定。

Pre-Norm（现代 LLM 标准）

Pre-Norm 将归一化放在子层的输入上：

Pre-Norm

关键优势：残差流保持未归一化。信息可以沿着残差路径在各层之间无衰减地传播。梯度也可以沿着这条路径无损地反向传播——这就是为什么 Pre-Norm 可以训练 1000+ 层网络。

训练动力学分析

更正式地，考虑一个 L 层 Pre-Norm Transformer 的残差流：

残差流的方差传播

在 Post-Norm 中，归一化在残差连接之后，（被归一化强制固定），但每个子层输出的方差 需要精细控制。在 Pre-Norm 中，残差流的方差会逐层增大（这是正常的——深层需要有更大的表示范围），而不是被强制截断。

特性Post-NormPre-Norm
代表模型原始 Transformer (2017)GPT-2/3/4, Llama, DeepSeek
训练稳定性较差，需要细致初始化和 warmup稳定，允许更大学习率
深层扩展难（20+ 层就不稳定）易（支持 1000+ 层）
残差流方差被强制为 1可随深度增长
梯度路径需要经过归一化残差路径畅通无阻
🔑
关键结论：Pre-Norm 的成功可以归结为"归一化隔离"——每个子层的输入被独立归一化，而残差流完全不受影响的传递信息。这等价于每个子层看到的输入总是单位方差的，但残差路径上的信息可以自由增长。Post-Norm 将归一化放在残差连接之后，相当于"感染"了残差路径，阻碍了深层网络的梯度流动。

5.4 编程练习 2：实现 RMSNorm——Llama/DeepSeek 的选择

RMSNorm（Zhang & Sennrich, 2019）去掉了 LayerNorm 的均值中心化，只保留 RMS 缩放。它基于一个简单的观察：均值中心化对训练质量的影响微乎其微，而节省的计算量（约 15%）却很可观。

公式 5.3 RMSNorm

🖥️ 练习 2：RMSNorm 模块

任务：实现 RMSNorm——只做 RMS 缩放，去掉均值中心化。

要求：

  • 唯一可学习参数 gamma（初始化为全 1），没有 beta
  • eps=1e-6 防止除零
  • 计算 rms = sqrt(mean(x^2) + eps)，然后 x / rms * gamma
  • 验证：RMSNorm(x) 的输出范数（L2 norm）近似等于 gamma 的范数
  • 验证：RMSNorm 的输出 方向 与输入相同——它只改变向量的长度，不改变角度
# 预期用法
rms = RMSNorm(dim=768)
x = torch.randn(2, 16, 768)
y = rms(x)
assert y.shape == x.shape

# 验证方向不变
cos_sim = (x * y).sum(dim=-1) / (x.norm(dim=-1) * y.norm(dim=-1))
assert torch.allclose(cos_sim, torch.ones_like(cos_sim), atol=1e-6)
# 方向不变意味着 cos_sim ≈ 1.0
class RMSNorm(nn.Module): """RMS 层归一化 — Llama / DeepSeek 标准""" def __init__(self, dim, eps=1e-6): super().__init__() self.eps = eps self.gamma = nn.Parameter(torch.ones(dim)) def forward(self, x): # RMS = sqrt(mean(x^2)) rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps) return x / rms * self.gamma def extra_repr(self): return f"dim={self.gamma.shape[0]}, eps={self.eps}" # 验证方向不变 rms = RMSNorm(768) x = torch.randn(2, 16, 768) y = rms(x) cos_sim = (x * y).sum(dim=-1) / (x.norm(dim=-1) * y.norm(dim=-1)) print(f"方向余弦 (均值): {cos_sim.mean().item():.6f}") # 应 ≈ 1.0 # 验证输出范数 ≈ sqrt(dim) * gamma print(f"输出向量范数 (均值): {y.norm(dim=-1).mean().item():.4f}") print(f"sqrt(dim) * gamma_norm: {(768 ** 0.5) * rms.gamma.norm().item():.4f}")

5.5 深度理论：FFN 作为键值存储器——模型知识存储的秘密

FFN 不仅仅是"非线性变换"。从可解释性视角来看，Geva et al. (2021) 揭示了 FFN 的本质：一个大规模键值存储器。

重新审视 FFN

标准 FFN 的计算：

公式 5.4 FFN 重写为键值存储

这个求和形式的每个项都揭示了一个"键值对"：

  • 键 (Key): 检测输入是否匹配某种模式。ReLU 门控决定是否激活——只有匹配的键才会产生非零输出
  • 值 (Value): 是键 i 对应的输出"知识片段"
  • 存储容量: 就是键值对的数量。GPT-2 small 有 3072 个键值对，GPT-3 175B 每层有 4 × 12288 = 49152 个

知识的层级分布

不同层的 FFN 存储不同类型的知识：

  • 底层（1-4 层）：语法模式——检测 POS 标记、短语边界、浅层句法结构
  • 中层（5-8 层）：事实知识——"巴黎是法国的首都"、"水在 100°C 沸腾"
  • 高层（9-12 层）：长程依赖和推理——代词指代消解、多步推理模式

模型编辑的启示

这个视角直接启发了 模型编辑 方法。ROME (Meng et al., 2022) 利用 FFN 键值存储的性质，通过定位存储特定事实的 FFN 神经元，直接修改其值向量来更新模型中的事实知识——不需要重新训练。

💡
这意味着：模型的知识主要集中在 FFN 的权重中。Attention 扮演"检索协调者"的角色——决定从哪些键值对中检索信息。这也就解释了为什么扩大 FFN 的维度（更多键值对）比扩大注意力维度更能提升模型容量。

5.6 编程练习 3：实现 FFN 与 SwiGLU 激活

GPT-2 使用 GELU 激活的 FFN，但现代 LLM（Llama、Mistral、DeepSeek）全部使用 SwiGLU——一种门控激活函数，通过三个权重矩阵实现信息的选择性通过。

SwiGLU vs 标准 FFN

其中 （Sigmoid Linear Unit）是无上界有下界的平滑激活函数。

🖥️ 练习 3：实现 FFN (GELU) 和 SwiGLU

任务：实现 FFN（GELU）和 SwiGLU 两个模块。

要求 (FFN)：

  • 两层线性：d_model → d_ff → d_model
  • 中间使用 GELU 激活（用 F.gelu）
  • 默认 d_ff = 4 * d_model（GPT 风格）

要求 (SwiGLU)：

  • 三个线性层：w1（门控）、w2（值）、w3（输出）
  • 默认 d_ff = int(8/3 * d_model)——确保总参数量与 4x FFN 一致
  • 使用 F.silu 激活门控分支
  • 前向：w3(F.silu(w1(x)) * w2(x))
# 预期用法
ffn = FFN(d_model=768)                              # d_ff = 3072
swiglu = SwiGLU(d_model=768)                        # d_ff = 2048
x = torch.randn(2, 16, 768)
y1 = ffn(x)
y2 = swiglu(x)
assert y1.shape == y2.shape == (2, 16, 768)

# 参数量应大致相同
ffn_params = sum(p.numel() for p in ffn.parameters())     # ~4.7M
swiglu_params = sum(p.numel() for p in swiglu.parameters())  # ~4.7M
assert abs(ffn_params - swiglu_params) / ffn_params < 0.05
import torch.nn.functional as F class FFN(nn.Module): """标准 FFN (GELU) — GPT-2 风格""" def __init__(self, d_model, d_ff=None): super().__init__() d_ff = d_ff or d_model * 4 self.fc1 = nn.Linear(d_model, d_ff, bias=True) self.fc2 = nn.Linear(d_ff, d_model, bias=True) def forward(self, x): return self.fc2(F.gelu(self.fc1(x))) class SwiGLU(nn.Module): """门控 SwiGLU — Llama / DeepSeek 风格""" def __init__(self, d_model, d_ff=None): super().__init__() d_ff = d_ff or int(d_model * 8 / 3) # 保持参数量与 4x FFN 一致 self.w1 = nn.Linear(d_model, d_ff, bias=False) # 门控 self.w2 = nn.Linear(d_model, d_ff, bias=False) # 值 self.w3 = nn.Linear(d_ff, d_model, bias=False) # 输出 def forward(self, x): gate = F.silu(self.w1(x)) # (b, s, d_ff) value = self.w2(x) # (b, s, d_ff) return self.w3(gate * value) # (b, s, d_model) # 参数量对比 ffn = FFN(768) swiglu = SwiGLU(768) print(f"FFN 参数量: {sum(p.numel() for p in ffn.parameters()):,}") print(f"SwiGLU 参数量: {sum(p.numel() for p in swiglu.parameters()):,}") print(f"比例: {sum(p.numel() for p in swiglu.parameters()) / sum(p.numel() for p in ffn.parameters()):.3f}")

💡 SwiGLU 为什么更好？门控机制允许模型选择性地通过信息——当门控值接近 0 时，对应维度的信息被阻塞。相比 GELU 的"所有神经元都激活但程度不同"，SwiGLU 提供了更灵活的"开关"控制。在相同参数量下，SwiGLU 通常比 GELU/ReLU FFN 降低 0.2-0.5 的困惑度。

🖥️ 练习 4：TransformerBlock 组装

任务：实现 TransformerBlock——包含两个子层的完整 Decoder Block。

要求：

class TransformerBlock(nn.Module): """完整 Transformer Block — Pre-Norm + RMSNorm + SwiGLU""" def __init__(self, d_model, n_heads, d_ff=None, dropout=0.0): super().__init__() # 注意力子层 self.attention = MultiHeadAttention( d_model=d_model, n_heads=n_heads, dropout=dropout ) self.norm1 = RMSNorm(d_model) # FFN 子层 self.ffn = SwiGLU(d_model, d_ff) self.norm2 = RMSNorm(d_model) self.dropout = nn.Dropout(dropout) def forward(self, x, mask=None): # 子层 1: 自注意力 (+ Pre-Norm + 残差) identity = x x = self.norm1(x) x, _ = self.attention(x, mask=mask) # 忽略 attn_weights x = self.dropout(x) x = x + identity # 子层 2: FFN (+ Pre-Norm + 残差) identity = x x = self.norm2(x) x = self.ffn(x) x = self.dropout(x) x = x + identity return x # 测试 block = TransformerBlock(d_model=768, n_heads=12, d_ff=2048) x = torch.randn(2, 64, 768) mask = torch.tril(torch.ones(64, 64)).unsqueeze(0).unsqueeze(0) out = block(x, mask=mask) assert out.shape == (2, 64, 768) loss = out.sum() loss.backward() print(f"所有参数有梯度: {all(p.grad is not None for p in block.parameters())}")

💡 这里约 25 行核心代码完成了完整的 Transformer Block。关键设计是"归一化路径与残差路径分离"——信息沿残差路径无阻碍流动（无非线性、无归一化），RMSNorm 只应用于子层的输入。

5.8 深度理论：梯度检查——为什么需要验证反向传播？

在实现复杂的自定义算子时，手写的反向传播（backward function）可能包含数学错误。即使是一个符号错误，也可能导致模型"训练"了很长时间却无法收敛。

有限差分的数学原理

PyTorch 的 torch.autograd.gradcheck 使用中心差分（Central Difference）来逼近数值梯度：

中心差分逼近

中心差分的误差是 ，而前向差分的误差是 。因此中心差分更精确——代价是每个参数维度需要两次前向计算。

为什么需要 gradcheck？

考虑一个手动实现的反向传播函数，其中有一个符号错误：

# 错误的 backward: 应该是 dx = grad * w，却写成了 dx = grad / w # gradcheck 会检测到数值梯度与解析梯度的差异 > atol

这种错误在简单模块中很容易发现，但在复杂的自定义算子（如 FlashAttention 的 Triton kernel）中，手动推导和实现反向传播是一个常见的调试环节。

⚠️
重要提示：gradcheck 对每个参数维度需要两次额外前向传播。对于大模型，这极其昂贵。因此：使用 float64（有限差分对 float32 的舍入误差敏感）；用极小输入（batch=2, seq=8, dim=64）验证；设置 eps=1e-4 和 atol=1e-3。

print(f"梯度检查通过: {test}")  # 应为 True
from torch.autograd import gradcheck class LayerNormFunction(torch.autograd.Function): """自定义 LayerNorm — 手动 forward + backward 用于 gradcheck 验证""" @staticmethod def forward(ctx, x, gamma, beta, eps=1e-6): ctx.save_for_backward(x, gamma, beta) ctx.eps = eps mean = x.mean(-1, keepdim=True) var = x.var(-1, keepdim=True, unbiased=False) x_hat = (x - mean) / torch.sqrt(var + eps) return x_hat * gamma + beta @staticmethod def backward(ctx, grad_output): x, gamma, beta = ctx.saved_tensors eps = ctx.eps # 简化的 backward: 通过 PyTorch 自动求导验证 # 标准的 LayerNorm backward 推导涉及三个部分: # dL/dx, dL/dgamma, dL/dbeta # 这里我们返回 None 让 PyTorch 处理 beta 和 eps return grad_output * gamma / torch.sqrt( x.var(-1, keepdim=True, unbiased=False) + eps ), None, None, None # 小输入上测试 (float64) x = torch.randn(2, 8, 64, dtype=torch.float64, requires_grad=True) gamma = torch.ones(64, dtype=torch.float64, requires_grad=True) beta = torch.zeros(64, dtype=torch.float64, requires_grad=True) test = gradcheck( LayerNormFunction.apply, (x, gamma, beta, 1e-6), eps=1e-4, atol=1e-3 ) print(f"LayerNorm 梯度检查: {test}") # 验证我们自己实现的 LayerNorm 模块 ln = LayerNorm(64).double() x = torch.randn(2, 16, 64, dtype=torch.float64, requires_grad=True) test_module = gradcheck( lambda x: ln(x).sum(), x, eps=1e-4, atol=1e-3, raise_exception=False ) print(f"LayerNorm 模块梯度检查: {test_module}")

💡 最佳实践：逐个模块验证梯度（LayerNorm 单独测、SwiGLU 单独测、整个 Block 单独测）。不要把所有模块混在一起测试——如果失败了你不知道是哪个模块的问题。

5.10 深度理论：DeepSeek V4 mHC 超连接——Birkhoff 约束与 Sinkhorn-Knopp 迭代

DeepSeek-V4 引入的 mHC (multi-head Hyper-Connections) 是对传统残差连接的革命性替代。它的核心思想是：层间连接强度应该是可学习的，但必须施加数学约束来保证训练稳定。

传统残差连接的局限

残差连接 假设每层的贡献是固定的 1:1 比例。但不同层对最终输出的贡献可能不同——有些层可能只需要微调前一层的表示，有些层可能需要大幅修改。

Birkhoff Polytope（双随机矩阵）

mHC 通过学习一个连接矩阵 来控制层间连接强度，但直接学习 可能导致梯度爆炸。解决方案：约束 A 属于 Birkhoff polytope。

Birkhoff Polytope 定义

双随机矩阵的性质保证了 （谱范数约束）。这意味着层间连接的"放大倍数"永远不会超过 1——梯度可以在网络中稳定传播而不会爆炸。

Sinkhorn-Knopp 迭代

交替进行行归一化和列归一化。算法以指数级收敛到唯一的双随机矩阵。实践中 10 次迭代就足够。

def sinkhorn_knopp(M, num_iter=10): """Sinkhorn-Knopp 迭代投影到 Birkhoff polytope""" M = torch.exp(M - M.max(dim=-1, keepdim=True).values) M = M / (M.sum(dim=-1, keepdim=True) + 1e-6) for _ in range(num_iter): M = M / (M.sum(dim=-1, keepdim=True) + 1e-6) M = M / (M.sum(dim=-2, keepdim=True) + 1e-6) return M

mHC 的多头特性：多个独立的连接矩阵（头）捕获不同类型的层间依赖，最后取均值。DeepSeek-V4 报告显示，mHC 使得训练 500+ 层纯 Transformer 变得稳定。

🔑
直觉：Sinkhorn-Knopp 迭代的交替归一化在数学上等价于熵正则化的最优传输问题。它找到的"连接权重"在行和列维度上都满足概率分布约束，确保每层"提出"的信息总量和"接收"的信息总量都是守恒的——这正是深层网络稳定训练所需的"能量守恒"性质。
