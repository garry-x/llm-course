# Dataset, Model, and Artifact Provenance Registry

复核日期：2026-06-05

本登记表把课程作业、capstone、评测脚本、HTML 发布和学生项目可能用到的数据集、合成语料、tokenizer、模型 artifact、checkpoint、runtime asset 与评测集合集中管理。它补充 [Data and Ethics Review](data-ethics-review.md)、[External Source Inventory](external-source-inventory.md)、[Environment and Reproducibility Guide](environment-reproducibility.md)、[Experimental Rigor and Evaluation Statistics Guide](experimental-rigor-evaluation-statistics.md)、[Project Report Template and Reproducibility Checklist](project-report-template.md)、[Project Submission Dossier](project-submission-dossier.md)、[Paper-to-Code Traceability Matrix](paper-to-code-traceability-matrix.md) 和 [Compute Resource and Cost Guide](compute-resource-guide.md)。

本文件不替代论文引用或模型卡引用。它回答的是资产级问题：这个 artifact 从哪里来、课程如何使用、是否能发布、是否有 PII 或 contamination 风险、学生提交时需要提供什么证据。

## Artifact Type Taxonomy

| artifact_type | 定义 | 课程例子 | 默认风险 |
|---------------|------|----------|----------|
| dataset | 可训练、验证、测试或检索的文本、表格、图像或多模态数据 | 训练 capstone sample corpus、自定义项目数据 | medium |
| synthetic_fixture | 由课程手工构造的 toy sample、tensor、prompt 或标签 | Ch01-Ch06 单元测试 fixture | low |
| evaluation_set | 固定评测、benchmark、hidden-safe public sample 或 SLO case | Ch10 API cases、Ch11 指标例子 | medium |
| tokenizer | BPE、char vocab、chat template 或第三方 tokenizer 定义 | Ch01 BPE merges、Ch07 char vocab、Ch09 TinyChatTokenizer | low |
| model_artifact | 权重、checkpoint、optimizer state、scheduler state、LoRA adapter 或 model card | 训练 capstone `latest.pt`、学生自带模型卡 | medium |
| runtime_asset | 支持展示或运行但不支撑课程事实的资源 | KaTeX CDN、badge、本地服务 URL | low |
| generated_sample | 模型、脚本或人工模板生成的示例输出 | project exemplar synthetic reports、demo outputs | medium |
| student_project_data | 学生团队自带或收集的数据、日志、API 输出、RAG 文档 | final project data package | high |

## Registry Schema

每个 artifact 记录必须包含以下字段；学生项目可复制该 schema 放入 `artifact_manifest.md` 或报告附录。

| 字段 | 必填内容 |
|------|----------|
| artifact_id | 稳定 ID，例如 `ART-CH07-SAMPLE-CORPUS` |
| artifact_type | taxonomy 中的类型 |
| course_use | 使用位置和评分用途 |
| storage_location | 仓库路径、外部 URL、LMS 位置或 `student-submitted` |
| source_or_origin | 手工构造、脚本生成、公开来源、第三方模型卡或团队收集 |
| license_or_access | MIT、public web terms、course-internal、student-declared 等 |
| pii_risk | none、low、medium、high |
| contamination_risk | none、low、medium、high |
| student_action | 学生提交或引用时必须做什么 |
| review_gate | staff 或 verifier 检查动作 |

## License and Access Rules

| rule_id | 规则 | 不通过例子 |
|---------|------|------------|
| license_gate | 数据、模型、tokenizer 和 runtime asset 必须有 license、terms 或 course-authored 声明 | 使用公开网页大规模抓取语料但没有条款说明 |
| redistribution_gate | 不能把不允许再分发的数据、权重、日志或私有 API 输出提交到公开仓库 | 把受限数据集原文放入 GitHub |
| attribution_gate | 外部模型卡、API 文档、benchmark 和课程页面只能按来源等级引用，不能当作无条件事实 | 写“某模型绝对更强”但只给厂商宣传页 |
| local_fixture_gate | 课程内置 synthetic fixture 只能支撑格式、形状、公式或流程验收 | 用 toy prompt pass rate 宣称真实用户质量 |
| runtime_asset_gate | CDN、badge、本地 URL 和浏览器资源不支撑课程事实 | 用 KaTeX 或 badge 链接当作内容来源 |

## Contamination and Privacy Gates

| gate_id | 检查动作 | 最低证据 |
|---------|----------|----------|
| pii_gate | 检查姓名、邮箱、电话、地址、账号、病历、财务信息、聊天记录和可识别组合字段 | data ethics review 中列出字段和处理结果 |
| contamination_gate | 检查 train/dev/test 重复、near duplicate、prompt leakage、retrieval contamination 和 benchmark contamination | split_card、leakage_check 或人工抽查日志 |
| split_gate | 明确 train_source、dev_policy、test_policy 和冻结时间 | experimental rigor guide 中的 split card |
| hidden_test_gate | 不公开 hidden tests 输入、seed、断言和评分脚本细节 | release builder 排除私有文档 |
| artifact_size_gate | 大 checkpoint、权重、日志和数据包必须说明存储位置和保留周期 | compute guide 和 archive policy 中的 retention rule |
| model_card_gate | 第三方模型或 API 必须记录版本、访问日期、context、限制和安全说明 | report 中有模型卡或官方文档链接 |
| safety_boundary_gate | 高影响场景必须说明拒答、人工复核和残余风险 | data ethics review 和 project rubric 同时覆盖 |

## Student Submission Requirements

学生最终项目必须在报告或附录中包含 `artifact_manifest`。最低字段：

```text
artifact_id:
artifact_type:
course_use:
storage_location:
source_or_origin:
license_or_access:
pii_risk:
contamination_risk:
student_action:
review_gate:
```

项目只使用课程内置 toy corpus 时，也必须列出 `ART-CH07-SAMPLE-CORPUS` 或相应 fixture，并说明它只能支撑 smoke test、格式测试或工程流程，不能支撑泛化质量 claim。

项目使用外部模型、API、embedding、reranker、benchmark 或 RAG 文档时，必须同时提交：

- data/source provenance：链接、版本、访问日期、许可证或 terms。
- split_card：train/dev/test 或 prompt tuning/final eval 的用途区分。
- model_card_record：模型名、提供方、版本、访问日期、context 和限制。
- leakage_check：重复、prompt 泄漏、RAG 答案泄漏或 benchmark contamination 检查。
- artifact_retention：哪些日志、metrics、checkpoint 或数据包可公开、私有保留或删除。

## Staff Review Workflow

| 阶段 | review_action | 输出 |
|------|---------------|------|
| proposal | 检查 `ART-PROJECT-STUDENT-DATA` 是否有来源、许可证和 PII 初筛 | approve、revise 或 reject data source |
| milestone | 抽查 split_card、data audit、model card 和 checkpoint/log 对齐 | required changes before final |
| final report | 对照 registry、data ethics、experimental rigor 和 rubric 检查 claim 强度 | grading notes with artifact IDs |
| archive | 按 showcase/archive policy 检查 consent、redaction、artifact_manifest 和 archived_label | public-safe archive package |

## Release Checklist

| 检查项 | 通过标准 |
|--------|----------|
| registry coverage | 课程公开作业、capstone、runtime asset 和学生项目通用 artifact 均有 ID |
| schema completeness | registry rows 包含 artifact_id、type、use、location、origin、license、PII、contamination、student_action 和 review_gate |
| linked governance | README、data ethics、external source inventory、environment guide、project report template 和 experimental rigor guide 均链接本文件 |
| verifier gate | `verify_course.py` 检查 taxonomy、required IDs、risk gates、链接和表格结构 |
| release builder | student-facing site 包含本文件，私有 hidden tests 和 staff-only artifacts 仍被排除 |
| evidence manifest | evidence manifest 记录本文件及核心 markers |
