3.1 为什么需要 Attention：RNN 的限制和突破点

Transformer 之前，序列建模长期由 RNN/LSTM 主导。RNN 的做法是逐步读入 token，把历史信息压进 hidden state：

h_t = tanh(W_h h_{t-1} + W_x x_t + b)

这个结构有两个硬限制：

  • 串行瓶颈：算 h_t 必须先有 h_{t-1}。1000 个 token 就有 1000 个时间步，很难像 Transformer 那样并行处理整段序列。
  • 长程依赖衰减：早期信息要经过很多次非线性变换才能传到后面，梯度容易消失或爆炸。LSTM 的门控能缓解，但没有彻底去掉串行路径和固定维度状态瓶颈。

Attention 的突破是：不要强迫信息一级级传递。每个位置都可以直接读取同一层里的其他位置。任意两个位置之间的信息路径在单层里是 O(1)，这让长程依赖和并行训练都更容易。

3.2 Q/K/V 三元组：Attention 的检索接口

Scaled Dot-Product Attention 可以类比数据库检索：

  • Q Query：我现在想找什么。
  • K Key：我能被别人怎样找到。
  • V Value：真正要被取走的信息内容。

同一个 token 在“找信息”和“被别人找”时角色不同，所以 Q 和 K 要分开。比如处理句子里的动词时，它作为 Query 可能想找主语；作为 Key 时，它可能告诉别的位置“我是动作”。Q/K 的不对称性让 attention 能学到有方向的依赖。

从输入到 Q、K、V

给定输入序列 X ∈ R^{n x d}，Q/K/V 都来自同一个 X，但经过三组不同的可训练线性投影：

Q = X W_Q K = X W_K V = X W_V

“self-attention”的 self 就在这里：Q、K、V 都来自同一个序列，每个 token 和自己以及其他 token 建立关系。

3.3 编程练习 1：实现 QKV 线性投影

Attention 的第一步，是把输入 x: [batch, seq_len, d_model] 投影成 Q、K、V。

练习 1：QKVProjection Module

  • 使用三个 nn.Linear，分别得到 Q/K/V。
  • 支持配置 d_model 和 d_k。
  • 返回三个 tensor，shape 都是 [batch, seq_len, d_k]。
class QKVProjection(nn.Module): def __init__(self, d_model, d_k): super().__init__() self.W_q = nn.Linear(d_model, d_k, bias=False) self.W_k = nn.Linear(d_model, d_k, bias=False) self.W_v = nn.Linear(d_model, d_k, bias=False) def forward(self, x): return self.W_q(x), self.W_k(x), self.W_v(x)

3.3A 例子：Q/K/V 的 shape 和参数量

设 B=2、T=3、d_model=4、d_k=2。输入 X 的 shape 是 [2,3,4]，三组 projection weight 都是 [4,2]。

Q = X @ W_Q -> [2, 3, 2] K = X @ W_K -> [2, 3, 2] V = X @ W_V -> [2, 3, 2]

参数量是 3 * d_model * d_k = 24。batch 和 sequence length 不是参数，只是同一组权重被应用了多少次。后续 QK^T 会得到 [B,T,T] 的 token-to-token score matrix。

3.4 编程练习 2：实现 Scaled Dot-Product Attention

核心公式：

Attention(Q, K, V) = softmax(Q K^T / sqrt(d_k)) V

练习 2：ScaledDotProductAttention

  • Q/K/V shape：[batch, seq_len, d_k]。
  • 返回 output：[batch, seq_len, d_k]。
  • 返回 attention weights：[batch, seq_len, seq_len]。
  • 如果有 mask，要在 softmax 前把不可见位置设成很大的负数。
def scaled_dot_product_attention(Q, K, V, mask=None): d_k = Q.size(-1) scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k) if mask is not None: scores = scores.masked_fill(mask == 0, -1e9) attn_weights = torch.softmax(scores, dim=-1) output = torch.matmul(attn_weights, V) return output, attn_weights

3.4A 例子：手算带 causal mask 的 attention

假设当前 query 位于 t=2，可以看位置 0、1、2，不能看未来位置 3。未缩放点积是 [1, 1, 2, 2]，除以 sqrt(2) 得到约 [0.707, 0.707, 1.414, 1.414]。causal mask 必须在 softmax 前把未来位置设为 -inf：

masked logits = [0.707, 0.707, 1.414, -inf] attention weights ≈ [0.248, 0.248, 0.504, 0.000]

masked 位置不进入 softmax 分母。哪怕未来 token 的 value 很大，只要它的 key 被 mask 掉，权重就是 0，不会影响输出。

3.4B Self-Attention 的置换等变性：为什么还需要位置编码

没有位置编码、也没有 causal mask 时，self-attention 只看 token 内容之间的相似度。如果把输入序列重排，Q/K/V 也会同样重排，输出也只是同样重排。也就是说：

Attn(PX) = P Attn(X)

这对集合输入是自然的，但语言需要顺序。“I hit you”和“you hit me”包含相同 token，却不是同一件事。所以 Transformer 必须额外引入位置信息，例如 absolute position embedding、RoPE、ALiBi 或 causal mask。

3.5 为什么要除以 sqrt(d_k)：方差和梯度行为

假设 Q 和 K 的每个元素独立同分布，均值为 0、方差为 1。点积 q · k 是 d_k 个随机变量相加，所以方差约为 d_k，标准差约为 sqrt(d_k)。

如果不缩放，d_k 越大，attention logits 的尺度越大，softmax 越容易接近 one-hot。softmax 一旦饱和，梯度会很小，Q/K 路径就很难学习。

除以 sqrt(d_k)，是在初始化假设下把 logits 的方差拉回稳定范围。它不是玄学常数，而是方差传播推出来的尺度控制。

🔑
核心：sqrt(d_k) 不改变同一行 logits 的排序，但会改变 softmax 的尖锐程度和梯度规模。

3.6 Softmax temperature、梯度饱和和 attention 分布尖锐度

softmax 的梯度和概率本身有关：当某个概率接近 1 时，p(1-p) 接近 0；当分布过于尖锐时，很多位置的梯度都会变小。

temperature tau 会控制 softmax 的尖锐度：

  • tau < 1：分布更尖，模型更“确定”，但梯度更容易小。
  • tau > 1：分布更平，随机性更高，梯度也更分散。

attention 中除以 sqrt(d_k)，可以看成给 logits 加了一个固定 temperature。Ch08 生成阶段还会用 temperature 控制采样随机性，但那是推理时对输出 logits 的控制。

从 output 梯度回传到 Q/K/V

attention 输出是 O = P V，其中 P = softmax(S)，S = QK^T / sqrt(d_k)。反向传播时：

  • dV 来自 attention weights 对 value 的加权。
  • dQ 和 dK 来自 score matrix 对点积两边的梯度。
  • 1/sqrt(d_k) 也会出现在 Q/K 梯度里。

本章作业里的 softmax_jacobian、attention_logits_gradient 和 attention_qkv_gradients，就是让你把这条链路和 autograd 对齐。

3.6A Attention 诊断记录：logits、entropy、gradients 和 mask

Attention heatmap 很直观，但训练排错时更该先看数值诊断。建议记录：

指标含义异常通常说明什么
score_mean/std/max/minattention logits 尺度缺少 1/sqrt(d_k)、Q/K 初始化异常、LayerNorm 输出尺度异常
max_attention_prob每行最大注意力概率大面积接近 1 时，softmax 可能过早饱和
attention_entropyattention 分布熵接近 0 表示过尖；接近 log(visible_keys) 表示接近均匀
masked_position_prob_summask 位置拿到的概率和非 0 通常是 mask 方向、broadcast 或应用时机错误
all_masked_rows是否存在整行不可见softmax 可能 NaN，常见于 padding mask 和 causal mask 组合错误
grad_norm_q/k/vQ/K/V 路径梯度Q/K 梯度消失常见于 softmax 饱和或 mask 截断 score 路径

注意：低 entropy 不一定坏，复制、括号匹配、delimiter 定位等行为可能自然产生尖锐 attention。诊断要结合层、head、训练阶段和任务。

3.7 编程练习 3-5：实现和应用 Attention Mask

GPT 风格自回归模型中，位置 t 只能看自己和过去，不能看未来。causal mask 通常保留下三角（含对角线），屏蔽上三角未来位置。

练习 3：生成 causal mask

def create_causal_mask(seq_len): return torch.ones(seq_len, seq_len).tril().bool()

练习 4：组合 causal mask 和 padding mask

padding_mask shape 是 [B,T]，True 表示真实 token。把 causal mask 扩成 [1,T,T]，把 padding key mask 扩成 [B,1,T]，两者取 AND 得到 [B,T,T]。

def combine_causal_padding_mask(padding_mask): valid_keys = padding_mask.bool() T = valid_keys.size(1) causal = create_causal_mask(T).to(valid_keys.device) return causal.unsqueeze(0) & valid_keys.unsqueeze(1)

练习 5：把 causal mask 接入 attention

mask 必须在 softmax 前进入 logits。不要先 softmax 再把未来位置乘 0，否则可见位置的概率和会小于 1。

3.7A Attention 实现边界：从教学函数到真实模型

教学版 attention 几十行就能写完，真实模型的错误多半出在边界条件：mask 语义、dtype、broadcast、全 mask query、混合精度 sentinel、incremental decoding 和高性能 kernel API。

  • 先固定 mask 语义：本课程约定 True/1 表示可见，False/0 表示屏蔽。不要和 additive mask 混用。
  • mask 必须在 softmax 前进入 logits：这样 softmax 分母只包含可见 key。
  • 全 mask 行要单独处理：整行都是 -inf 时 softmax 会 NaN。padding query 的输出通常用 loss/output mask 跳过，而不是把它能看的 key 全屏蔽。
  • 低精度 sentinel 不是随便选：-1e9 在教学 float32 中直观，但 fp16/bf16 或 fused kernel 里要遵守库的推荐方式。
  • 训练和 incremental decoding 的 shape 不同：训练是 [B,T,T]，decode 时常是新 query 对历史 KV cache 的 [B,1,T_cache]。

能写教学版 attention 是第一步；能判断它和 FlashAttention、SDPA、KV cache、packed sequence 是否等价，才是工程能力。

3.7B Packed Sequences、文档边界和变长 attention

真实训练管线常把多个短样本 pack 到一条长 token 序列里，减少 padding 浪费。但这会改变 mask 语义：token 可以看同一个 packed segment 里更早的 token，不能看前一个样本或文档的 token。

如果只用普通 causal mask，模型会悄悄学到跨样本泄漏。高性能 kernel 通常通过 cu_seqlens、varlen attention 或显式 block mask 表达边界。工程上必须保证 padding removal、document boundary mask 和高性能 kernel 描述的是同一套可见关系。

3.8A Heatmap 解释边界：权重不是因果解释

Attention weight 可以告诉你“某个 head 在这次 forward 里把概率分给了哪里”，但不能单独证明“模型因为看了这个 token 才做出这个判断”。原因包括：

  • Value 内容、后续层、残差路径和 FFN 都会改变最终输出。
  • 多个 head 之间会互相补充或抵消。
  • 高 attention weight 不等于高因果贡献，低 weight 也不等于无贡献。

Heatmap 更适合做定位和排错，例如检查未来泄漏、padding 泄漏、delimiter 是否被 attend、长距离依赖是否有候选路径。要做因果解释，需要 ablation、activation patching 或对照实验。

3.9 Attention 复杂度的成本和出路：从 O(n²) 到 MLA

标准 attention 最大的问题是 score matrix 的大小是 [B,H,T,T]。序列长度翻倍，score 内存和计算大致变成四倍。

计算和显存瓶颈

长上下文训练时，显式保存 QK^T 和 softmax 中间结果会非常贵。推理时，prefill 仍要处理长输入；decode 阶段虽然每步只有一个新 query，但 KV cache 会随上下文长度增长。

FlashAttention

FlashAttention 不改变 attention 数学结果，改变的是计算方式：它用分块和 online softmax 减少 HBM 读写，不显式 materialize 完整 score matrix，从 IO 角度提升速度和显存效率。

MLA 和 KV 压缩

DeepSeek MLA 这类方法的重点不是直接把 QK^T 的复杂度变成线性，而是压缩 KV cache 的存储和带宽压力。Ch04 会进一步讲 MHA、GQA、MLA 的关系。

3.10 Attention 作为可微字典查找

可以把 attention 理解成一个软字典：

  • Key 是每条记录的索引标签。
  • Value 是记录内容。
  • Query 是查询条件。

传统字典查找通常返回一个确定条目；attention 先计算 query 和所有 key 的相似度，用 softmax 得到概率分布，再对 value 做加权和。这个过程可微，所以模型可以通过梯度学习“怎么问问题”“怎么贴标签”“取回哪些内容”。

和卷积相比，attention 的连接关系不是固定窗口，而是由内容动态决定；但代价是 O(T^2) 的 token-to-token 比较。
