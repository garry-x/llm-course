11.1 为什么 LLM 课程仍要学习 Classic NLP

CS224N 风格的神经 NLP 主线从词向量、RNN、dependency parsing、seq2seq attention、Transformer、BERT 一路发展到大语言模型。跳过中间模型会带来一个常见误解：以为所有 NLP 问题都应被改写成 prompt-based generation。实际上，许多生产系统把不同模型族组合起来：BERT 或 DeBERTa 做 reranking，bi-encoder 做 retrieval embedding，encoder-only head 做抽取式 QA，decoder-only LLM 做答案生成。

本章的重点不是记住旧模型名字，而是掌握三类稳定概念：

  • 归纳偏置：RNN 按时间递归，parser 输出树，seq2seq 分离源序列和目标序列，BERT 双向读上下文。
  • 目标函数：causal LM 预测下一个 token，MLM 只在 mask 位置计算 loss，seq2seq 最大化条件生成概率，parser 最大化结构动作或 head/label 概率。
  • 评测指标：BLEU、ROUGE、EM/F1、UAS/LAS 和 retrieval metrics 各自回答不同问题，不能用一个总分替代。

11.3 Dependency Parsing：结构化预测不是 attention heatmap

Dependency parsing 为每个 token 预测一个 head 和一个 dependency label，输出是一棵有向树。它和 attention heatmap 的区别很重要：attention 是连续权重，可用于诊断；dependency tree 是离散结构，要满足单根、无环、每个词一个 head 等约束。

Arc-standard transition system 用 stack、buffer 和 arcs 构造树：

动作前置条件效果
SHIFTbuffer 非空把 buffer 首 token 推入 stack
LEFT_ARC(label)stack 至少两个元素次栈顶依附于栈顶，然后移除次栈顶
RIGHT_ARC(label)stack 至少两个元素栈顶依附于次栈顶，然后移除栈顶
ROOT(label)只剩一个根候选把根候选连到 ROOT

Worked Example：Arc-standard 解析 I saw her

目标依存树是 saw -> I (nsubj)、saw -> her (obj)，最后 ROOT -> saw (root)。用 arc-standard transition system 可以按下表构造：

stepstackbufferactionnew arc
0[][I, saw, her]SHIFT-
1[I][saw, her]SHIFT-
2[I, saw][her]LEFT_ARC(nsubj)saw -> I
3[saw][her]SHIFT-
4[saw, her][]RIGHT_ARC(obj)saw -> her
5[saw][]ROOT(root)ROOT -> saw

这个例子说明 parsing 不是给每个 token 独立分类：动作是否合法取决于当前 stack/buffer 状态，错误 action 会破坏最终树结构。Ch11 作业中的 run_arc_standard_transitions 正是让学生执行这张 transition table，并返回每个 token 的 head、label、arcs 和 trace。

11.5 BERT / Encoder-only：MLM、分类与抽取式 QA

BERT-style encoder 可以双向看见输入中的左右上下文，不是自回归生成器。它的预训练目标之一是 Masked Language Modeling：只在被 mask 的位置计算 cross entropy，未 mask 的 token 使用 ignore index。课程作业中的 masked_lm_loss_from_logits 会显式返回 masked-token loss、accuracy 和参与 loss 的位置，帮助区分“输入中有哪些 token 可见”和“哪些位置贡献训练目标”。

维度Encoder-only / BERTDecoder-only / GPT
可见性双向 self-attentioncausal mask，只看左侧前缀
预训练目标MLM：预测 mask 位置next-token prediction
典型用途分类、reranking、embedding、token classification、span QA开放式生成、代码、对话、reasoning
输出约束常有固定 label 或输入 span生成新 token 序列

抽取式 QA 常把输入组织为 [CLS] question [SEP] passage [SEP]，模型分别预测每个位置作为答案起点和终点的 logits。合法 span 的分数为：

若 [CLS] 的 no-answer 分数最高，模型应返回空答案。这和 decoder-only LLM 生成“我不知道”不同：BERT QA 的答案必须绑定到输入 token span。

BIO sequence labeling

NER、slot filling 和 token classification 常把每个 token 标成 B-TYPE、I-TYPE 或 O。B-ORG I-ORG 表示一个连续组织实体，O 表示实体外部。课程作业中的 bio_tags_to_spans(tokens, tags) 把 token-level 标签解码成 span-level 输出，例如 ["B-LOC", "I-LOC"] 对应一个 LOC span。若一个序列以 I-LOC 开头，很多评测脚本会把它容错为新实体开始；严格系统也可以选择直接报错，但必须在数据协议中写清楚。

实体识别通常用 span-level F1，而不是只看 token accuracy。一个预测实体只有在 type、start 和 end 都和 gold 完全一致时才算 true positive；边界少一个 token 或类型错了都算 false positive，同时对应 gold span 仍是 false negative。Ch11 作业中的 span_f1(gold_spans, pred_spans) 把这个定义落到可计算的 TP/FP/FN、precision、recall 和 F1。

Viterbi structured decoding

逐 token argmax 会忽略标签之间的约束：例如 B-ORG 后面接 I-ORG 通常比跳回 O 更合理，O 后面直接接 I-PER 往往应被惩罚。Linear-chain CRF 或类似结构化解码把路径分数写成 emission score 加 transition score：

公式 11.7 Linear-chain path score

Viterbi 动态规划在第 t 步保存“以某个 tag 结尾的最佳路径分数”和 backpointer，复杂度为 O(TK^2)。Ch11 作业中的 viterbi_decode 让学生手写这个解码过程，理解结构化预测如何把局部分类器输出变成全局一致的标签序列。

训练 CRF 时还需要归一化所有可能路径，而不是只看最优路径。Forward algorithm 把 Viterbi 里的 max 换成 logsumexp，计算 log-partition：

公式 11.8 CRF log-partition

Ch11 作业中的 linear_chain_log_partition 用 log-space forward DP 计算这个归一化项，linear_chain_crf_nll 再显式计算 gold path score，并返回 log_partition - gold_score。它们和 viterbi_decode 的状态形状相同，但语义不同：Viterbi 用于预测最优路径，CRF NLL 用于训练时比较 gold path 与所有竞争路径。

11.5A Worked Example：MLM loss、masked-token accuracy 与抽取式 QA span

Encoder-only 模型的训练和预测常见错误，是把“输入中所有 token 都可见”和“所有位置都参与 loss”混为一谈。BERT 的 self-attention 可以双向看整句，但 MLM loss 只在被 mask 的位置计算。

MLM：只在 mask 位置求平均

masked-token accuracy 是 1/2，因为位置 2 预测正确，位置 3 预测成 dog。未 mask 的 [CLS]、the 和 [SEP] 虽然参与上下文编码，但不进入 loss 分母，也不计入 MLM accuracy。这个定义对应作业函数 masked_lm_loss_from_logits 返回的 num_masked、positions 和 accuracy。

抽取式 QA：从 start/end logits 选择合法 span

BERT-style QA head 不生成自由文本，而是在输入 passage 中选择一个连续 span。设输入是 [CLS] who founded OpenAI [SEP] Sam Altman founded OpenAI [SEP]，候选答案必须来自 passage token 位置。start/end logits 最高的组合可能如下：

tokenpositionstart logitend logit
[CLS]01.01.0
Sam54.00.2
Altman60.53.5
OpenAI80.21.8

合法 span 要满足 start <= end 且长度不超过 max_answer_len。若 max_answer_len >= 2，Sam Altman 的分数为 4.0 + 3.5 = 7.5，高于 no-answer 的 [CLS] 分数 1.0 + 1.0 = 2.0，所以返回该 span。若 [CLS] 分数最高，系统返回 no-answer；这与 decoder-only LLM 生成一段解释不同，是一个受输入 span 约束的预测问题。

🔑
统一视角：MLM、token classification、CRF、extractive QA 都把 encoder 输出接到不同 prediction head 上。它们的共同优势是输出空间受控；共同边界是不能自然生成输入中不存在的新文本。

11.6 评测：BLEU、ROUGE、EM/F1 与结构指标

评测指标必须匹配任务结构。翻译常用 BLEU，摘要常用 ROUGE，抽取式 QA 常用 Exact Match 和 token-level F1，dependency parsing 用 UAS/LAS。它们都不是“语言质量”的完整度量。

指标核心计算适用任务主要局限
BLEUclipped n-gram precision + brevity penalty机器翻译对同义改写和语义等价不敏感
ROUGE-L最长公共子序列的 precision/recall/F1摘要、生成比较偏向词面重合，不能保证事实一致
EM/F1答案规范化后 exact match 或 token overlap抽取式 QA、短答多答案、别名和推理型答案需要额外处理
UAS/LAShead 是否正确，label 是否也正确依存句法分析只评估句法结构，不等于下游任务成功

Worked Example：BLEU clipping 与 ROUGE-L

BLEU 不是简单数候选句里有多少词出现在参考句中。设 candidate 为 the the the the，reference 为 the cat is here。只看 unigram 时，candidate 中 the 出现 4 次，但 reference 中最多只出现 1 次，所以 clipped match 只能算 1 次：

BLEU-1 clipped precision

这一步防止模型靠重复高频词刷高 precision。若 candidate 是 cat sat，reference 是 cat sat on mat，unigram precision 为 1，但候选句长度 2 小于参考句长度 4，需要 brevity penalty：

Brevity penalty

因此短句不能因为每个词都命中就得到满分。ROUGE-L 关注最长公共子序列。若 candidate 为 a b c d，reference 为 a x b c，最长公共子序列是 a b c，长度为 3；precision = 3/4，recall = 3/4，F1 = 0.75。Ch11 作业中的 sentence_bleu 与 rouge_l_f1 正是把这些定义写成可运行函数。

因此评测报告不能只写“BLEU 高”或“ROUGE 高”。必须说明任务结构、候选输出、参考答案集合和指标局限，否则一个分数很容易被误读成完整的语言质量判断。

11.7 任务选型：什么时候不用 Decoder-only LLM

选择模型族时先看输出空间和错误成本：

  • 固定标签分类：encoder-only fine-tuning 或小模型分类器通常更便宜、更稳定。
  • RAG reranking：cross-encoder 可以同时看 query 和 document，给候选排序；decoder-only LLM rerank 成本高且格式不稳定。
  • 抽取式答案：若答案必须来自文档原文，span QA 比自由生成更可控。
  • 结构输出：dependency parser、NER、slot filling 等任务可用结构化 head 或 constrained decoding，不应默认开放式生成。
  • 开放式生成与推理：当输出需要新文本、代码、解释或多步 reasoning，decoder-only LLM 更自然。
⚠️
常见错误：把 BERT 和 GPT 说成“只是 mask 不同”。它们的 attention 可见性、训练目标、fine-tuning head、输出空间和服务成本都不同。

11.9 小结

  • RNN/LSTM 解释了顺序递归、BPTT 和长距离依赖问题。
  • Dependency parsing 是离散结构预测，不等于 attention heatmap。
  • Seq2Seq attention 把源序列和目标前缀分开建模，是 encoder-decoder 的基本形式。
  • BERT/encoder-only 适合分类、reranking、embedding、token classification 和抽取式 QA。
  • BLEU、ROUGE、EM/F1、span F1、UAS/LAS 各自衡量不同任务结构；现代 LLM 评测应说明指标边界。
