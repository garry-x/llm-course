5.1 Decoder-only 架构：Transformer Block 是主复用单元

很多主流自回归 LLM，例如 GPT、Llama、DeepSeek 系列，都使用 decoder-only 架构：没有单独的 encoder，主体是一层层 Transformer Block。每个 Block 有两个核心子层：

  • Self-Attention：让 token 之间交换信息。前两章已经实现。
  • Feed-Forward Network (FFN)：对每个 token 独立做非线性变换。

现代 LLM 常用 Pre-Norm 写法：

x' = Attention(Norm(x)) + x x'' = FFN(Norm(x')) + x'

Attention 像“取信息”：每个 token 从上下文里拿相关内容。FFN 像“处理信息”：拿到上下文后，在当前 token 位置做非线性计算。没有 FFN，模型主要只是加权求和，表达力会弱很多。

Block 的关键约束是：输入和输出都必须是 [B,T,D]。只有这样，Block 才能一层接一层堆起来。

5.1A 例子：Pre-Norm 和 Post-Norm 的梯度路径

用一个标量残差块近似看差异。设子层是 f，norm 是 n。Pre-Norm 写成 y = x + f(n(x))，Post-Norm 写成 y = n(x + f(x))。局部线性化后：

Pre-Norm: dy/dx = 1 + f'(n(x)) n'(x) Post-Norm: dy/dx = n'(x + f(x)) (1 + f'(x))

如果三层 toy block 的子层斜率是 [0.2, -0.1, 0.3]，norm 局部斜率近似 0.5：

Pre-Norm factors = [1.10, 0.95, 1.15], product = 1.20175 Post-Norm factors = [0.60, 0.45, 0.65], product = 0.1755

这只是局部模型，不是完整证明。但它说明了一个重要现象：Pre-Norm 的残差路径保留显式 identity term，深层网络里的梯度更容易传过去。

5.2 编程练习 1：手写 LayerNorm

LayerNorm 沿最后一维，也就是 feature dimension 做归一化。它不依赖 batch size，因此适合变长序列和自回归模型。

LayerNorm(x) = gamma * (x - mean) / sqrt(var + eps) + beta

练习 1：LayerNorm from Scratch

  • 不要使用 nn.LayerNorm。
  • 可学习参数：gamma 初始化为 1，beta 初始化为 0。
  • eps=1e-6。
  • 支持任意输入 shape，只沿最后一维归一化。
  • 计算方差时用 unbiased=False，和 PyTorch 官方实现对齐。
class LayerNorm(nn.Module): def __init__(self, dim, eps=1e-6): super().__init__() self.eps = eps self.gamma = nn.Parameter(torch.ones(dim)) self.beta = nn.Parameter(torch.zeros(dim)) def forward(self, x): mean = x.mean(dim=-1, keepdim=True) var = x.var(dim=-1, keepdim=True, unbiased=False) x_norm = (x - mean) / torch.sqrt(var + self.eps) return x_norm * self.gamma + self.beta

unbiased=False 很容易漏。LayerNorm 用总体方差，除以 N，不是统计课里常见的样本方差 N-1。

5.3 Pre-Norm vs Post-Norm：Norm 位置怎样影响训练稳定性

Norm 放在子层前还是子层后，会明显影响深层 Transformer 的训练。

Post-Norm

原始 Transformer 使用 Post-Norm：

x' = LayerNorm(Attention(x) + x)

Norm 在 residual addition 之后。这样残差路径本身也会经过 normalization 的 Jacobian，深层堆叠时更依赖初始化、warmup 和 residual scale。

Pre-Norm

现代 decoder-only LLM 更常用 Pre-Norm：

x' = Attention(LayerNorm(x)) + x

Norm 只作用在子层输入上，residual stream 保持一条更直接的通路。它通常让深层训练更稳定，但不是无限深网络的保证；学习率、初始化、optimizer、数据和 residual scaling 仍然会影响训练。

属性Post-NormPre-Norm
代表原始 TransformerGPT、Llama、DeepSeek 常见路线
Norm 位置残差相加之后子层输入之前
梯度路径shortcut 也受 norm 影响identity path 更直接
深层训练更敏感通常更稳

5.4 编程练习 2：实现 RMSNorm

RMSNorm 去掉了 LayerNorm 的 mean-centering，只保留 RMS scaling。很多 Llama/DeepSeek 风格模型都用 RMSNorm，因为它更简单，计算也更省。

RMSNorm(x) = gamma * x / sqrt(mean(x^2) + eps)

例子：RMSNorm 不会把均值变成 0

向量 x=[1,2,2,3] 的 RMS 是 sqrt((1^2+2^2+2^2+3^2)/4) = sqrt(18/4) ≈ 2.121。忽略 eps 和 gamma，输出约为 [0.471, 0.943, 0.943, 1.414]。它的 RMS 约为 1，但均值不是 0。

练习 2：RMSNorm Module

  • 只有一个可学习参数 gamma，没有 beta。
  • 计算 rms = sqrt(mean(x^2) + eps)。
  • 返回 x / rms * gamma。
  • 验证输出方向和输入方向一致，只改变长度。
class RMSNorm(nn.Module): def __init__(self, dim, eps=1e-6): super().__init__() self.eps = eps self.gamma = nn.Parameter(torch.ones(dim)) def forward(self, x): rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps) return x / rms * self.gamma

RMSNorm 的 backward 里容易漏掉耦合项：每个维度都会影响共享的 RMS 分母，所以某一维的梯度不只取决于自己的 upstream gradient。

5.5 FFN 作为 Key-Value Memory：模型知识存在哪里

FFN 不只是“非线性层”。从 mechanistic interpretability 的角度看，它可以像一组大规模 key-value memory。

FFN(x) = sum_i activation(W1_i · x + b_i) * W2_i
  • Key：W1_i · x + b_i 检测输入是否匹配某种模式。
  • Gate：activation 决定这个 key 是否打开。
  • Value：W2_i 是被写回 residual stream 的信息片段。
  • 容量：d_ff 越大，可以容纳的 key-value pair 越多。

这解释了为什么 FFN 宽度对模型能力很重要。Attention 更像检索协调器，决定上下文信息怎么流动；FFN 则承担大量 token-wise 计算和知识存储。

模型编辑的启发

ROME 等 model editing 方法正是利用了 FFN 的 key-value 视角：定位和某个事实相关的 FFN 表示或 value 方向，再修改权重以改变模型事实记忆。这里要有边界：局部编辑可能有副作用，不能只看目标事实是否改成功。

5.5A 机制可解释性：从“相关”到“改了会影响输出”

看到某个 head、neuron 或 residual direction 和行为相关，不等于它就是因果机制。课程里按证据强度区分几类方法：

方法做法能回答不能单独证明
Probing用小模型从 hidden state 预测标签信息是否可读模型生成时是否真的使用它
Logit attribution把组件输出投到 unembedding 方向哪些组件直接推高/压低目标 token后续层的改写和非线性相互作用
Activation patching把 clean run 的激活替换到 corrupted run某处激活是否足以恢复行为是否只有这一条路径负责
Ablation清零或替换某个组件输出拿掉组件后行为变多少模型是否被冗余路径补偿

比如主谓一致：The keys to the cabinet ... 应该更偏向 are 而不是 is。合理实验不是看一张 attention heatmap，而是构造 singular/plural 对照，定义 logit(are)-logit(is)，再做 patching 和 ablation。

5.5B 可解释性方法谱系：读表示、做归因、再干预行为

可解释性不是单个技巧，而是一组证据逐渐增强的方法。一个实用顺序是：先观察表示，再做局部归因，然后做因果干预，最后才谈 steering 或 model editing。

层级方法支持的结论主要风险
读表示Probing、logit lens、tuned lens某类信息是否能从表示里读出来可读不等于被模型使用
局部归因Logit attribution、path attribution哪个组件直接推高目标 logit忽略后续层重写
因果干预Activation patching、causal tracing、ablation某个激活是否能恢复或破坏行为干预可能让样本分布变怪
特征分解Sparse autoencoder、dictionary feature把混合方向拆成更可解释特征特征命名有主观性
行为干预Activation steering、model editing某个方向或权重改动能稳定改变输出副作用、泛化失败、回滚困难

如果要从解释走到干预，必须报告副作用：locality、generalization、capability retention 和 rollback。否则“改成功一个例子”不能说明系统可用。

5.6 编程练习 3：实现 FFN 和 SwiGLU

GPT-2 使用 GELU FFN。很多现代 decoder-only LLM 改用 SwiGLU 或类似 GLU 变体。SwiGLU 的核心是 gate branch 和 value branch 相乘，让模型按 token 条件选择哪些中间特征通过。

FFN_GELU(x) = W2 GELU(W1 x) SwiGLU(x) = W3 (SiLU(W1 x) * W2 x)

练习 3：FFN (GELU) and SwiGLU

  • 标准 FFN：d_model -> d_ff -> d_model，默认 d_ff=4*d_model。
  • SwiGLU：三组 Linear，w1 是 gate，w2 是 value，w3 是 output。
  • SwiGLU 默认 d_ff=int(8/3*d_model)，让参数量接近 4x FFN。
class FFN(nn.Module): def __init__(self, d_model, d_ff=None): super().__init__() d_ff = d_ff or 4 * d_model self.fc1 = nn.Linear(d_model, d_ff) self.fc2 = nn.Linear(d_ff, d_model) def forward(self, x): return self.fc2(F.gelu(self.fc1(x))) class SwiGLU(nn.Module): def __init__(self, d_model, d_ff=None): super().__init__() d_ff = d_ff or int(d_model * 8 / 3) self.w1 = nn.Linear(d_model, d_ff, bias=False) # gate self.w2 = nn.Linear(d_model, d_ff, bias=False) # value self.w3 = nn.Linear(d_ff, d_model, bias=False) # output def forward(self, x): return self.w3(F.silu(self.w1(x)) * self.w2(x))

5.6A 为什么现代 LLM 喜欢 SwiGLU

8/3 从哪里来

标准 4x GELU FFN 忽略 bias 时参数量是：

d_model * 4d_model + 4d_model * d_model = 8 d_model^2

SwiGLU 有三组矩阵，参数量约为 3 * d_model * d_ff。为了和 4x FFN 参数量接近，令 3 * d_model * d_ff ≈ 8 * d_model^2，得到 d_ff ≈ 8/3 * d_model。

Gate 改变了什么

GELU FFN 是单路径激活；SwiGLU 是“条件 A 控制值 B 是否写入”的结构。代码缩进、括号闭合、实体属性、格式约束这类行为，经常需要条件式组合，而不是单个特征独立起作用。

SwiGLU 和 MoE 的区别

机制选择发生在哪里每 token 激活主要收益主要风险
SwiGLU单个 FFN 的 hidden dimension 内所有矩阵都参与同等参数预算下增强条件交互仍是 dense compute
MoE多个 expert 之间只选 top-k experts总参数可大幅增加，激活参数可控路由、负载均衡、通信更复杂

5.6C 单个 Transformer Block 的参数、FLOPs 和 activation memory

会写 TransformerBlock.forward 只是第一步。训练工程还要知道贵在哪里：参数主要来自投影矩阵，FLOPs 主要来自矩阵乘法和 attention 的二次项，activation memory 主要来自 Q/K/V、attention scores 和 FFN 中间激活。

参数量近似

  • MHA 的 Q/K/V/O projection：约 4 * d_model^2。
  • SwiGLU：约 3 * d_model * d_ff。
  • 两个 RMSNorm：只有 2 * d_model 个 scale 参数。

例子：教学 Block 预算

设 B=2、T=128、d_model=768、n_heads=12、d_ff=2048，fp16/bf16 activation 2 bytes。

组件公式参数量
Q/K/V/O projection4 * d_model^22,359,296
SwiGLU FFN3 * d_model * d_ff4,718,592
两个 RMSNorm2 * d_model1,536
总计7,079,424

短序列 T=128 时，FFN 往往已经是主要计算项。随着 T 增长，attention score 的 B * H * T^2 会迅速变大。FlashAttention 能避免显式 materialize 完整 attention matrix，但教学实现通常会存。

Activation checkpointing 不改变模型函数。它少存一部分 forward 中间结果，在 backward 时重算，用更多计算换更低训练显存。

5.7 编程练习 4：组装完整 TransformerBlock

现在把 RMSNorm、MHA/GQA、SwiGLU 和 residual connection 组合成完整 decoder block。

练习 4：TransformerBlock Assembly

  • 子层 1：RMSNorm -> MultiHeadAttention -> Dropout -> Residual。
  • 子层 2：RMSNorm -> SwiGLU -> Dropout -> Residual。
  • 使用 Pre-Norm。
  • 支持传入 attention mask。
  • 输出 shape 必须等于输入 shape。
class TransformerBlock(nn.Module): def __init__(self, d_model, n_heads, d_ff=None, dropout=0.0): super().__init__() self.norm1 = RMSNorm(d_model) self.attention = MultiHeadAttention(d_model, n_heads, dropout=dropout) self.norm2 = RMSNorm(d_model) self.ffn = SwiGLU(d_model, d_ff) self.dropout = nn.Dropout(dropout) def forward(self, x, mask=None): x = x + self.dropout(self.attention(self.norm1(x), mask=mask)[0]) x = x + self.dropout(self.ffn(self.norm2(x))) return x

核心设计是把 normalization path 和 residual path 分开。信息沿 residual path 直接传下去，Norm 只放在每个子层输入前。

5.7A Block 实现稳定性清单

数学形式很短，但训练是否稳定取决于一批实现细节。

1. 控制 residual branch 初始扰动

Pre-Norm 保留直接梯度路径，但如果每个子层一开始写入 residual stream 的更新太大，深层训练仍然会不稳。常见做法包括更小初始化、按深度缩放 residual projection，或显式 residual gate：

x_{l+1} = x_l + alpha_l * f_l(Norm(x_l))

2. Dropout 位置要说清楚

常见教学写法是：

x = x + dropout(attention(norm1(x), mask)) x = x + dropout(ffn(norm2(x)))

这表示训练时随机丢掉子层写入 residual stream 的一部分更新。推理时 dropout 必须关闭。把 dropout 放在 residual 之后，会随机破坏 identity path，含义完全不同。

3. Norm 和 softmax 常需要更高精度

bf16/fp16 训练中，matmul 可以走低精度 tensor core，但 norm 统计量、softmax 分母和某些累加最好用 fp32，再 cast 回模型 dtype。否则 RMSNorm 统计误差、attention overflow/underflow、mask sentinel 或 all-masked row 都可能制造 NaN。

4. Bias 策略要和架构一致

现代 decoder-only LLM 常在 QKV、output projection 和 SwiGLU Linear 里省略 bias，并用没有 beta 的 RMSNorm。如果复现 checkpoint，bias 策略必须和原模型一致，否则参数量、state_dict key 和输出分布都会对不上。

5.8 Gradient Check：为什么要验证反向传播

如果你手写 backward 或写自定义 kernel，一个符号错就可能让模型训练很久也不收敛。torch.autograd.gradcheck 会用 central difference 估计数值梯度，再和解析梯度比较。

df/dx_i ≈ (f(x + eps * e_i) - f(x - eps * e_i)) / (2 * eps)

central difference 的误差是 O(eps^2)，比 forward difference 更准，但每个参数维度要多做两次 forward。大模型上不能直接全量做，应该用很小输入、float64、合适的 eps 和 atol。

5.9 编程练习 5：用 torch.autograd.gradcheck 验证 LayerNorm

练习 5：LayerNorm Gradient Check

  • 定义 LayerNormFunction，继承 torch.autograd.Function。
  • 手写 forward 和 backward。
  • 用 gradcheck 对比 numerical gradient 和 analytical gradient。
  • 先用小输入：batch=2, seq=8, dim=64。
  • 使用 float64。
from torch.autograd import gradcheck x = torch.randn(2, 8, 64, dtype=torch.float64, requires_grad=True) gamma = torch.ones(64, dtype=torch.float64, requires_grad=True) beta = torch.zeros(64, dtype=torch.float64, requires_grad=True) ok = gradcheck( LayerNormFunction.apply, (x, gamma, beta, 1e-6), eps=1e-4, atol=1e-3, ) print(ok)

实际排错时按模块单独验证：LayerNorm、SwiGLU、整个 Block 分开做。全堆在一起测，一旦失败很难知道错在哪里。

5.10 Learnable Hyper-Connections：Birkhoff 约束和 Sinkhorn-Knopp

Hyper-Connections 想扩展传统 residual connection：不再让每层只和相邻层做固定 1:1 相加，而是学习更灵活的跨层连接权重。问题是，连接矩阵一旦可学习，就可能放大信号和梯度，所以需要约束。

Birkhoff Polytope

一种思路是把连接矩阵约束到近似 doubly stochastic matrix：元素非负，每行和每列都为 1。

B_n = { A in R^{n*n} | A >= 0, A 1 = 1, A^T 1 = 1 }

这样的矩阵能让每层“给出多少信息”和“接收多少信息”更受控，从而减少连接权重放大梯度的风险。实际稳定性还要看 residual scaling、optimizer、初始化和数据。

Sinkhorn-Knopp 迭代

Sinkhorn-Knopp 通过交替做行归一化和列归一化，把一个非负矩阵推向近似 doubly stochastic 结构：

def sinkhorn_knopp(M, num_iter=10): M = torch.exp(M - M.max(dim=-1, keepdim=True).values) for _ in range(num_iter): M = M / (M.sum(dim=-1, keepdim=True) + 1e-6) M = M / (M.sum(dim=-2, keepdim=True) + 1e-6) return M

注意：10 次迭代不保证任意输入都严格收敛。工程实现里要关注矩阵条件数、temperature/scaling、迭代次数和数值稳定性。

5.11 本章小结

  • Transformer Block 由 Attention、FFN、Norm 和 Residual 组成，输入输出都保持 [B,T,D]。
  • LayerNorm 沿 feature dimension 做均值和方差归一化；RMSNorm 只做 RMS scaling，没有 mean-centering，也没有 beta。
  • Pre-Norm 把 Norm 放在子层输入前，residual path 更直接，通常更利于深层训练。
  • FFN 可以理解成 token-wise key-value memory，模型很多知识和非线性计算都在这里发生。
  • 可解释性方法要区分“信息可读”“组件有归因”和“干预会改变行为”。
  • SwiGLU 用 gate/value 分支做条件式信息通过，8/3 hidden width 来自和 4x FFN 对齐参数预算。
  • 一个可靠的 Block 实现不只要 shape 对，还要能解释 residual update、norm 精度、dropout 位置、bias 策略和 checkpointing。
  • gradcheck 用数值梯度验证手写 backward，适合小输入、float64、模块级排错。
  • mHC/hyper-connections 通过受约束的连接矩阵学习跨层混合，但稳定性仍需要实证验证。
