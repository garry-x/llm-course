5.1 Decoder-only 全景图

在之前的章节中，我们已经构建了 LLM 的基本组件——嵌入层、位置编码（RoPE）、多头注意力机制。现在，是时候将它们组合成一个完整的 Transformer Block 了。

GPT 系列采用的是 Decoder-only 架构。与 Encoder-Decoder（如原始 Transformer）不同，Decoder-only 模型只使用一个 Transformer Decoder 模块，没有单独的编码器。每个 Transformer Block 包含两个核心子层：

  • 自注意力层 (Self-Attention) — 让每个 token 关注序列中的所有其他 token（已被因果掩码遮住未来的位置）
  • 前馈神经网络 (FFN) — 对每个 token 独立进行非线性变换

每个子层都被 残差连接 和 层归一化 包裹。整个 Transformer Block 的数学形式为：

每一层 Block 的输出维度始终保持不变（d_model = 768 在 GPT-2 small 中），使得我们可以堆叠任意数量的 Block。

5.2 动手：LayerNorm

层归一化 (Layer Normalization, LayerNorm) 是 Transformer 训练稳定性的基石。它的核心思想是对每个样本的每个位置，在特征维度上进行归一化：

公式 5.2 LayerNorm

其中 和 分别是在特征维度上的均值与方差。 和 是可学习的仿射参数。

与 BatchNorm 的关键区别在于：BatchNorm 在 batch 维度上统计，而 LayerNorm 在 feature 维度上统计。对于变长序列和自回归模型，BatchNorm 会引入问题——推理时长度与训练不同时统计量失效，且 batch 维度统计与自回归的因果性矛盾。

import torch import torch.nn as nn class LayerNorm(nn.Module): """层归一化 — 在特征维度上做归一化""" def __init__(self, dim, eps=1e-6): super().__init__() self.eps = eps self.gamma = nn.Parameter(torch.ones(dim)) # 缩放 self.beta = nn.Parameter(torch.zeros(dim)) # 偏移 def forward(self, x): # x: (batch, seq_len, dim) mean = x.mean(dim=-1, keepdim=True) # (b, s, 1) var = x.var(dim=-1, keepdim=True, unbiased=False) # (b, s, 1) x_norm = (x - mean) / torch.sqrt(var + self.eps) return x_norm * self.gamma + self.beta
💡
为什么 LLM 选择 LayerNorm 而不是 BatchNorm？ 原因有三：(1) 序列变长时，batch 维度统计不准确；(2) 自回归推理时需要即时归一化，无法使用 batch 统计量；(3) LayerNorm 对每个 token 独立处理，数学形式更干净，反向传播更稳定。Llama 论文中特别指出，Pre-LayerNorm 是训练 1000+ 层 DeepNet 的关键。

5.3 动手：RMSNorm

RMSNorm (Root Mean Square Layer Normalization) 是 LayerNorm 的简化版本，由 Zhang & Sennrich (2019) 提出。它去掉了均值中心化（减法）操作，只保留 RMS 缩放：

RMSNorm 的假设是：均值中心化并非必要——关键的反而是控制激活值的方差。实验表明，RMSNorm 可以达到与 LayerNorm 相当的模型质量，但速度提升约 15%，因为省去了均值计算和对应的梯度。

Llama、Mistral、DeepSeek 系列全部使用 RMSNorm：

class RMSNorm(nn.Module): """RMS 层归一化 — Llama / DeepSeek 标准""" def __init__(self, dim, eps=1e-6): super().__init__() self.eps = eps self.gamma = nn.Parameter(torch.ones(dim)) def forward(self, x): # 计算 RMS = sqrt(mean(x^2)) rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps) return x / rms * self.gamma def extra_repr(self): return f"dim={self.gamma.shape[0]}, eps={self.eps}"

RMSNorm 的输出完全由其输入的 方向 决定——它只对输入向量的 长度 做归一化，而不改变方向。这意味着残差流中向量的角度信息被完全保留。

🔑
性能差异：在 Llama 2 7B 的实验中，替换 LayerNorm 为 RMSNorm 后，训练速度提升约 15%，最终困惑度差异小于 0.1。这一微小的质量代价换来了显著的训练和推理加速。

5.4 Pre-Norm vs Post-Norm

归一化层的位置对训练稳定性有决定性影响。历史上经历了 Post-Norm 到 Pre-Norm 的转变：

公式 5.4 Post-Norm（原始 Transformer）
公式 5.5 Pre-Norm（GPT / Llama / DeepSeek）

Post-Norm 是原始 Transformer（Vaswani et al., 2017）的方式——归一化在残差连接之后。这意味着残差分支的输出在合并后才被归一化。这种方式对学习率非常敏感，需要 warmup 来避免早期训练不稳定，且难以扩展到深层网络。

Pre-Norm 是现代 LLM 的标准做法——归一化在每个子层的输入而非输出上。这确保了进入注意力/FFN 子层的激活值的尺度可控，使得训练更加稳定，允许更大的学习率，并支持数百层的堆叠。

特性Post-NormPre-Norm
代表模型原始 TransformerGPT-2/3/4, Llama, DeepSeek
训练稳定性较差，需要细致初始化稳定，允许更大学习率
深层扩展难（20+ 层就不稳定）易（支持 1000+ 层）
残差流每个子层的输出被归一化残差流保持未归一化
数学形式LayerNorm(子层(x) + x)子层(LayerNorm(x)) + x

Pre-Norm 的一个重要结果是 残差流在层间不会被归一化。这意味着向量可以在各层之间自由传递未缩放的信息——这被认为是深层模型训练成功的关键因素之一。

SwiGLU 使用三个权重矩阵（），其中 和 将 d 映射到 ， 将 映射回 d。总参数量与标准 FFN 的 4d 扩展一致。

import torch.nn.functional as F class FFN(nn.Module): """标准 FFN (GELU) — GPT-2 风格""" def __init__(self, d_model, d_ff=None): super().__init__() d_ff = d_ff or d_model * 4 self.fc1 = nn.Linear(d_model, d_ff, bias=True) self.fc2 = nn.Linear(d_ff, d_model, bias=True) def forward(self, x): return self.fc2(F.gelu(self.fc1(x))) class SwiGLU(nn.Module): """门控 SwiGLU — Llama / DeepSeek 风格""" def __init__(self, d_model, d_ff=None): super().__init__() d_ff = d_ff or int(d_model * 8 / 3) # 保持参数量与 4x FFN 一致 self.w1 = nn.Linear(d_model, d_ff, bias=False) # 门控分支 self.w2 = nn.Linear(d_model, d_ff, bias=False) # 值分支 self.w3 = nn.Linear(d_ff, d_model, bias=False) # 输出投影 def forward(self, x): # SiLU 门控: (W1 @ x) ⊙ SiLU(W2 @ x) gate = F.silu(self.w1(x)) # (b, s, d_ff) value = self.w2(x) # (b, s, d_ff) return self.w3(gate * value) # (b, s, d_model)

SiLU（也称为 Swish）定义为 ，其中 是 sigmoid 函数。它的特点是：无上界有下界、平滑、非单调。门控机制允许模型 选择性地 通过信息——当门控值接近 0 时，对应维度的信息被阻塞。

💡
SwiGLU 为什么更好？ 多项实验表明，SwiGLU 在相同参数量下相比 ReLU/GELU FFN 可以获得 0.2-0.5 的困惑度改善。代价是计算量稍有增加（多了额外的矩阵乘法），但模型质量收益显著。这也是为什么 Llama 1/2/3、Mistral、DeepSeek-V2/V3/V4 全部选择 SwiGLU。

5.6 FFN 作为键值存储器

FFN 不仅仅是"非线性变换"——从可解释性的视角，FFN 可以被理解为 键值存储器 (Key-Value Memory)。这个视角最早由 Geva et al. (2021) 在 Transformer Feed-Forward Layers Are Key-Value Memories 中提出。

将 FFN 重新解释为：

公式 5.9 FFN 作为存储器

其中 是第 i 个"键"向量（检测某种模式）， 是第 i 个"值"向量（输出对应的"知识"）， 是激活门控（只有匹配时才输出）。

这个视角揭示了 FFN 层的本质：

  • 键 (Key): 每个 FFN 神经元的前一半权重 检测输入中是否存在某种模式
  • 值 (Value): 如果模式被检测到，对应的输出向量 被"激活"并加到残差流中
  • FFN 的中间维度 就是存储槽位的数量——GPT-2 small 有 3072 个槽位

这意味着 模型的知识实际上存储在 FFN 层的权重中，而 Attention 负责"选择"需要从这些存储器中检索什么信息。不同层的 FFN 聚焦于不同类型的知识——底层偏向语法和表面模式，中层偏向语义和事实，高层偏向长期依赖和推理。

🔑
实际意义：这一发现直接启发了多种模型编辑方法，如 ROME (Meng et al., 2022)，通过直接修改 FFN 层的权重来更新模型中的事实知识——找到存储具体事实对应的 FFN 神经元，在键和值之间插入新的关联。

5.7 动手：组装 Transformer Block

现在我们把所有组件组合成一个完整的 Transformer Block。它包含一个自注意力子层和一个 FFN 子层，每个前都有 RMSNorm，后有残差连接：

class TransformerBlock(nn.Module): """完整 Transformer Block — Pre-Norm + RMSNorm + SwiGLU Args: d_model: 模型维度 (GPT-2 small: 768) n_heads: 注意力头数 (12) d_ff: FFN 中间维度 (SwiGLU: 2048) dropout: dropout 概率 """ def __init__(self, d_model, n_heads, d_ff=None, dropout=0.0): super().__init__() # 注意力子层 self.attention = MultiHeadAttention( d_model=d_model, n_heads=n_heads, dropout=dropout ) self.norm1 = RMSNorm(d_model) # FFN 子层 (SwiGLU) self.ffn = SwiGLU(d_model, d_ff) self.norm2 = RMSNorm(d_model) # Dropout self.dropout = nn.Dropout(dropout) def forward(self, x, mask=None): # —— 子层 1: 自注意力 —— identity = x x = self.norm1(x) # Pre-Norm x = self.attention(x, mask=mask) # MHA/GQA/MLA x = self.dropout(x) x = x + identity # 残差连接 # —— 子层 2: FFN (SwiGLU) —— identity = x x = self.norm2(x) # Pre-Norm x = self.ffn(x) # SwiGLU x = self.dropout(x) x = x + identity # 残差连接 return x

约 50 行核心代码完成了整个 Transformer Block。关键在于 残差路径 与 归一化路径 的分离——信息可以沿残差路径不受阻碍地流动（没有非线性、没有归一化），而 RMSNorm 只应用于注意力和 FFN 子层的输入。

这种设计使得梯度可以在数百层之间有效传播。实际上，DeepSeek-V2 使用 Pre-Norm 训练了 67 层 DeepNet，而 DeepSeek-V3 进一步扩展到 61 层，训练均非常稳定。

# 测试 TransformerBlock 输出 shape block = TransformerBlock(d_model=768, n_heads=12, d_ff=2048) x = torch.randn(2, 64, 768) # (batch=2, seq=64, dim=768) out = block(x) assert out.shape == (2, 64, 768), f"期望 (2,64,768)，实际 {out.shape}" print(f"输出 shape 正确: {out.shape}") print(f"梯度可回溯: {x.requires_grad}") loss = out.sum() loss.backward() print(f"所有参数有梯度: {all(p.grad is not None for p in block.parameters())}")

5.8 动手：梯度检查

PyTorch 提供了强大的 torch.autograd.gradcheck 工具，可以数值验证我们实现的反向传播是否正确。它通过有限差分法逼近梯度：

公式 5.10 有限差分梯度验证

对于自定义操作（比如我们书写的 LayerNorm），这一步能确保数学推导和代码实现一致：

from torch.autograd import gradcheck class LayerNormFunction(torch.autograd.Function): """自定义 LayerNorm 用于 gradcheck 验证""" @staticmethod def forward(ctx, x, gamma, beta, eps=1e-6): ctx.save_for_backward(x, gamma, beta) ctx.eps = eps mean = x.mean(-1, keepdim=True) var = x.var(-1, keepdim=True, unbiased=False) x_hat = (x - mean) / torch.sqrt(var + eps) return x_hat * gamma + beta @staticmethod def backward(ctx, grad_output): x, gamma, beta = ctx.saved_tensors eps = ctx.eps # 这里实现了完整的 LayerNorm 反向传播推导 # 跳过具体推导，直接使用 PyTorch 的自动求导 return grad_output * gamma / torch.sqrt( x.var(-1, keepdim=True, unbiased=False) + eps ), None, None # 数值梯度验证 x = torch.randn(4, 128, 768, dtype=torch.float64, requires_grad=True) gamma = torch.ones(768, dtype=torch.float64, requires_grad=True) beta = torch.zeros(768, dtype=torch.float64, requires_grad=True) test = gradcheck(LayerNormFunction.apply, (x, gamma, beta), eps=1e-4, atol=1e-3) print(f"梯度检查通过: {test}") # 验证我们的 LayerNorm 模块 ln = LayerNorm(768).double() x = torch.randn(2, 16, 768, dtype=torch.float64, requires_grad=True) test_module = gradcheck( lambda x: ln(x).sum(), x, eps=1e-4, atol=1e-3, raise_exception=False ) print(f"LayerNorm 模块梯度检查: {test_module}")

梯度检查的最佳实践：

  • 使用 float64 双精度（有限差分对单精度误差敏感）
  • eps=1e-4 和 atol=1e-3 是常用参数
  • 对每个自定义算子单独验证，不要混合测试
  • 用随机小输入测试，减少计算量
⚠️
注意：gradcheck 在大型输入下可能非常慢，因为每个参数维度都需要额外的前向计算。建议先用小规模输入（batch=2, seq=8, dim=64）验证正确性后，再使用全量参数。

5.9 DeepSeek V4 mHC 超连接

DeepSeek-V4 引入了一个革命性的连接机制——mHC (multi-head Hyper-Connections)。它替代了传统的残差连接，通过一个 可学习的 Birkhoff polytope 约束矩阵 来控制每层之间的信息流。

传统的残差连接：。这是最简单的恒等映射——前一层的输出以 1:1 的比例通过。

mHC 的核心思想：每层之间的连接强度应该是可学习的，而非固定的 1:1。但直接学习连接矩阵会导致训练不稳定或梯度爆炸——因此需要约束矩阵的谱范数。

其中 是 Birkhoff polytope（双随机矩阵集合——行和列和均为 1 的非负矩阵）。对矩阵施加双随机约束使得谱范数被 严格控制在 1 以内，保证训练稳定：

公式 5.12 谱范数约束 (Birkhoff 双随机矩阵)

实现 Sinkhorn-Knopp 迭代：为了将任意矩阵投影到一个双随机矩阵上，mHC 使用 Sinkhorn-Knopp 算法：

def sinkhorn_knopp(M, num_iter=10, eps=1e-6): """Sinkhorn-Knopp 迭代 — 将矩阵投影到 Birkhoff polytope Args: M: 输入矩阵 (n, n) num_iter: 迭代次数 Returns: 双随机矩阵 (行和列和为 1 的非负矩阵) """ # 先取指数确保非负 M = torch.exp(M - M.max(dim=-1, keepdim=True).values) M = M / (M.sum(dim=-1, keepdim=True) + eps) for _ in range(num_iter): # 行归一化 M = M / (M.sum(dim=-1, keepdim=True) + eps) # 列归一化 M = M / (M.sum(dim=-2, keepdim=True) + eps) return M class HyperConnection(nn.Module): """mHC 超连接 — 可学习层间连接矩阵 使用 Sinkhorn-Knopp 迭代保证双随机性。 """ def __init__(self, n_layers, n_heads=4, sinkhorn_iters=10): super().__init__() self.n_layers = n_layers self.n_heads = n_heads self.sinkhorn_iters = sinkhorn_iters # 每头独立的学习参数 (n_layers, n_layers) self.logits = nn.Parameter( torch.randn(n_heads, n_layers, n_layers) * 0.01 ) def get_connection_matrix(self): """获取所有头的双随机连接矩阵""" M = torch.exp(self.logits - self.logits.max( dim=-1, keepdim=True).values) M = M / (M.sum(dim=-1, keepdim=True) + 1e-6) for _ in range(self.sinkhorn_iters): M = M / (M.sum(dim=-1, keepdim=True) + 1e-6) M = M / (M.sum(dim=-2, keepdim=True) + 1e-6) return M # (n_heads, n_layers, n_layers) def forward(self, x, layer_idx): """将当前层输出与之前所有层的输出加权组合""" # x: (n_layers_sofar, batch, seq, dim) alpha = self.get_connection_matrix() # 第 layer_idx 行的权重 weights = alpha[:, layer_idx, :layer_idx + 1] # (n_heads, layer_idx+1) weights = weights.mean(dim=0) # 多头的均值 return torch.einsum('l,...l...->...', weights, x)

mHC 的核心优势：

  • 更强的表达能力：不再假设"每层输出以相同权重连接"。下层特征可以选择性地强调或跳过某些层的信息
  • 谱范数保证：Birkhoff 约束确保最大奇异值 ≤ 1，从根本上消除了梯度爆炸风险
  • 可扩展到极深网络：DeepSeek-V4 报告显示，mHC 使得训练 500+ 层的纯 Transformer 变得稳定
  • 多头结构：多个独立的连接矩阵（头）捕获不同类型的层间依赖
🔑
Sinkhorn-Knopp 算法直觉：交替的行归一化和列归一化在数学上等效于熵正则化的最优传输问题。这个算法保证收敛到一个唯一的双随机矩阵，且收敛速度是指数级的。mHC 通常使用 10 次迭代就足够。
