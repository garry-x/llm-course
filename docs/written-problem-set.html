Written Derivation and Conceptual Question Bank

This question bank corresponds to the 20% "Written Derivation and Conceptual Questions" in the README. It supplements mathematics, complexity, experimental explanations, and engineering judgment that cannot be covered by automated programming tests. It is recommended to select 2-3 questions per chapter as formal assignments, with the remaining questions used for discussion sessions or final review; step-by-step recalculable small examples for each chapter can be found in the Worked Example Pack.

Grading Rubric

DimensionScoreCriteria
Mathematical Correctness35Correct derivation of symbols, conditional probabilities, matrix dimensions, gradients, or complexity
Shape and Boundaries20Clearly specify shapes for batch, sequence, head, vocab, mask, etc., and cover boundary conditions
Engineering Explanation20Ability to connect formulas with PyTorch implementation, memory, latency, and stability
Experimental Judgment15Ability to explain test outputs, failure cases, ablation, or metric limitations
Expression and Citation10Clear answer structure, cite papers/official documentation/model cards when necessary

Ch01 Tokenization / BPE

  1. Given a corpus token sequence of length N, with vocabulary size increasing from V0 to V. Explain how each BPE merge changes the sequence length; manually compute the first two merges for the string abababa, including pair, count, new id, before/after length, tokens saved, final ids, and compression ratio, and provide a counterexample idea showing that greedy merging cannot guarantee global optimal compression.
  2. Compare the trade-offs of word-level, character-level, and byte-level BPE in terms of OOV, multilingualism, sequence length, and embedding parameter count.
  3. Design a tokenizer evaluation experiment: group by English, Chinese, code, mathematical formulas, and emoji/mixed text, compare the average token count, P95 token count, tokens/character, round-trip success rate, and embedding parameter count for each group; calculate the disparity between the highest and lowest tokens/character, and explain how these results affect context, training FLOPs, KV Cache cost, and multilingual user experience.
  4. Given a small corpus, manually compute the first 3 BPE merges, and explain how tie-breaking would affect the final vocabulary.

Ch02 Embedding / Position Encoding / RoPE

  1. Derive embedding lookup from one-hot matrix multiplication, explaining why E[token_id] is equivalent to one_hot @ E; then, using a token id tensor of shape [B,T], specify the shapes of one_hot(token_ids), E, and the output tensor respectively.
  2. Given the token sequence [0,1,2,1] and window size 1, manually compute the directed co-occurrence matrix; then, given center/positive/negative vectors, write the SGNS loss and derive the center vector gradient dL/dv_c under mean reduction; compute PMI and shifted PMI from the co-occurrence matrix; given a non-zero co-occurrence count, word/context vectors, and biases, compute one term of the GloVe weighted least-squares residual.
  3. Given 5 two-dimensional word vectors, compute the cosine similarity matrix, and use 3CosAdd to solve man:king :: woman:?; explain why the input words must be excluded.
  4. Compare skip-gram negative sampling with GloVe: what statistical signals do they learn from local context samples and the global co-occurrence matrix respectively? What does the relationship between SGNS and shifted PMI reveal? Why is analogical reasoning not a property directly guaranteed by the training objective?
  5. Prove that the relative displacement of sinusoidal position encoding can be represented by a linear transformation, i.e., PE(pos+k) can be obtained by rotating the sin/cos block of PE(pos).
  6. Prove that the RoPE dot product depends only on relative position: (R_m q)^T (R_n k) = q^T R_{n-m} k, and explain why this is helpful for long-context extrapolation but not a sufficient condition.

Ch03 Scaled Dot-Product Attention

  1. If q_i, k_i are independent with mean 0 and variance 1, derive the variance of q^T k, and explain why attention scores should be divided by sqrt(d_k).
  2. Prove or numerically verify that position-encoding-free, unmasked self-attention satisfies Attn(PX)=P Attn(X); explain why causal mask or position encoding breaks full permutation equivariance.
  3. Write the causal mask matrix for T=4; given a padding mask [[1,1,0,0],[1,1,1,0]], write the combined [B,T,T] attention mask, and explain whether the mask values should be applied before or after softmax.
  4. Derive the softmax Jacobian J_ij = p_i(delta_ij - p_j), and explain how the attention output gradient is passed back to logits via p @ V; further, for S = QK^T/sqrt(d_k), derive dQ = dS K / sqrt(d_k), dK = dS^T Q / sqrt(d_k), and dV = P^T dO, specifying the shape of each tensor.
  5. Given attention weights [1,0,0] and [1/3,1/3,1/3], compute the attention entropy, and explain how it reflects the sharpness of the softmax distribution.
  6. Analyze the time complexity and memory complexity of self-attention; given B, H, T, dtype_bytes, compute the memory of the dense attention score matrix, and distinguish between saving activations during training and KV Cache during inference.
  7. Given an attention heatmap, explain what routing diagnoses it can support and what causal explanations it cannot support alone; design an ablation or counterfactual prompt to compare model behavior changes.

Ch04 MHA / GQA / MLA

  1. Prove that standard MHA has the same number of parameters in Q/K/V/O projections as a single-head attention with the same width, but with different representations.
  2. Given n_heads=8, n_kv_heads=2, write the mapping of each Q head to its corresponding KV head; if the K/V tensor shape is [B,2,T,D], explain how to obtain [B,8,T,D] after repetition, and state what special cases n_kv_heads=1 and n_kv_heads=n_heads correspond to.
  3. Given n_layers=24, batch=4, seq_len=2048, n_heads=32, n_kv_heads=8, head_dim=128, d_latent=512, and fp16, calculate the number of KV Cache elements per token, total memory, and compression ratio relative to MHA for MHA, GQA, and MLA.
  4. Use low-rank approximation to explain why MLA can reduce cache; given q_h, latent c_s, and K decompression matrix W_UK,h, derive q_h^T (W_UK,h c_s) = (q_h W_UK,h)^T c_s, explaining how to compute attention scores in latent space without explicitly decompressing K; also explain that "caching low-dimensional latent" is not equivalent to "completely free decompression".
  5. Explain why head redundancy occurs; compare what head pruning, GQA, and MLA compress respectively, and what representational capabilities they may lose.

Ch05 Transformer Block / Norm / FFN

  1. Derive the mu, var, x_hat of LayerNorm, and explain why mu and var cannot be treated as constants during backpropagation; then derive dL/dx for RMSNorm(x)=gamma*x/sqrt(mean(x^2)+eps), and point out why a coupling term across feature dimensions appears in the gradient.
  2. Compare the gradient paths of Pre-Norm and Post-Norm; given the sub-layer local slope and norm local slope for several layers, calculate the per-layer gradient factor in a linearized scalar model, and explain why deep decoder-only models commonly use Pre-Norm.
  3. Compare the parameter count, activated parameter count, and training stability trade-offs of GELU FFN, SwiGLU, and MoE FFN.
  4. Derive why d_ff = 8/3 * d_model in SwiGLU yields approximately the same parameter count as a 4x GELU FFN; let d_model=24, calculate the hidden width and parameter count of a bias-free GELU FFN and SwiGLU, and explain what token-conditional interactions the gated multiplication can express.
  5. Given B, T, d_model, n_heads, d_ff, dtype_bytes, estimate the attention parameter count, SwiGLU parameter count, major FLOPs, attention score memory, and major activation memory for a single Transformer block; then given a checkpointed fraction, calculate the activation memory saved by activation checkpointing, the extra recomputation FLOPs, and the training FLOPs multiplier, and explain which items are changed by FlashAttention.
  6. Design a subject-verb agreement mechanistic interpretability experiment: write the clean/corrupted prompt, target logit difference, activation patching location, ablation target, and explain what conclusions probing, patching, and ablation can each support.

Ch06 GPT Assembly / MoE

  1. Calculate the parameter count of GPT-2 small item by item: embedding, position embedding, 12 blocks, final LayerNorm, and tied LM head.
  2. Explain why weight tying both reduces parameters and potentially improves input/output embedding consistency; given logits = H E^T, target ids, and CE mean reduction, derive dH = G E and dE = G^T H, and explain how ignore_index changes the effective denominator.
  3. Given a bias-free SwiGLU expert with d_model=16, expert_hidden=32, 256 routed experts, top_k=8, and 1 shared expert, calculate the single expert parameter count, router parameter count, total expert parameters, activated expert parameters per token, and capacity-to-compute ratio; then given the expert token counts for a batch and the per-expert capacity, calculate the overflow and drop rate, and explain why load imbalance leads to throughput degradation or token dropping.
  4. Given router probabilities for 4 tokens to 3 experts [[0.7,0.2,0.1],[0.6,0.3,0.1],[0.1,0.8,0.1],[0.2,0.2,0.6]], top-1 expert ids [0,0,1,2], calculate the load fraction for each expert, the mean router probability, and the 3 * sum_i load_i * prob_i load-balancing loss; explain why looking only at router probabilities or only at top-k counts is insufficient.
  5. Given the token sequence [BOS, a, b, c, EOS], write the inputs, labels, and the prediction target for each logits[:, t, :] during GPT training; given a small vocab logits, manually calculate the next-token cross entropy using logits[:, :-1, :] and input_ids[:, 1:], and explain how causal leakage can make the training loss deceptively low.

Ch07 Training Loop

  1. Derive the next-token negative log likelihood from the autoregressive factorization, and explain its relationship with cross entropy.
  2. Derive the gradient of the single-sample cross entropy with respect to logits p_i - 1[i=y]; if the batch contains ignore_index positions, explain why the denominator for mean reduction should be the number of valid tokens, and why the gradient for logits at ignored positions should be 0.
  3. Given vocab size, target token, epsilon, and logits, construct the label-smoothed target distribution, calculate the label-smoothed cross entropy, and explain its relationship with hard one-hot CE and distillation soft targets.
  4. Let the tied LM head be z_t = h_t E^T, explain how the CE gradient updates the output embedding rows for the correct token and incorrect tokens, and how it continues to propagate back to the Transformer hidden state.
  5. Explain the difference between "decoupled weight decay" in AdamW and L2 regularization, and describe the role of warmup + cosine decay; given base_lr=0.2, num_warmup_steps=2, num_training_steps=6, min_lr_ratio=0.25, and each optimizer step consuming 1000 tokens, manually calculate the lr multiplier, actual lr, phase, and cumulative consumed tokens for steps 0, 1, 2, 6.
  6. Given micro_batch=4, seq_len=2048, grad_accum=8, data_parallel=16, training budget D=20B tokens, and dense LM parameter count N=1B, calculate the global batch tokens, number of training steps, and approximate training FLOPs; then calculate the training memory for bf16 parameters/gradients and fp32 AdamW two moment states, and explain which items are changed by optimizer-state sharding.
  7. Given a training token sequence and an evaluation token sequence, calculate the n-gram repetition rate and the train/eval overlap rate, and explain how they affect val loss and generalization judgment.
  8. Given two data sources: web 600k tokens, 600 documents, duplicate 0.03, eval overlap 0.001, quality pass 0.92; code 400k tokens, 400 documents, duplicate 0.02, eval overlap 0, quality pass 0.90. Under the thresholds min_total_tokens=900k, max_duplicate_rate=0.05, max_eval_overlap_rate=0.005, min_quality_pass_rate=0.88, min_domain_count=2, max_domain_token_share=0.7, calculate the weighted duplicate rate, weighted quality pass rate, domain shares, and determine whether the training_data_curation_report passes. Then explain why, if the PII rate or eval overlap fails, a low val loss cannot be directly used to support scaling up.
  9. Explain the core insight of the Chinchilla-style scaling law: why, under a fixed compute budget, it is necessary to simultaneously consider model parameters, training token count, and data quality, rather than only scaling up the parameter count.
  10. Given a set of logits, targets, and confidence bins, calculate the accuracy, mean confidence, and ECE for each bin, and explain how model overconfidence can affect the rejection threshold or risk control.
  11. Given the gradients of two parameter tensors, calculate the global grad norm, clip coefficient, and the clipped gradients; explain the difference from per-parameter clipping.
  12. Given grad_accum_steps=4, four micro-batch mean losses, the token count for each micro-batch, and a warmup scheduler, calculate the scaled loss used for each backward pass, the number of tokens consumed in one optimizer step, how many times the scheduler should advance, and if forgetting to divide by grad_accum_steps is equivalent to changing the learning rate in what way.
  13. Given training logs, list the possible causes and investigation order for a loss spike, NaN, a sudden increase in grad_norm, and a drop in tokens/s.
  14. Given a curve where train loss decreases but val loss increases, determine whether it is more likely overfitting, a data split issue, or a training objective error; explain which data and log fields you would check first.
  15. Given a 7B parameter model, 8 GPUs, bf16 parameters/gradients, fp32 AdamW m/v states, calculate the per-GPU model state memory for DDP, ZeRO-1, ZeRO-2, and ZeRO-3/FSDP respectively; then write the strategy, global batch tokens, memory gate, and action item in the distributed_training_strategy_report format, and explain that these estimates do not include activation, communication buffers, temporary tensors, and allocator fragmentation, and therefore cannot alone prove training feasibility.
  16. Given model parameters N=7B, throughput tokens/s=20000, number of GPUs 8, single-card peak 300 TFLOP/s, roughly estimate MFU using 6N FLOPs/token; if the MFU is only 18%, list at least 4 possible reasons, and explain how to use a profiler or logs to distinguish between a batch size that is too small, communication waiting, insufficient data loading, checkpoint writing to disk, and unfused kernels. If the strategy uses FP8/MXFP8, also write the verification evidence for scale/amax history, loss spike, gradient range, and checkpoint state. 17. Given two checkpoints: A is step=100, type=export, format=single_file, components=[model], write_complete=false, checkpoint_overhead_pct=0.12; B is step=90, type=resume, format=single_file, components=[model, optimizer, scheduler, global_step], interval_steps=500, checkpoint_overhead_pct=0.08. The target strategy is ZeRO/FSDP with world_size=4, requiring components model, optimizer, scheduler, global_step, RNG, sampler, grad_scaler, a maximum checkpoint interval of 200 steps, a maximum overhead of 5%, and async/overlap saving. According to the checkpoint_resume_integrity_report format, determine state completeness, write integrity, distributed reshard, interval, overhead gate, and action items; explain why a model-only export cannot support training resumption, and why sharded/DCP-type checkpoints are important for FSDP/ZeRO reshard.

Ch08 Generation / Decoding

  1. Compare the quality/diversity/reproducibility/latency trade-offs of greedy, temperature, top-k, top-p, and repetition penalty.
  2. Manually compute the top-p nucleus for a 5-token vocabulary: given probabilities [0.40, 0.25, 0.20, 0.10, 0.05], which tokens are retained when p=0.7 and p=0.9.
  3. Given logits [4.0, 3.0, -1.0], generated token ids [0,2], and a repetition penalty of 2.0, manually compute how the logits change before sampling; then use top_p=0.8 to compute the retained candidate tokens, renormalized probabilities, entropy, and the greedy token, and explain why the processing direction for positive and negative logits differs.
  4. Explain why beam search often requires a length penalty, and compare its applicability with sampling in open-ended generation tasks.
  5. Given a beam table, sort using raw logprob and length-normalized score respectively, and explain whether a change in ordering necessarily implies better quality.
  6. Given three rounds of speculative decoding records: gamma=4, the number of accepted draft tokens per round is [4,2,3], the actual number of output tokens written is [5,3,4], and the draft single-step cost is 0.2 times the target single-step cost. Calculate proposed, accepted, acceptance rate, baseline target steps, target verify calls, draft steps, approximate latency, and speedup, and explain why a draft model that is too weak or too strong might not be cost-effective.
  7. Given two rows of logits [[5,4,3],[1,2,9]] and valid token sets [[1,2],[0,1]], write down which positions are -inf after applying the constraint mask, and give the greedy decoding result; explain why constrained decoding requires renormalization.
  8. Design a small experiment comparing greedy, top-p, and temperature sampling: write down the prompt set, random seed, output length, distinct-n, repetition rate, task accuracy or human preference metric, and explain the limitations of each metric.
  9. Design a reasoning generation experiment for math or code problems: compare single-sample, self-consistency, best-of-N, and verifier reranking; given 5 candidate outputs, write down the final-answer extractor, the vote distribution for majority voting, and the tie-breaking rule; given c correct out of n samples, calculate pass@k = 1 - C(n-c,k)/C(n,k), and report the average output token count, latency, and cost per correct answer.

Ch09 Fine-tuning / Alignment

  1. Given a prompt/response token sequence, mark the positions in the SFT labels that should be -100, and explain the reason.
  2. Given tokenized messages: system 2 tokens, user 3 tokens, assistant 4 tokens, user 2 tokens, assistant 3 tokens, where the second assistant has weight=0; also given max_seq_len=10, pack_examples=True, block_diagonal_attention=False. Mark the assistant spans, supervised tokens, supervised_token_ratio, truncated assistant tokens, and the gates and action items that sft_chat_template_mask_report should fail on.

  4. Given policy chosen/rejected log-probs [3,1], [1,2], reference chosen/rejected log-probs [1,1], [1,1], and beta=0.5, calculate the DPO implicit chosen/rejected reward, reward margin, preference probability, and preference accuracy.
  5. Given a set of chosen/rejected rewards, calculate the Bradley-Terry pairwise reward loss and preference accuracy, and explain the relationship between this loss and reward model training.
  6. Given a set of chosen/rejected response lengths, calculate the mean length delta, chosen_longer_rate, rejected_longer_rate, and tie_rate, and explain how length bias affects DPO or RLHF.
  7. Given a table of SFT/preference/RLVR samples, manually calculate task coverage, safety slice coverage, max task share, label conflict rate, mean length delta ratio, eval overlap rate, and unsafe chosen rate, and determine which gates (coverage, label_quality, leakage, safety) in post_training_data_audit fail.

  9. Given the log probabilities of the policy/ref on sampled tokens, calculate the approximate KL exp(log_ref-log_policy) - (log_ref-log_policy) - 1, and explain why the padding mask should not be included in the mean denominator.
  10. Given old/new policy log-probs, advantages, and clip_range=0.2, calculate the PPO unclipped surrogate, clipped surrogate, final policy loss, clip fraction, and approximate KL; explain what PPO clipping and the KL penalty constrain respectively.

Given GRPO rewards [[1],[2],[3]], old log-probs all 0, new/old ratios [[[1.0,1.5]],[[1.0,1.0]],[[0.5,1.0]]], completion mask [[[1,1]],[[1,0]],[[1,1]]], reference log-probs all 0, clip_range=0.2, kl_beta=0.04, calculate the within-group whitened advantages, PPO clipped surrogate, masked policy loss, masked approximate KL, and total loss; explain which reward hacking problems GRPO within-group whitening cannot solve.
  12. Compare the impact of LoRA rank, alpha, and target modules on training parameter count, expressiveness, and merged inference.
  13. Design an evaluation plan to determine if a DPO model is better than the SFT/reference: it should at least cover helpfulness, factuality, safe refusal, over-refusal, and math/code capability retention, and explain why the preference win rate alone cannot serve as a conclusion.

Given tokenized prompts [[1,2,3,4],[1,2,3,9,10],[1,2,8],[7,8]], calculate the longest reusable prefix history, new prefill tokens, total saved prefill tokens, effective prefill tokens, and prefix cache hit rate for each request in request order; explain why random timestamps can break the prefix cache.

  8. Given an MCP/remote tool event: the tool server is not on the allowlist, risk=write, permissions=read/write, user approval not obtained, parameters contain sensitive data, the tool returns an external webpage containing a prompt injection, the observation is not isolated, the server requests recursive LLM sampling, and the total number of calls and observation tokens exceed the policy. Determine which gates (server trust, permission/consent, data privacy, observation isolation, sampling/budget) fail, and give action items; explain why passing the JSON schema does not mean the tool is executable.
  9. Given a stable baseline: pass_rate=0.88, safety_pass_rate=0.97, cost_per_1k_requests_usd=1.20; a candidate release package: pass_rate=0.90, safety_pass_rate=0.965, p95_latency_ms=860, error_rate=0.008, cost_per_1k_requests_usd=1.32, canary_traffic_pct=5, canary_sample_size=1200, already compared with control, rollback prepared, monitoring fields include pass/safety/P95/error/cost. Under the policy min_pass_rate=0.86, max_quality_regression_abs=0.02, min_safety_pass_rate=0.95, max_safety_regression_abs=0.02, max_p95_latency_ms=900, max_error_rate=0.01, max_cost_regression_pct=0.2, min_canary_sample_size=1000, max_canary_traffic_pct=10, evaluate each gate (offline quality, safety, SLO, cost, canary, rollback/monitoring) item by item, and decide whether to promote, continue low-traffic canary, or rollback/block. Then explain why an improvement in offline evaluation cannot skip canary, control, per-version monitoring, and rollback readiness.
  10. Given runtime metrics: waiting_requests=80, p95_queue_wait_ms=900, p95_ttft_ms=1800, p95_tpot_ms=95, kv_cache_usage_pct=0.96, swapped_requests=3, error_rate=0.03, timeout_rate=0.02, degradation_ready=False; policy: max_waiting_requests=20, max_p95_queue_wait_ms=200, max_p95_ttft_ms=900, max_p95_tpot_ms=60, max_kv_cache_usage_pct=0.9, max_swapped_requests=0, max_error_rate=0.01, max_timeout_rate=0.005; the request rate for the free-tier tenant is 4 times its quota. Evaluate each gate (queue pressure, KV pressure, decode saturation, error budget, tenant fairness, degradation readiness) item by item, give severity, decision, and action items; explain why load shedding, degradation, scaling, rollback, and noisy-neighbor isolation cannot be conflated into a single "service is slow" conclusion.
  11. Compare the goals, risks, and validation metrics of INT8 weight-only quantization, KV Cache quantization, and FP8/FP4 mixed precision.
  12. Explain why benchmark numbers for frontier models must simultaneously report the task, dataset, inference settings, and evaluation version.
  13. Design a RAG ablation experiment: compare dense-only, BM25-only, hybrid, hybrid+rerank, hybrid+MMR, and two sets of chunk size/overlap, and specify which retrieval metrics, generation quality metrics, latency, and token costs should be reported for each.
  14. Design a multimodal LLM evaluation: covering image QA, OCR/document understanding, chart numerical reasoning, and visual grounding respectively, and specify input resolution, visual token count, latency, KV Cache cost, and failure modes for each task type.
  15. Write a structured conclusion summary for an LLM benchmark: including task, sample_size, baseline, metrics, risks, uncertainty, and conclusion, and specify which conclusions cannot be drawn from this set of data.
  16. Given a service workload: QPS=10, average prompt tokens 800, average output tokens 200, single-GPU prefill capacity 120k prompt tokens/s, decode capacity 2.5k output tokens/s, calculate the prefill/decode token rate, and determine the main bottleneck; explain why when QPS continues to rise, TPOT, queueing time, and TTFT may deteriorate in a different order.
  17. Given a 32-layer GQA model, n_kv_heads=8, head_dim=128, fp16 KV Cache, average active context 6000 tokens, available KV memory 48 GiB, calculate the KV bytes per token, KV Cache per request, and the theoretical maximum number of active requests; then, considering a 20% safety margin, give the admission limit, and explain why limiting only by the number of concurrent requests would unfairly penalize short requests or allow long requests to pass through.

Given long-context service records: each contains context_tokens, max_context_tokens, reserved_output_tokens, needle_position_pct, answer_correct, citation_hit, ttft_ms, latency_ms, kv_cache_usage_pct and prefix_cache_hit_rate. According to long_context_serving_gate_report, calculate truncation rate, answer/citation recall, head/middle/tail bucket accuracy, P95 TTFT, P95 latency, maximum KV cache usage rate and average prefix cache hit rate; judge context fit, long-context quality, position robustness, and serving cost gate, and explain why "supporting 1M tokens" cannot replace needle/citation/position/SLO regression.

Classic NLP Topic Questions

  1. RNN/LSTM: Given scalar RNN parameters, manually compute hidden state, BPTT gradient chain product, and explain how LSTM's forget/input/output gates alter the long-term information path.
  2. Dependency Parsing: Given a sentence and gold arcs, write a valid shift/reduce sequence using the arc-standard transition system, and compute UAS/LAS.
