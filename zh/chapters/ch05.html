5.1 仅解码器架构概述——Transformer 块作为主要可复用单元

主流自回归大语言模型（如 GPT、Llama 和 DeepSeek 系列中的语言模型）采用 仅解码器（decoder-only）架构：没有独立的编码器，模型主要由一层层 Transformer 块 堆叠而成。仅编码器、编码器-解码器和混合架构在 NLP 和多模态系统中仍然很重要；本章聚焦于仅解码器路线。每个块包含两个核心子层：

  • 自注意力层（Self-Attention Layer）—— 允许 token 之间交换信息（第 4 章已介绍）
  • 前馈神经网络（Feed-Forward Neural Network, FFN）—— 对每个 token 独立执行非线性变换

每个子层都包裹了 残差连接 和 层归一化。整体形式（Pre-Norm 规范）为：

公式 5.1 Transformer 块（Pre-Norm）

这两个子层分工明确：注意力机制提供“信息检索”——每个 token 从整个序列中聚合信息；FFN 提供“信息处理”——每个 token 在接收信息后进行复杂的非线性变换。如果没有 FFN，模型就只是一个线性加权求和系统，表达能力极其有限。

每个块的输出维度保持为 ，这允许模型堆叠很多层。但深度能否稳定训练，还取决于归一化位置、初始化、残差缩放、优化器、学习率调度和数据规模等因素。

5.1A 示例：Pre-Norm 与 Post-Norm 的梯度路径

用标量残差块来近似归一化位置的影响。设子层为 f，归一化为 n。Pre-Norm 写作 y = x + f(n(x))，Post-Norm 写作 y = n(x + f(x))。局部线性化后得到：

局部梯度因子

考虑一个三层玩具块：子层斜率分别为 [0.2, -0.1, 0.3]，归一化的局部斜率约为 0.5。Pre-Norm 中每一层的梯度因子为：

Pre-Norm: [1 + 0.2*0.5, 1 - 0.1*0.5, 1 + 0.3*0.5] = [1.10, 0.95, 1.15] product = 1.20175

Post-Norm 中，归一化直接应用在残差输出之后：

Post-Norm: [0.5*(1 + 0.2), 0.5*(1 - 0.1), 0.5*(1 + 0.3)] = [0.60, 0.45, 0.65] product = 0.1755

这个例子说明 Pre-Norm 为什么更有利于深层网络中的梯度传播：它在残差路径中保留了一个显式的 1。不过，这并不保证无限深网络也能稳定训练；真实 Transformer 的 Jacobian 矩阵是高维的，还受 attention、FFN、初始化、学习率、残差缩放以及优化器的影响。Ch05 作业中的 residual_gradient_path_factors 让学习者重新计算这个局部线性化模型。

5.2 编程练习 1：实现 LayerNorm（不使用 nn.LayerNorm）

层归一化（Layer Normalization）是 Transformer 训练稳定的基础。与 BatchNorm 不同，LayerNorm 沿特征维度归一化，不依赖 batch size——这对变长序列和自回归模型很重要。

公式 5.2 LayerNorm

🖥️ 练习 1：从零实现 LayerNorm

任务：实现 LayerNorm 类，且不使用 nn.LayerNorm。

要求：

  • 可学习参数 gamma（尺度，初始化为全 1）和 beta（偏移，初始化为全 0）
  • eps=1e-6 防止除零
  • 支持任意输入形状：在最后一维进行归一化
  • 使用 unbiased=False 计算方差
  • 验证：随机输入下，输出与 nn.LayerNorm 的误差小于 1e-5
# Expected usage
ln = LayerNorm(dim=768)
x = torch.randn(2, 16, 768)
y = ln(x)
assert y.shape == x.shape

# Comparison with PyTorch official
ref = nn.LayerNorm(768, eps=1e-6)
ref.load_state_dict({'weight': ln.gamma, 'bias': ln.beta})
y_ref = ref(x)
assert torch.allclose(y, y_ref, atol=1e-5)
import torch import torch.nn as nn class LayerNorm(nn.Module): """Layer Normalization — normalizes along the feature dimension (without nn.LayerNorm)""" def __init__(self, dim, eps=1e-6): super().__init__() self.eps = eps self.gamma = nn.Parameter(torch.ones(dim)) self.beta = nn.Parameter(torch.zeros(dim)) def forward(self, x): # Calculate mean and variance (along the last dimension) mean = x.mean(dim=-1, keepdim=True) var = x.var(dim=-1, keepdim=True, unbiased=False) # Normalize x_norm = (x - mean) / torch.sqrt(var + self.eps) # Affine transformation return x_norm * self.gamma + self.beta # Verification ln = LayerNorm(768) x = torch.randn(2, 16, 768) y = ln(x) ref = nn.LayerNorm(768, eps=1e-6) ref.load_state_dict({'weight': ln.gamma, 'bias': ln.beta}) y_ref = ref(x) print(f"Max error: {(y - y_ref).abs().max().item():.2e}") assert torch.allclose(y, y_ref, atol=1e-5)

💡 关于 var(unbiased=False)：LayerNorm 使用总体方差（除以 N 而非 N-1），与样本方差不同。设置 unbiased=False 可确保与 nn.LayerNorm 的实现一致。

5.3 理论：Pre-Norm vs. Post-Norm — 归一化位置如何影响训练稳定性

归一化层放在子层的输入还是输出，对训练动态有决定性影响。这是大语言模型缩放过程中反复验证的一个关键设计选择。

Post-Norm（原始 Transformer）

原始 Transformer（Vaswani 等人，2017）使用了 Post-Norm：

Post-Norm

归一化发生在残差连接之后。残差分支的输出（Attention/FFN 的输出）会干扰残差流中原始信息的尺度。在深层网络中，这种干扰不断累积，导致梯度不稳定。

Pre-Norm（现代 LLM 标准）

Pre-Norm 将归一化放在子层的输入：

Pre-Norm

关键优势：残差流不被归一化。信息可以沿着残差路径跨层传播，较少受到归一化层的直接压制，梯度路径也更短、更直接。Pre-Norm 通常更容易训练深层网络；实际可达到的深度取决于初始化、优化器、残差缩放、数据和实现细节。

训练动态分析

更形式化地，考虑一个 L 层 Pre-Norm Transformer 的残差流：

残差流中的方差传播

在 Post-Norm 中，归一化发生在残差连接之后，因此 （被归一化强制为常数），但每个子层输出的方差 需要仔细控制。在 Pre-Norm 中，残差流的方差逐层增加（这是正常的——深层需要更大的表示范围），而不是被强制截断。

一个最小手工计算模型将每个子层线性化为局部斜率 f'，将归一化线性化为局部斜率 n'。Pre-Norm 残差层的近似梯度因子是 1 + f' n'，因为恒等捷径绕过了归一化；Post-Norm 的近似因子是 n'(1 + f')，其中归一化直接应用在捷径之后。Ch05 作业中的 residual_gradient_path_factors 使用这个标量模型让学习者手动计算多层梯度因子；它是一个局部诊断工具，并非针对任意深度的训练稳定性的完整证明。

属性Post-NormPre-Norm
代表性模型原始 Transformer（2017）GPT-2/3/4、Llama、DeepSeek
训练稳定性对初始化、warmup 和残差缩放更敏感通常更稳定，对学习率可能有更高的容忍度
深度缩放扩展到非常深的网络更困难更容易扩展到深层网络

🔑
关键结论：Pre-Norm 的成功源于“归一化隔离”——每个子层的输入被独立归一化，而残差分支保留了一条未经归一化的信息通道。这有助于深层网络中的梯度传播，但它并不是任意深度和学习率下稳定性的保证。Post-Norm 将归一化放在残差连接之后，通常需要更小心地处理初始化、warmup 和残差缩放。作业中的梯度因子函数对应了这种局部线性化解释。

5.4 编程练习 2：实现 RMSNorm——Llama/DeepSeek 的选择

RMSNorm (Zhang & Sennrich, 2019) 去掉了 LayerNorm 的均值中心化步骤，只保留 RMS 缩放。这是因为均值中心化对训练质量影响很小，而节省的计算量（约 15%）却很显著。

公式 5.3 RMSNorm

示例：RMSNorm 不进行均值中心化

取一个 4 维向量 x=[1, 2, 2, 3]，暂时忽略 eps 和 gamma。RMSNorm 计算平方的均值：

手动 RMS 计算

因此输出近似为 [0.471, 0.943, 0.943, 1.414]。它的 RMS 约等于 1，但均值仍然约为 0.883，并非 0。LayerNorm 会先减去均值 2.0 再除以标准差；RMSNorm 则保持向量方向不变，只缩放长度到稳定幅度。所以课后测验会要求检查输出的 RMS 是否接近 1，并确认输出均值不一定为 0。

RMSNorm 输入梯度

第二项是 RMSNorm 反向传播中容易遗漏的耦合项：每个特征都会影响共享的 RMS 分母，因此某一维度的梯度不仅受自身上游梯度影响。Ch05 作业中的 rms_norm_input_gradient 要求将此公式写成张量运算，并与 PyTorch autograd 对齐。

🖥️ 练习 2：RMSNorm 模块

任务：实现 RMSNorm ——仅执行 RMS 缩放，去掉均值中心化。

要求：

  • 唯一可学习的参数是 gamma（初始化为全1），无 beta
  • eps=1e-6 防止除零
  • 计算 rms = sqrt(mean(x^2) + eps)，然后 x / rms * gamma
  • 验证：RMSNorm(x) 的输出范数（L2 范数）约等于 gamma 的范数
  • 验证：RMSNorm 的输出 方向 与输入相同——它只改变向量的长度，不改变角度
# Expected usage
rms = RMSNorm(dim=768)
x = torch.randn(2, 16, 768)
y = rms(x)
assert y.shape == x.shape

# Verify direction invariance
cos_sim = (x * y).sum(dim=-1) / (x.norm(dim=-1) * y.norm(dim=-1))
assert torch.allclose(cos_sim, torch.ones_like(cos_sim), atol=1e-6)
# Direction invariance means cos_sim ≈ 1.0
class RMSNorm(nn.Module): """RMS Layer Normalization — Llama / DeepSeek standard""" def __init__(self, dim, eps=1e-6): super().__init__() self.eps = eps self.gamma = nn.Parameter(torch.ones(dim)) def forward(self, x): # RMS = sqrt(mean(x^2)) rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps) return x / rms * self.gamma def extra_repr(self): return f"dim={self.gamma.shape[0]}, eps={self.eps}" # Verify direction invariance rms = RMSNorm(768) x = torch.randn(2, 16, 768) y = rms(x) cos_sim = (x * y).sum(dim=-1) / (x.norm(dim=-1) * y.norm(dim=-1)) print(f"Direction Cosine (Mean): {cos_sim.mean().item():.6f}") # Should be ≈ 1.0 # Verify output norm ≈ sqrt(dim) * gamma print(f"Output Vector Norm (Mean): {y.norm(dim=-1).mean().item():.4f}") print(f"sqrt(dim) * gamma_norm: {(768 ** 0.5) * rms.gamma.norm().item():.4f}")

5.5 理论：FFN 作为键值记忆——模型如何存储知识

FFN 不只是“非线性变换”。从可解释性的角度看，Geva 等人（2021）指出 FFN 可以像一个大规模键值记忆那样运作。

重温 FFN

标准 FFN 的计算方式：

公式 5.4 FFN 重写为键值存储

这个求和中的每一项都对应一个“键值对”：

  • 键： 检测输入是否匹配某个模式。ReLU 门控决定是否激活该键：只有匹配的键才产生非零输出
  • 值： 是对应于键 i 的输出“知识片段”
  • 存储容量： 是键值对的数量。GPT-2 small 有 3072 个键值对，GPT-3 175B 每层有 4 × 12288 = 49152 个

知识的分层分布

不同层的 FFN 存储不同类型的知识：

  • 低层（1-4）： 句法模式——识别词性标注、短语边界等浅层句法结构
  • 中层（5-8）： 事实知识——例如“巴黎是法国的首都”“水在 100°C 沸腾”
  • 高层（9-12）： 长距离依赖与推理——例如代词消解、多步推理模式

对模型编辑的启示

这种视角直接启发了模型编辑方法。ROME（Meng 等人，2022）利用 FFN 的键值存储特性，找到存储特定事实的 FFN 神经元，直接修改它们的值向量来更新模型中的事实知识——无需重新训练。

💡
这意味着： 模型的知识主要集中在 FFN 权重中。注意力机制充当“检索协调员”——决定从哪些键值对中检索信息。这也解释了为什么增加 FFN 维度（更多键值对）比增加注意力维度更能提升模型容量。

5.5A 机制可解释性：从“看起来相关”到“改变它就能改变输出”

把 FFN 看作键值记忆后，自然会问：如何确定某个特定的 neuron、head、layer 或 residual stream 方向是否真正参与了某个具体行为？本课程区分三类问题：表示中是否包含该信息，模型是否使用了该信息，以及改变该信息是否真的会改变输出。

四种常用方法

方法核心做法能回答的问题不能单独解释的
Probing用一个线性/小型模型从 hidden states 中预测 POS、句法、实体等标签某类信息是否可以从表示中读取出来模型在生成时是否真的使用了该信息
Logit Attribution将 residual stream 或 neuron 的输出投影到 unembedding 方向上哪些组件直接推高或压低某些 token后续层产生的间接路径和重写效应
Activation Patching将某个扰动样本中某层的 activation 替换为干净样本中的 activation特定位置/层的 activation 是否足以恢复该行为是否只有这条通路对该行为负责
Ablation将特定 head/neuron/layer 的输出置零、替换或掩码移除某个组件后，性能或行为变化有多大模型是否通过冗余通路进行了补偿

一个最小案例：主谓一致

给定提示：The keys to the cabinet ...，模型应该更倾向于 are 而非 is。可解释性实验可以这样设计：

  1. 构建对比：将主语从复数 keys 改为单数 key，保持中间短语 to the cabinet 不变。
  2. 定位层和位置：比较主语 token、介词短语 token 和预测位置的 residual stream。
  3. 执行 patching：将复数样本中特定层和位置的 activation 替换到单数样本中，观察 are 和 is 之间的 logit 差是否恢复。
  4. 执行 ablation：掩码候选的 head 或 FFN neuron，观察一致性的 logit 差是否下降。

结论的边界

高 probe 准确率只说明 hidden state 中包含线性可读的信息。如果 patching 恢复了输出，则该 activation 对这一组 prompt 的行为具有因果效应。如果 ablation 使性能下降，则该组件对当前任务很重要。这些结论依赖于 prompt 分布、层位置、替换方法和评估指标，因此不能推广为“模型的全部句法能力都在这个 head 里”。

🔑
工程要点：可解释性不是画热力图。严谨的做法是先定义一个行为度量（例如目标 logit 差），再用对比样本、patching 和 ablation 来确定哪些内部状态对该行为有影响。

5.5B 可解释性方法的谱系：从读取表征到行为干预

机制可解释性包含一系列证据强度递增的方法。本课程的重点是阐明每种方法回答什么问题、需要哪些控制、以及结论可以推到多远。推荐的顺序是：先观察表征，再做归因，然后进行因果干预，最后再讨论引导或模型编辑。

2. Logit lens 观察“中间层已经接近哪个答案”

Logit lens 方法直接将某一层的残差流乘以最终的 unembedding 矩阵，观察中间表征偏向哪些 token。如果在国家-首都问题上，第 8 层已经把 Paris 排在前几位，说明中间表征已经包含了一些答案倾向；如果后续层将其改为另一个 token，说明后续层仍在重写或整合信息。

Logit lens

这里，target 是正确 token，foil 是对比 token。例如，在主谓一致任务中，使用 are 和 is 之间的 logit 差。这个指标比只看 top-1 token 更稳定，它直接对应研究问题。

3. 因果追踪需要干净/损坏的对比

可解释性实验不应只看单个 prompt。更稳健的做法是构造一对输入：触发正确行为的干净 prompt，以及破坏关键信息的损坏 prompt。然后将干净运行的某一层某个位置的激活修补到损坏运行中，观察目标行为是否恢复。

  1. 定义行为指标：例如 logit(Paris) - logit(Rome)，或 logit(are) - logit(is)。
  2. 构建最小扰动：只改变国家名、主语数、实体名称或事实槽位，尽量保持其他 token 不变。
  3. 扫描层和位置：分别修补残差流、注意力输出和 MLP 输出。
  4. 报告恢复量：不要说“有效果”，而要报告恢复了多少比例的干净-损坏差距。

如果修补某一层的 MLP 输出能恢复目标 logit 差，说明该激活对该 prompt 对的行为有因果效应；但还不能证明所有相似事实都存储在同一位置，也不能证明编辑该位置没有副作用。

示例：从归因升级到修补的结论

假设模型看到 The capital of France is，目标 token 是 Paris，对比 token 是 Rome。你发现第 10 层的 MLP 对 Paris 有很高的 logit 归因。这仅仅表明它朝着 Paris 的方向直接写入残差流。如果你进一步将第 10 层 MLP 从干净 prompt 的输出修补到损坏 prompt The capital of Italy is，并观察到 Paris 相对于 Rome 显著上升，这时才能说该激活足以承载法国→巴黎的信息。如果对同一个 MLP 进行消融后，法国 prompt 仍能正确回答，则说明模型可能有冗余路径。

4. 从解释到干预必须评估副作用

激活引导和模型编辑很有吸引力，它们似乎能直接改变模型行为。但从工程角度看，必须同时评估目标成功率和副作用：

  • 定位性（Locality）：编辑法国→巴黎是否会影响到意大利、德国等其他事实？
  • 泛化性（Generalization）：在不同的措辞、语言或上下文长度下是否仍然有效？
  • 能力保持（Capability retention）：数学、代码、长上下文以及拒绝边界是否下降？
  • 回滚能力（Rollback capability）：干预是否可以版本化、比较和撤销？
🔑
工程要点：可解释性的价值不在于“命名神经元”，而在于将模型行为分解为可检验的假设。你越接近干预和编辑，就越需要报告副作用、泛化性和回滚边界。

🖥️ 练习 3：实现 FFN (GELU) 和 SwiGLU

任务：实现 FFN (GELU) 和 SwiGLU 模块。

要求 (FFN)：

  • 两个线性层：d_model → d_ff → d_model
  • 中间使用 GELU 激活（使用 F.gelu）
  • 默认 d_ff = 4 * d_model（GPT 风格）

要求 (SwiGLU)：

  • 三个线性层：w1（门控）、w2（值）、w3（输出）
  • 默认 d_ff = int(8/3 * d_model)——确保总参数量匹配 4 倍 FFN
  • 对门控分支使用 F.silu 激活

import torch.nn.functional as F class FFN(nn.Module): """Standard FFN (GELU) — GPT-2 style""" def __init__(self, d_model, d_ff=None): super().__init__() d_ff = d_ff or d_model * 4 self.fc1 = nn.Linear(d_model, d_ff, bias=True) self.fc2 = nn.Linear(d_ff, d_model, bias=True) def forward(self, x): return self.fc2(F.gelu(self.fc1(x))) class SwiGLU(nn.Module): """Gated SwiGLU — Llama / DeepSeek style""" def __init__(self, d_model, d_ff=None): super().__init__() d_ff = d_ff or int(d_model * 8 / 3) # Keep parameter count consistent with 4x FFN self.w1 = nn.Linear(d_model, d_ff, bias=False) # Gate self.w2 = nn.Linear(d_model, d_ff, bias=False) # Value self.w3 = nn.Linear(d_ff, d_model, bias=False) # Output def forward(self, x): gate = F.silu(self.w1(x)) # (b, s, d_ff) value = self.w2(x) # (b, s, d_ff) return self.w3(gate * value) # (b, s, d_model) # Parameter count comparison ffn = FFN(768) swiglu = SwiGLU(768) print(f"FFN parameter count: {sum(p.numel() for p in ffn.parameters()):,}") print(f"SwiGLU parameter count: {sum(p.numel() for p in swiglu.parameters()):,}") print(f"Ratio: {sum(p.numel() for p in swiglu.parameters()) / sum(p.numel() for p in ffn.parameters()):.3f}")

💡 为什么常采用 SwiGLU：门控机制让模型能够选择性地传递信息。当门控值接近 0 时，该维度的信息被阻断。与 GELU 那种“所有神经元都激活，但程度不同”的行为相比，SwiGLU 提供了更灵活的 token 级别控制。SwiGLU 在许多 Transformer 报告中表现良好，但具体的困惑度（PPL）提升取决于模型规模、数据、宽度平衡、训练预算和实现细节；不应将其视为固定收益。

在GELU前馈网络中，中间神经元的输出主要由单一路径 决定。SwiGLU引入了逐元素乘法：

SwiGLU与MoE的区别

SwiGLU是一个稠密前馈网络：每个token都经过同一组矩阵，沿隐藏维度施加软门控。MoE是一个稀疏前馈网络：每个token只激活少数专家，由路由器选择其中几个。两者都引入了“选择性计算”，但粒度不同：

SwiGLU单个前馈网络的维度内部所有矩阵参与计算在相同参数预算下增强条件交互激活内存和稠密计算仍然存在
MoE跨越多个专家仅选择top-k个专家总参数量显著增加，而激活参数可控负载均衡、通信和路由稳定性更复杂
🔑
设计结论：SwiGLU不是简单的“多了一个矩阵所以更强”。它在近似相同的参数预算下，通过门控乘法增强了token条件化的表示能力；而MoE则以更高的系统复杂度为代价，将前馈网络容量扩展到多个稀疏专家上。

class TransformerBlock(nn.Module): """Complete Transformer Block — Pre-Norm + RMSNorm + SwiGLU""" def __init__(self, d_model, n_heads, d_ff=None, dropout=0.0): super().__init__() # Attention sublayer self.attention = MultiHeadAttention( d_model=d_model, n_heads=n_heads, dropout=dropout ) self.norm1 = RMSNorm(d_model) # FFN sublayer self.ffn = SwiGLU(d_model, d_ff) self.norm2 = RMSNorm(d_model) self.dropout = nn.Dropout(dropout) def forward(self, x, mask=None): # Sublayer 1: Self-Attention (+ Pre-Norm + Residual) identity = x x = self.norm1(x) x, _ = self.attention(x, mask=mask) # Ignore attn_weights x = self.dropout(x) x = x + identity # Sublayer 2: FFN (+ Pre-Norm + Residual) identity = x x = self.norm2(x) x = self.ffn(x) x = self.dropout(x) x = x + identity return x # Test block = TransformerBlock(d_model=768, n_heads=12, d_ff=2048) x = torch.randn(2, 64, 768) mask = torch.tril(torch.ones(64, 64)).unsqueeze(0).unsqueeze(0) out = block(x, mask=mask) assert out.shape == (2, 64, 768) loss = out.sum() loss.backward() print(f"All parameters have gradients: {all(p.grad is not None for p in block.parameters())}")

5.7A 模块实现稳定性手册：残差、初始化、Dropout 与精度

Transformer 模块的数学形式很简洁，但训练是否稳定取决于很多实现细节。课程级实现可以先写出 Pre-Norm + RMSNorm + Attention + SwiGLU；工程级实现还必须解释残差分支如何初始化、dropout 放在哪里、norm 使用什么精度、是否保留 bias、如何选择 checkpointing，以及这些选择如何影响损失曲线、梯度范数和吞吐量。

1. 控制残差分支的初始扰动

Pre-Norm 保留了直接的梯度路径，但如果每个子层从一开始就对残差流写入过大的更新，深层堆叠仍可能变得不稳定。常见做法包括更小的初始化、依赖于深度的残差投影缩放，或使用可学习的残差门。抽象写法如下：

残差更新尺度

可以是固定缩放、初始化策略的隐式结果，或显式门控。它的作用不是改变模型表示能力，而是在训练早期控制每个层对残差流的扰动幅度。如果在最初几百步中 grad_norm 频繁出现尖峰，请检查残差输出尺度和初始化，同时检查学习率和数据异常。

2. Dropout 位置影响训练与推理等价性

教学代码通常将 dropout 放在 attention/FFN 输出之后、残差相加之前：

x = x + dropout(attention(norm1(x), mask)) x = x + dropout(ffn(norm2(x)))

这个位置的含义是：训练时随机丢弃子层写入残差流的部分更新；推理时必须关闭 dropout。如果将 dropout 放在残差之后，会随机破坏恒等路径；如果在注意力概率上使用 dropout，会随机丢弃 key 的贡献，其效果与输出 dropout 不同。如果训练报告比较消融实验，必须明确说明 dropout 的位置和概率。

3. Norm 和 softmax 通常需要更高精度

在 bf16/fp16 训练中，矩阵乘法可以使用低精度张量核心，但归一化统计量、softmax 分母以及某些累加最好在 fp32 中完成，然后再转换回模型 dtype。否则可能出现两类问题：

  • RMSNorm/LayerNorm 统计误差：低精度的平方和与均值会在较大维度上累积误差，影响尺度稳定性。
  • Attention softmax 溢出或下溢：过大的 logits、不合适的 mask 哨兵值或完全被掩码的行会导致 NaN。

许多高性能内核在内部处理这些细节；手写教学实现必须显式选择哪些张量以更高精度累加。

4. Bias 策略必须与架构惯例一致

现代仅解码器 LLM 通常省略 QKV、输出投影和 SwiGLU 线性层中的 bias，并使用 RMSNorm 代替带 beta 的 LayerNorm。这不是严格定理，而是一套工程惯例：减少参数和内核分支，并与 Pre-Norm/RMSNorm 的尺度控制配合。如果复现特定 checkpoint，bias 策略必须与原始模型一致；否则参数数量、state_dict 键和输出分布都会不匹配。

5. 激活检查点按子层选择

Checkpointing 无需对整个模块使用统一设置。更有效的选择取决于哪些激活占据内存主导：

子层主要激活何时优先使用检查点代价
AttentionQ/K/V、注意力分数或内核内部状态长序列，注意力分数内存占主导重算 QKV、分数、softmax/值混合
SwiGLU FFN门控/值中间激活短序列但 d_ff 很大，FFN 激活占主导重算三个线性层和 SiLU/门控乘法
完整模块层两个子层的所有中间状态极端内存压力，需要最大节省最多重算，tokens/s 下降更明显

因此，第五章中的 activation_checkpointing_tradeoff 只是第一步；实际训练需要结合 profiler 查看每个子层的激活字节、前向时间和反向重算时间。

5.8 理论：梯度检查——为什么要验证反向传播？

在实现复杂自定义算子时，手写的反向函数可能包含数学错误。即使是单个符号错误，也可能导致模型“训练”很久却不收敛。

有限差分的数学原理

PyTorch 的 torch.autograd.gradcheck 使用中心差分来近似数值梯度：

中心差分的误差为 ，前向差分则为 。因此，中心差分更精确——代价是在每个参数维度上需要两次前向传播。

为什么 gradcheck 很重要

考虑一个手动实现的反向函数，其中有一个符号错误：

# 错误的反向：应该是 dx = grad * w，但写成 dx = grad / w # gradcheck 会检测到数值梯度和解析梯度之间的差异 > atol

这类错误在简单模块中容易发现，但在复杂的自定义算子（如用于 FlashAttention 的 Triton kernel）中，手动推导和实现反向传播是常见的调试环节。

⚠️
重要提示： gradcheck 对每个参数维度都需要额外两次前向传播。对于大模型，这代价极高。因此：使用 float64（有限差分对 float32 的舍入误差很敏感）；用很小的输入进行验证（batch=2, seq=8, dim=64）；设置 eps=1e-4 和 atol=1e-3。

5.9 编程练习 5：用 torch.autograd.gradcheck 验证反向传播

🖥️ 练习 5：LayerNorm 梯度检查

任务：用 torch.autograd.gradcheck 检查 LayerNorm 实现的反向传播是否正确。

要求：

  • 定义 LayerNormFunction（继承 torch.autograd.Function），手动实现前向和反向传播
  • 在 backward 中手动实现反向公式，并验证其正确性
  • 使用 gradcheck 比较数值梯度和解析梯度
  • 先在小的输入上测试（batch=2, seq=8, dim=64），再在更大的输入上测试
  • 使用 float64 保证数值精度
# Expected usage
ln_func = LayerNormFunction.apply
x = torch.randn(2, 8, 64, dtype=torch.float64, requires_grad=True)
gamma = torch.ones(64, dtype=torch.float64, requires_grad=True)
beta = torch.zeros(64, dtype=torch.float64, requires_grad=True)

test = gradcheck(ln_func, (x, gamma, beta, 1e-6), eps=1e-4, atol=1e-3)
print(f"Gradient check passed: {test}")  # Should be True
from torch.autograd import gradcheck class LayerNormFunction(torch.autograd.Function): """Custom LayerNorm — manual forward + backward for gradcheck verification""" @staticmethod def forward(ctx, x, gamma, beta, eps=1e-6): mean = x.mean(-1, keepdim=True) var = x.var(-1, keepdim=True, unbiased=False) inv_std = torch.rsqrt(var + eps) x_hat = (x - mean) * inv_std ctx.save_for_backward(x_hat, inv_std, gamma) return x_hat * gamma + beta @staticmethod def backward(ctx, grad_output): x_hat, inv_std, gamma = ctx.saved_tensors n = x_hat.shape[-1] grad_x_hat = grad_output * gamma grad_x = inv_std / n * ( n * grad_x_hat - grad_x_hat.sum(dim=-1, keepdim=True) - x_hat * (grad_x_hat * x_hat).sum(dim=-1, keepdim=True) ) reduce_dims = tuple(range(grad_output.dim() - 1)) grad_gamma = (grad_output * x_hat).sum(dim=reduce_dims) grad_beta = grad_output.sum(dim=reduce_dims) return grad_x, grad_gamma, grad_beta, None # Test on small inputs (float64) x = torch.randn(2, 8, 64, dtype=torch.float64, requires_grad=True) gamma = torch.ones(64, dtype=torch.float64, requires_grad=True) beta = torch.zeros(64, dtype=torch.float64, requires_grad=True) test = gradcheck( LayerNormFunction.apply, (x, gamma, beta, 1e-6), eps=1e-4, atol=1e-3 ) print(f"LayerNorm gradient check: {test}") # Verify our own LayerNorm module ln = LayerNorm(64).double() x = torch.randn(2, 16, 64, dtype=torch.float64, requires_grad=True) test_module = gradcheck( lambda x: ln(x).sum(), x, eps=1e-4, atol=1e-3, raise_exception=False ) print(f"LayerNorm module gradient check: {test_module}")

💡 最佳实践：建议逐个模块检查梯度：先单独测 LayerNorm，再测 SwiGLU，最后测整个 Block。一次性测所有模块遇到错误时，很难定位问题来源。

5.10 理论：可学习超连接（Learnable Hyper-Connections）——Birkhoff约束与Sinkhorn-Knopp迭代

超连接（Hyper-Connections）是一类扩展传统残差连接的设计：与每层只有固定的1:1相邻残差相加不同，它们学习更灵活的层间连接权重。一些前沿设计将连接矩阵约束到Birkhoff多面体（Birkhoff polytope）之类的结构化集合上，以避免可学习连接矩阵带来的过度信号放大。

传统残差连接的局限性

残差连接 假设每层有固定的1:1贡献比例。然而，不同层对最终输出的贡献可能不同——某些层可能只需微调上一层的表示，而另一些层可能需要大幅修改它。

Birkhoff多面体（双随机矩阵）

可学习超连接可以使用连接矩阵 来控制层间连接的强度，但直接学习 可能导致梯度放大。一种常见思路是约束 逼近Birkhoff多面体内的双随机矩阵。

Birkhoff多面体定义

双随机矩阵有一个有用的性质：其谱范数上界为 （在相应假设下成立）。这个性质可以约束层间连接的放大效应，降低梯度放大风险；但实际训练稳定性还取决于残差缩放、优化器、数据分布和实现细节。

Sinkhorn-Knopp迭代

将任意矩阵投影到Birkhoff多面体的常用算法如下：

Sinkhorn-Knopp迭代

交替进行行归一化和列归一化。当矩阵元素为正、目标边际可行时，Sinkhorn-Knopp迭代收敛到满足给定行/列边际的缩放矩阵；收敛速度和数值稳定性取决于矩阵条件数、温度/缩放和迭代次数。实践中通常使用少量迭代获得近似双随机结构，但不能保证10次迭代对任意输入都足够。

def sinkhorn_knopp(M, num_iter=10): """Sinkhorn-Knopp迭代，投影到Birkhoff多面体上""" M = torch.exp(M - M.max(dim=-1, keepdim=True).values) M = M / (M.sum(dim=-1, keepdim=True) + 1e-6) for _ in range(num_iter): M = M / (M.sum(dim=-1, keepdim=True) + 1e-6) M = M / (M.sum(dim=-2, keepdim=True) + 1e-6) return M

多头超连接可以使用多个独立的连接矩阵来捕获不同类型的层间依赖，然后合并连接结果。本课程关注这类方法的通用数学结构：连接强度可学习，但需通过非负性、行/列归一化、残差缩放等约束来控制信号放大。具体收益必须在给定模型、深度、优化器、初始化和训练数据下通过实证衡量；单个技术报告中的数据不能视为通用常数。

🔑
直觉理解：Sinkhorn-Knopp迭代的交替归一化与熵正则化最优传输密切相关。得到的连接权重在行和列两个维度上近似满足概率分布约束，使得每层“提出”和“接收”的信息量更可控。这有助于解释该约束为何可能改善深度训练动态，但不能独立保证完全的训练稳定性。
