怎么学这一章：看模型边界，不是背历史

  • 本章产出：RNN 梯度路径、arc-standard parsing trace、seq2seq additive attention、MLM mask、BERT QA span、NER span F1、BLEU/ROUGE/UAS/LAS/EM/F1。

  • 练习产出：RNN 梯度、句法转移、seq2seq attention、MLM masking、BERT QA span、NER span F1、评估总结。可选练习在 assignments/ch11_classic_nlp/。
本章统一框架：先判断任务类型，再选择模型家族、输出空间和指标。很多生产问题不该直接变成“让大模型自由生成”。

11.1 为什么 LLM 课程还要学经典 NLP

跳过中间模型，很容易误以为所有 NLP 问题都应该写成 prompt generation。真实系统通常是混合的：bi-encoder 做向量召回，cross-encoder 做 rerank，BERT/DeBERTa 做分类和抽取式 QA，decoder-only LLM 做生成、解释和复杂推理。

本章不要求背旧模型名，而是抓住三件稳定的事：

  • Inductive bias：RNN 按时间递推，parser 输出树，seq2seq 区分源序列和目标序列，BERT 双向看上下文。
  • Objective：Causal LM 预测下一个 token，MLM 只在 mask 位置算 loss，seq2seq 最大化条件生成概率，parser 最大化结构动作或 head/label 概率。
  • Metric：BLEU、ROUGE、EM/F1、UAS/LAS、retrieval metrics 回答的是不同问题，不能被一个总分替代。

11.2 RNN 与梯度路径

RNN 的隐藏状态递推是：

RNN recurrence

它把任意长度序列压进一个状态向量。优点是有强序列偏置，缺点是长距离依赖困难。如果 loss 在最后一步，早期输入的梯度要穿过很多局部 Jacobian：

BPTT gradient path

这些因子持续小于 1，梯度消失；持续大于 1，梯度爆炸。LSTM/GRU 用门控和近似线性记忆路径缓解问题。Transformer self-attention 把任意两点之间的路径缩短到一次 attention，但付出了二次 attention 矩阵和显存/带宽成本。

11.3 依存句法：结构化预测不是 attention 热力图

Dependency parsing 为每个 token 预测 head 和 dependency label，输出一棵有向树。Attention 是连续权重，可以做诊断；dependency tree 是离散结构，必须满足单 root、无环、每个词一个 head 等约束。

动作前提效果
SHIFTbuffer 非空把 buffer 第一个 token 推入 stack
LEFT_ARC(label)stack 至少两个元素让栈顶成为次栈顶的 head，并移除次栈顶
RIGHT_ARC(label)stack 至少两个元素让次栈顶成为栈顶的 head，并移除栈顶
ROOT(label)只剩一个 root 候选把它接到 ROOT

例子：解析 I saw her

目标树是 saw -> I (nsubj)、saw -> her (obj)、ROOT -> saw (root)。

stepstackbufferactionnew arc
0[][I, saw, her]SHIFT-
1[I][saw, her]SHIFT-
2[I, saw][her]LEFT_ARC(nsubj)saw -> I
3[saw][her]SHIFT-
4[saw, her][]RIGHT_ARC(obj)saw -> her
5[saw][]ROOT(root)ROOT -> saw

评估常用 UAS 和 LAS：

Attachment scores

UAS 高但 LAS 低，说明 head 找对了，关系类型错了。这会影响信息抽取、问答证据和语义角色分析。

11.4 Seq2Seq 与 Additive Attention

Encoder-decoder 明确区分 source sequence 和 target sequence。机器翻译、摘要、语音识别都常用这种条件生成形式：

Conditional generation

Additive attention 在 decoder 每一步给 source positions 打分：

Bahdanau attention

训练时 teacher forcing 喂 gold prefix；推理时只能喂模型自己生成的 prefix，会出现 exposure bias。Beam search 能缓解搜索错误，但会带来长度偏置，所以要配合长度归一化和任务指标。

11.5 BERT / Encoder-only：MLM、分类和抽取式 QA

BERT-style encoder 能双向看输入上下文，不是自回归生成器。MLM 只在 masked positions 上计算 cross-entropy，未 mask 位置通常设为 ignore index。不要把“输入里所有 token 可见”和“所有位置都参与 loss”混在一起。

维度Encoder-only / BERTDecoder-only / GPT
可见性双向 self-attentionCausal mask，只看左侧 prefix
预训练目标MLM，预测被 mask 的位置Next-token prediction
常见用途分类、rerank、embedding、token classification、span QA开放生成、代码、对话、推理
输出约束固定标签或输入 span生成新 token 序列

抽取式 QA 通常把输入组织为 [CLS] question [SEP] passage [SEP]，模型分别预测 answer start 和 end。合法 span 要满足 start <= end 且长度不超过上限：

Extractive QA span score

BIO 序列标注

NER、slot filling、token classification 常用 B-TYPE、I-TYPE、O 标注。实体评估通常看 span-level F1：只有 type、start、end 都完全匹配才算 TP。边界错一个 token，token accuracy 可能还高，但实体抽取已经错了。

Viterbi 结构化解码

逐 token argmax 会忽略标签转移约束。Linear-chain CRF 把路径分数写成 emission + transition：

Linear-chain path score

Viterbi 用动态规划找最高分路径，复杂度 O(TK^2)。CRF 训练不是只看最高分路径，还要用 Forward algorithm 对所有路径做 logsumexp 归一化。

11.5A MLM loss 与抽取式 QA span

例子：输入 [CLS] the cat sat [SEP]，mask 位置是 2 和 3：

masked input: [CLS] the [MASK] [MASK] [SEP] labels: None None cat sat None

MLM loss 只在两个 masked positions 上平均。若 position 2 预测对 cat，position 3 错预测 dog，masked-token accuracy 就是 1/2。未 mask token 参与上下文编码，但不进入 loss denominator。

BERT QA 也是受约束输出。若问题是 who founded OpenAI，passage 里有 Sam Altman founded OpenAI，模型通过 start/end logits 选择 Sam Altman 这个输入 span；如果 [CLS] 的 no-answer 分数最高，就返回无答案。这和 decoder-only LLM 生成解释不是同一类问题。

Viterbi 会发现最佳路径是 O, B-PER，分数 1.2 + 0.6 + 1.1 = 2.9。CRF NLL 则要比较 gold path 和所有路径：

即使当前 Viterbi 已经预测对，只要第二名路径分数很接近，CRF NLL 仍然会给梯度。这就是结构化训练比“预测对没对”更细的地方。

11.6 评估：BLEU、ROUGE、EM/F1 和结构指标

指标必须匹配任务结构。它们都不是“语言质量总分”。

指标核心计算适用任务主要限制
BLEUclipped n-gram precision + brevity penalty机器翻译不理解改写和语义等价
ROUGE-L最长公共子序列的 P/R/F1摘要、生成比较偏词面重叠，不保证事实正确
EM/F1答案归一化后的精确匹配或 token overlap抽取 QA、短答案别名、多答案、推理答案要额外处理
UAS/LAShead 是否正确，label 是否也正确依存句法只评估句法结构，不等于下游成功

BLEU 会裁剪重复词：candidate 是 the the the the，reference 是 the cat is here，unigram 里 the 最多只能算 1 次，所以 precision 是 1/4。ROUGE-L 看最长公共子序列。QA 的 EM/F1 要先做 lower、去标点、去冠词、合并空格，再和多个 gold answers 取最高分。

11.6A 现代 LLM 评估协议：从单分数到可复现结论

现代 LLM 还要评估开放回答、长上下文、工具调用、RAG、拒答、安全和多轮对话。最常见错误是把一个 benchmark score、一个 LLM-as-judge win rate 或几条人工偏好写成“模型更好”。

组成部分要讲清什么常见失败
Data样本来源、train/dev/test、去重、是否可能污染污染抬高分数，dev 被反复调参后不再代表泛化
Inference settings模型版本、prompt、temperature、top-p、max tokens、RAG/tool 配置两个模型用不同 prompt 或采样参数比较
Metrics自动指标、人评、judge、系统指标、安全指标各测什么平均分掩盖事实性、格式、延迟和拒答问题
Statistics样本量、重复运行、置信区间、pairwise test、错误类型小样本 win rate 波动被当成稳定结论
Boundary结果能外推到哪里，不能外推到哪里从单任务推到通用能力或生产可靠性

LLM-as-judge 必须校准

LLM-as-judge 会有位置偏差、长度偏差、风格偏差和同源模型偏差。更稳的做法是 blind pairwise comparison：隐藏模型名，随机交换 A/B 顺序，记录 tie，并拿一小部分人工 gold labels 校准。

win rate 可以写成：

Pairwise win rate with ties

但这个数字不能单独证明模型值得上线，还要看样本是否代表目标场景、失败类型、延迟、成本和安全。

安全评估要区分拒答和过度拒答

  • Attack success rate：有害请求里模型给出危险帮助的比例，越低越好。
  • Refusal rate：拒答比例，要按 harmful / benign 分开看。
  • Over-refusal rate：本该帮助的良性敏感请求被拒绝的比例。
  • Task utility：普通任务上的准确率、帮助性和完成质量。

严谨写法不是“Model A 好 8%”，而是：在固定 prompt、temperature=0、样本量 n=300 的中文客服 QA 上，A 相对 B 的 pairwise win rate 是 58%，主要改善来自格式遵循和引用完整性；事实错误率从 11% 降到 8%，但 P95 TTFT 从 720ms 增到 980ms。这个结论只支持进入灰度，不支持“通用更强”。

11.6B Benchmark 有效性：污染、置信区间和消融

Benchmark 难点不是选哪个榜，而是分数能不能支持工程决策。模型可能因为看过题、prompt 更适配、输出更长、采样更多、judge 偏它的风格而赢。

数据污染和 split leakage 不是一回事

Split leakage 常发生在自己的数据管线里：同一文档、会话、题库或近似改写同时进 train/dev/test。可以通过 document-level split、ID 去重、n-gram overlap、embedding nearest-neighbor 和人工抽查缓解。

Benchmark contamination 更广：公开测试题、答案解析、讨论帖、代码仓库可能早已进过预训练或 SFT 数据。不能完全消除时，报告必须写明数据来源、发布日期、去重方法和高相似样本处理。

小样本提升要有不确定性

如果 A 答对 61/100，B 答对 56/100，不能直接写“A 提升 5%”。当 n=100、p=0.61，95% 区间大约是 61% ± 9.6%。这不足以支持强结论。开放生成、RAG 和 judge win rate 常用 bootstrap 估计区间。

消融一次只回答一个问题

如果你同时改 base model、prompt、retriever、reranker、temperature 和 context length，分数提升了也不知道是谁的贡献。合理消融应该固定模型、prompt、数据和采样参数，只改变一个变量，并同时报告质量、延迟、成本和失败类型。

11.6C 安全与社会影响：风险分类、触发条件和工程缓解

安全不是上线前贴的一段声明。风险来自数据、训练目标、推理接口、应用场景和用户群的组合。同一个模型在闲聊、医疗摘要、金融建议、教育辅导和代码执行里的错误代价完全不同。

风险类别典型触发条件先检查什么常见缓解
幻觉和误导开放 QA、RAG 召回不足、用户要求确定答案事实错误率、引用覆盖、不知道时的表达带引用检索、不确定性表达、高风险人工复核
隐私和个人信息训练/日志/RAG index 含 PII数据来源、日志保留、访问权限、输出泄露PII 过滤、最小日志、租户隔离、权限检索
偏见和代表性不足语种、地区、群体数据不均衡按语言、地区、人群和任务切片错误率数据再平衡、切片评估、多样标注规范
滥用和越狱危险请求、多轮绕过、工具/代码权限有害完成率、越狱成功率、工具边界能力分级、sandbox、拒答策略、二次确认
过度拒答安全数据比例过高，把良性敏感请求当危险benign sensitive false refusal加入可回答样本，把拒答和帮助性分开训练

例如医疗教育 RAG，至少要报告 retrieval recall、citation accuracy、factual error、harmful advice、普通医学教育完成率、紧急情况转介率、PII 日志处理和 UI 限制说明。安全、alignment、evaluation、serving 是同一条流水线。

11.6D Agent 和 Workflow 评估：最终答案、轨迹、状态和成本

Agent 评估比单轮答案更严格，因为模型可以搜索、改文件、调用工具、写数据库、多轮交互。最终答案正确不够：轨迹可能用了禁用工具、改错状态、泄露隐私，或者耗时耗钱不可接受。

证据维度要回答的问题例子
Final result用户可见目标是否完成精确答案、hidden tests、最终状态 diff、人评 rubric
Trajectory legality中间动作是否合法tool allowlist、权限边界、无禁用命令、无危险外部调用
State integrity是否只改了应该改的对象数据库 diff、文件 diff、PASS_TO_PASS、rollback check
Reliability同一任务重复运行是否稳定pass@k、重复试验方差、按任务聚类失败
Cost and latency成功是否满足生产 SLOtokens、tool calls、wall time、cost per solved task、timeout rate
Safety and privacy是否产生敏感数据泄露或危险副作用PII leak、prompt injection success、side-effect audit、escalation rate

可用结论应该类似：在 500 个经人工验证的软件 issue 上，使用模型 X、脚手架 Y、最大步数 Z 和固定环境镜像，agent 解决了 62%；失败集中在跨文件推理和环境配置；平均成本 A，P95 wall time B，PASS_TO_PASS 回归 C。这个结果支持在低风险 issue triage 里灰度试用，不支持给它自主 merge 权限。

11.7 任务选择：什么时候不该用 decoder-only LLM

  • 固定标签分类：encoder-only fine-tuning 或小分类器通常更便宜、更稳定。
  • RAG reranking：cross-encoder 能同时看 query 和 document，decoder-only reranking 成本高且格式不稳。
  • 抽取式答案：答案必须来自原文 span 时，span QA 比自由生成更可控。
  • 结构化输出：依存句法、NER、slot filling 可以用结构化 head 或 constrained decoding。
  • 开放生成和推理：需要新文本、代码、解释、多步推理时，decoder-only LLM 更自然。
⚠️
常见误区：不要把 BERT 和 GPT 说成“只是 mask 不同”。它们的可见性、训练目标、fine-tuning head、输出空间和 serving 成本都不同。

11.8 实验结论：从指标到可判断结论

实验不是“demo 跑通了”，而是回答一个清楚的技术问题，并让读者判断结论在哪些条件下成立。

1. 把问题写成可检验命题

  • 固定 token budget 下，seq_len=128 是否比 seq_len=64 降低 validation PPL？tokens/s 和 step time 代价是多少？
  • DPO 是否提高偏好 win rate，同时保持基础任务准确率和拒答边界？KL、输出长度、失败类型怎么变？
  • RAG top-k 从 3 到 8 是否提高 pass rate？对 P95 TTFT、prompt tokens、context packing miss 和成本有什么影响？
  • 并发从 1 到 8 时，P95/P99 是否仍满足 SLO？瓶颈是 queue、prefill、decode、retrieval 还是 post-processing？

2. 结论结构要覆盖模型、系统和评估

每个结论至少要有 problem、baseline、method、metrics、ablation、failure analysis、decision。负结果也可以是强结论：如果 seq_len=128 只让 PPL 降 0.03，但 tokens/s 掉 35%，三种 seed 波动约 0.05，那么结论可以是“当前设置不值得采用更长序列”。

3. 绑定复现路径和限制

写清数据版本、commit、运行命令、seed、硬件、模型配置、checkpoint/service 配置、评估集和压测参数。同时写清不能外推的地方：小语料不代表 trillion-token 训练，本地 MockEngine 不代表真实 GPU serving，小知识库 RAG 不代表企业知识库。

11.10 本章小结

  • RNN/LSTM 解释了序列递推、BPTT 和长距离依赖问题。
  • Dependency parsing 是离散结构预测，不等于 attention heatmap。
  • Seq2Seq attention 把 source sequence 和 target prefix 分开建模，是 encoder-decoder 的基本形式。
  • BERT/encoder-only 适合分类、rerank、embedding、token classification 和抽取式 QA。
  • BLEU、ROUGE、EM/F1、span F1、UAS/LAS 各自测不同任务结构。
  • 现代 LLM 评估必须写清数据、推理设置、judge 偏差、安全拒答、统计不确定性和结论边界。
  • Agent 评估必须保留 trace，不能只看最终答案。
  • 选模型家族要从输出空间和错误代价出发，而不是默认用最大 decoder-only LLM。
