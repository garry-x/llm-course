2.1 从 One-Hot 到稠密嵌入——为什么需要分布式表示

第 1 章将文本转化为 token 序列 ，其中每个 。现在的问题是：如何将这个整数转化为神经网络可以处理的向量？

One-Hot 编码及其问题

最直接的想法是 One-Hot 编码：将 token 映射为一个长度为 的向量，第 位为 1，其余为 0。例如词表大小 5 时，"apple"（token 3）编码为 。

但 One-Hot 有三个根本性问题：

1. 维度灾难（Curse of Dimensionality）。典型词表 ，每个 token 就是一个 50000 维的向量。存储一个 batch（32 x 1024）的 One-Hot 需要 个 float——约 6.4 GB，完全不可接受。这不仅仅是存储问题：在高维空间中，所有点之间的距离趋于相等，"远近"概念消失（这是维度灾难在距离度量上的体现）。

2. 语义"零亲"（Semantic Flatness）。任意两个不同的 One-Hot 向量之间余弦相似度为 0。"king"和"queen"与"king"和"apple"在 One-Hot 空间中一样"不相似"——模型无法从表示中提取任何语义关系。

3. 参数效率极低。将 One-Hot 乘以一个 的权重矩阵等价于查表选择第 行。那为什么不直接查表呢？

分布式表示：核心突破

解决方案是可学习的稠密嵌入：用一个矩阵 （其中 ）将 token 映射到低维连续空间：

这就是 Token Embedding——等价于查表取第 行。关键区别在于：嵌入向量是可训练的。通过语言模型训练，语义相近的 token 会获得相近的向量。

分布假说（Distributional Hypothesis）

这是分布式表示的理论基础——"You shall know a word by the company it keeps"（Firth, 1957）。语义相近的词出现在相似的上下文中，因此它们应该具有相似的向量表示。在嵌入空间中：

  • "king" 和 "queen" 出现的位置（上下文）有大量重叠 → 向量距离近
  • "king" 和 "apple" 的上下文基本不重叠 → 向量距离远
  • 类比推理： ——语义关系被编码为向量空间的线性变换
🔑
参数量：嵌入矩阵 。GPT-2 124M 中 ，嵌入层约 3860 万参数——占模型总参数的 31%！嵌入矩阵通常与 LM Head（输出投影）共享权重（weight tying），进一步复用参数。

2.3 置换等变性——为什么位置编码不可或缺

嵌入层将每个 token 映射为一个固定的向量。这意味着 "dog bite" 和 "bite dog" 会输出完全相同的向量序列——模型无法区分"狗咬人"和"人咬狗"。

这源于 Self-Attention 的置换等变性（Permutation Equivariance）。更精确地说，Self-Attention 的输出满足：

置换等变性

其中 是一个置换矩阵。交换输入 token 的顺序，输出只会对应交换——每个位置的内容表示不变。注意力层对序列顺序完全是"无感"的。

为什么 Self-Attention 是置换等变的？

从公式看，注意力分数的计算 涉及所有 token 对之间的点积。如果置换输入行（token 顺序），Q 和 K 的行也相应置换，但 的值只取决于 token 和 的内容，不取决于它们的位置。因此输出行也被置换——每个 token 的输出不变，只换位置。

这意味着没有位置编码时：

  • "我打你"和"你打我"在模型内部表示完全等价
  • "第 1 个 token"和"第 1000 个 token"会得到相同处理——没有位置衰减
  • 模型无法利用词序这一最基本的语法信号

解决方案：向每个 token 的表示中加入位置信号，打破置换对称性。主流的方案分为两类：

  • 加性编码：在嵌入向量上加位置向量，如 Sinusoidal、Learned PE
  • 旋转编码：修改注意力计算，在 Q/K 中编码位置，如 RoPE
🔑
直觉：嵌入层给了每个 token 一个"身份证"（语义身份），位置编码给了每个 token 一个"座位号"（它在序列中的位置）。两者相加才是完整的输入表示。

2.4 正弦位置编码——多尺度频率的直觉

Sinusoidal Positional Encoding（Vaswani et al., 2017）是 Transformer 原始论文中的位置编码方案。它用不同频率的正弦和余弦函数为每个位置生成固定的编码向量。

为什么这样设计？

多尺度频率：低维度（小 ）使用高频率——sin/cos 波形变化快，相邻位置的编码差异大，适合编码局部位置信息。高维度（大 ）使用低频率——变化缓慢，适合编码全局位置信息。这种设计模仿了傅里叶特征：将位置编码在不同频率的基函数上，模型可以学习选择性地关注高频或低频维度。

无训练参数 + 可外推：Sinusoidal 是固定函数，不需要学习，且对任意长度（包括训练时未见过的）都有定义。

2.5 编程练习 2：实现 Sinusoidal Positional Encoding

🖥️ 练习 2：SinusoidalEncoding 模块

任务：实现 SinusoidalEncoding 模块，为输入序列生成固定频率的正弦位置编码。

要求：

  • 继承 nn.Module，内部无可训练参数
  • 创建形状为 (max_len, d_model) 的编码矩阵，用 register_buffer 持久化
  • forward(x) 返回 x + pe[:, :seq_len, :]（广播到 batch 维度）
  • 频率计算：div_term = exp(-(2i) * log(10000) / d_model) 用于数值稳定性
  • 偶数列用 sin，奇数列用 cos
# 预期用法
pe = SinusoidalEncoding(d_model=512, max_len=8192)
x = torch.randn(2, 128, 512)
out = pe(x)
assert out.shape == (2, 128, 512)
# 验证：位置 0 和位置 1 的编码不同
assert not torch.allclose(out[0, 0], out[0, 1])
import torch import torch.nn as nn import math class SinusoidalEncoding(nn.Module): """正弦位置编码——固定函数，无需训练参数。""" def __init__(self, d_model, max_len=8192): super().__init__() pe = torch.zeros(max_len, d_model) pos = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1) # 计算频率: 1 / (10000^(2i/d)) i = torch.arange(0, d_model, 2).float() div_term = torch.exp(i * (-math.log(10000.0) / d_model)) pe[:, 0::2] = torch.sin(pos * div_term) # 偶数列 pe[:, 1::2] = torch.cos(pos * div_term) # 奇数列 self.register_buffer("pe", pe.unsqueeze(0)) def forward(self, x): return x + self.pe[:, :x.size(1), :] pe = SinusoidalEncoding(512) x = torch.randn(2, 128, 512) out = pe(x) print(f"输出形状: {out.shape}") print(f"训练参数: {sum(p.numel() for p in pe.parameters())}") # 输出: (2, 128, 512), 训练参数: 0

💡 register_buffer 的作用：将 pe 注册为持久化缓冲区——它会随模型一起保存到 checkpoint 中（.state_dict()），且被 .to(device) 自动移动，但在训练中不更新梯度。这使得编码表随模型"带"着但不会影响反向传播。

2.6 RoPE 深度理论——旋转中编码相对位置

旋转位置编码（RoPE）由 Su et al.（2021）提出，是 Llama 2/3、Mistral、DeepSeek 等主流模型的首选方案。它的核心创新在于：不再将位置信息加到嵌入向量上，而是修改 Q 和 K 的表示，使点积自动编码相对位置。

RoPE 的优良性质

  • 相对位置：推导证明点积只依赖于
  • 范数保持：，因为旋转矩阵是正交的（）
  • 位置衰减：当 增大时，旋转角度增大，向量的对齐程度下降，点积上限变小——这是自然的远程衰减（distant decay）
  • 不影响 V：RoPE 只修改 Q 和 K，不影响 V。因为位置信息只影响"谁关注谁"，不影响"关注到什么内容"
🔑
与 Sinusoidal 的本质区别：Sinusoidal 在嵌入层加位置信息（破坏置换等变性），RoPE 在注意力层的 Q/K 上编码位置（利用置换等变性中的点积）。RoPE 的数学基础是旋转矩阵的群性质，不是简单的"加一个向量"。

2.7 编程练习 3：实现 RoPE

🖥️ 练习 3：RoPE 模块

任务：实现 RoPE 的旋转操作。给定 Q 和 K 张量，对每个位置施加对应角度的旋转。

要求：

  • 输入 Q, K 形状 (batch, n_heads, seq_len, d_head)
  • 预计算 cos 和 sin 表，形状 (1, max_len, 1, d_head//2)
  • 实现 _rotate_half(x)：将 x[..., d_head/2:] 取反后与 x[..., :d_head/2] 交换
  • 应用公式：q_out = q * cos + rotate_half(q) * sin
  • 返回 (q_out, k_out)，形状不变
# 预期用法
rope = RoPE(d_model=64, max_len=128)
q = torch.randn(2, 8, 10, 64)   # (batch, n_heads, seq_len, d_head)
k = torch.randn(2, 8, 10, 64)
q_rot, k_rot = rope(q, k)
assert q_rot.shape == q.shape
assert k_rot.shape == k.shape
# 验证：旋转不改变范数
assert torch.allclose(q.norm(dim=-1), q_rot.norm(dim=-1), atol=1e-5)
import torch import torch.nn as nn import math class RoPE(nn.Module): """旋转位置编码 (Rotary Position Embedding)。""" def __init__(self, d_model, max_len=8192, base=10000.0): super().__init__() # 计算频率 theta_i theta = torch.pow(base, -torch.arange(0, d_model, 2).float() / d_model) pos = torch.arange(max_len).float().unsqueeze(1) # (max_len, 1) angles = pos * theta.unsqueeze(0) # (max_len, d_model/2) self.register_buffer("cos", cos := torch.cos(angles).unsqueeze(0).unsqueeze(2)) self.register_buffer("sin", sin := torch.sin(angles).unsqueeze(0).unsqueeze(2)) def _rotate_half(self, x): """将 x 的后半部分取反并与前半部分交换位置。""" x1, x2 = x.chunk(2, dim=-1) return torch.cat((-x2, x1), dim=-1) def forward(self, q, k, pos_ids=None): batch, n_heads, seq_len, d_head = q.shape # 取对应长度的 cos/sin cos = self.cos[:, :seq_len, :, :d_head//2] sin = self.sin[:, :seq_len, :, :d_head//2] # 应用旋转 q_out = q * cos + self._rotate_half(q) * sin k_out = k * cos + self._rotate_half(k) * sin return q_out, k_out # 验证范数保持 rope = RoPE(d_model=64) q = torch.randn(2, 8, 10, 64) q_rot, _ = rope(q, q) assert torch.allclose(q.norm(dim=-1), q_rot.norm(dim=-1), atol=1e-5) print("范数保持 ✅")

2.9 编程练习 4：验证 RoPE 的相对位置性质

🖥️ 练习 4：验证 只依赖于

任务：编写验证代码，证明 RoPE 旋转后的 q_m 和 k_n 的点积只与 m-n 有关。

要求：

  • 构造单个固定的 q 和 k 向量（没有位置信息，形状 (1, 1, 1, d)）
  • 将它们复制到不同的位置 m 和 n，施加 RoPE 旋转
  • 计算所有 (m, n) 配对的中点积得分，存储为 (seq_len, seq_len) 矩阵
  • 验证：对角线偏移相同的元素值相等——即 S[m][n] 只取决于 m-n
  • 验证：对角线上值最大，远离对角线时值衰减
  • 不能使用 for 循环比较——用张量操作验证
# 预期结果: 打印出 8x8 的注意力分数矩阵
# 应观察到: 所有主对角线 (m=n) 值相等, 所有次对角线 (m=n+1) 值相等, ...
# 形成一个 Toeplitz 矩阵（Toeplitz matrix = 每条对角线上的元素都相同）
# ┌─────────────────────────────────────────────┐
# │ 1.00  0.98  0.92  0.84  ...                │  ← m-n=0: 最大
# │ 0.98  1.00  0.98  0.92  ...                │  ← m-n=1: 稍小
# │ 0.92  0.98  1.00  0.98  ...                │  ← m-n=2: 更小
# │ 0.84  0.92  0.98  1.00  ...                │
# │ ...                                         │
# └─────────────────────────────────────────────┘
# 每一条对角线上的值应完全相等。
import torch import torch.nn as nn import math def verify_rope_relative_property(d_model=64, seq_len=8): """验证 RoPE 的点积只依赖于相对位置。""" rope = RoPE(d_model=d_model) # 构造单个固定的 q 和 k (无位置信息) q = torch.randn(1, 1, 1, d_model) # (1, 1, 1, d) k = torch.randn(1, 1, 1, d_model) # 扩展到 seq_len 个位置: 每个位置的 q/k 向量相同 q_expanded = q.expand(1, 1, seq_len, -1) # (1, 1, seq_len, d) k_expanded = k.expand(1, 1, seq_len, -1) # 施加 RoPE: 不同位置的相同向量被旋转不同角度 q_rot, k_rot = rope(q_expanded, k_expanded) # 计算所有 (m, n) 对的点积 scores = torch.matmul(q_rot, k_rot.transpose(-2, -1)) # (1, 1, seq_len, seq_len) scores = scores.squeeze() # (seq_len, seq_len) print(f"注意力分数矩阵 ({seq_len}x{seq_len}):") # 显示到 4 位小数 for row in scores: print(" " + " ".join(f"{v:.4f}" for v in row)) # 验证 Toeplitz 性质: 每条对角线上的值应相等 print("\n验证对角线条目一致性:") all_equal = True for offset in range(1, seq_len): diag = scores.diagonal(offset=offset) diag_neg = scores.diagonal(offset=-offset) # 检查同一条对角线上的值是否都相等 if not torch.allclose(diag, diag[0].expand_as(diag)): print(f" ✗ 对角 offset={offset}: 值不一致!") all_equal = False else: print(f" ✓ 对角 offset={offset}: 值={diag[0].item():.4f}") # 检查正负对角是否对称 if not torch.allclose(diag, diag_neg): print(f" ✗ 对角 offset={offset} 正负不对称!") all_equal = False if all_equal: print("\n✅ 相对位置性质成立: Q_m^T K_n 只依赖于 (m-n)!") print("分数矩阵是一个 Toeplitz 矩阵。") # 验证位置衰减: |m-n| 越大, 分数越小 print("\n衰减趋势:") for offset in range(seq_len): val = scores.diagonal(offset=offset)[0].item() print(f" |m-n| = {offset}: 分数 = {val:.4f}") return scores scores = verify_rope_relative_property(d_model=64, seq_len=8)

💡 实验结论：旋转后的 Q 和 K 点积构成一个Toeplitz 矩阵——每条对角线上的值都相等。这意味着注意力分数只取决于 token 之间的相对距离。当 增大时分数逐渐衰减，这正是 RoPE 的"位置衰减"效应——远离的 token 天然更难对齐。

2.11 提示工程（Prompt Engineering）深入

在完成 Embedding + RoPE 的理论和实现后，让我们回到 LLM 应用中最实用的技能：如何写好 prompt。这不是"技巧"，而是对模型概率分布的精确引导。

Zero-Shot vs Few-Shot Prompting

Zero-Shot：仅给出指令，不给示例。"将以下句子翻译成英文：今天天气很好。"——模型根据指令和自身知识完成。

Few-Shot：在指令前给出 2-5 个示例。"输入→输出"对帮助模型理解任务格式和期望风格。GPT-3 论文的核心发现就是 few-shot 能力——模型不需要微调就能通过示例适应新任务。

🔑
为什么 Few-Shot 有效？从注意力机制的角度看，few-shot 示例在 KV Cache 中形成了"任务模板"——后续 token 的注意力会自然地参考这些示例的模式。这等价于一种上下文内的隐式微调。

Chain-of-Thought (CoT) Prompting

在 prompt 中加入"让我们一步步思考"（Let's think step by step），引导模型生成中间推理步骤而非直接给出答案。这是最简单却最有效的 prompt 技巧之一。

CoT 的数学直觉：标准的 next-token prediction 会直接跳到答案 token——但复杂问题需要多步推理。CoT 将推理过程显式化为 token 序列，使得模型在生成中间步骤时能看到自己的推理历史，逐步修正和深化。

System Prompt 与角色设定

ChatGPT 和 Claude API 将 system prompt 作为独立的消息类型，优先级高于用户消息。它设定了模型的"角色"和行为边界。从注意力实现的角度：system prompt 的 token 占据了序列的最开头位置，因为因果注意力中所有后续 token 都可以"看到"前面的 token，system prompt 的影响贯穿整个对话。

提示工程的核心原则

💡
1. 明确任务 + 输出格式 —— 不要把 LLM 当成会读心术
2. Few-shot > Zero-shot —— 2-3 个好示例胜过 100 字描述
3. CoT 对推理任务有效 —— 数学/逻辑/多步问题必加 "Let's think step by step"
4. 越具体越好 —— 给出格式、长度、风格、受众的明确约束
5. System prompt 设置边界 —— 角色、知识范围、安全约束
